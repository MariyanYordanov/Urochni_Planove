### **4. ЗАТВЪРЖДАВАНЕ (30 минути)**

#### **Практическа работа: Създаване на monitoring услуга**

**Задача:**
Създайте системна услуга, която:
1. Проверява дисковото пространство на всеки 5 минути
2. Записва резултата в log файл
3. Изпраща предупреждение при над 80% използване

**Стъпки:**

1. **Създаване на monitoring скрипт:**
```bash
#!/bin/bash
# /opt/monitor/disk_monitor.sh

THRESHOLD=80
LOGFILE="/var/log/disk_monitor.log"

usage=$(df -h / | awk 'NR==2 {print $(NF-1)}' | sed 's/%//')
timestamp=$(date "+%Y-%m-%d %H:%M:%S")

echo "$timestamp - Disk usage: $usage%" >> $LOGFILE

if [ $usage -gt $THRESHOLD ]; then
    echo "$timestamp - WARNING: High disk usage!" >> $LOGFILE
    # Тук може да се добави изпращане на email
fi
```

2. **Създаване на service файл:**
```ini
# /etc/systemd/system/disk-monitor.service
[Unit]
Description=Disk Usage Monitor
After=multi-user.target

[Service]
Type=oneshot
ExecStart=/opt/monitor/disk_monitor.sh

[Install]
WantedBy=multi-user.target
```

3. **Създаване на timer файл:**
```ini
# /etc/systemd/system/disk-monitor.timer
[Unit]
Description=Run disk monitor every 5 minutes

[Timer]
OnBootSec=5min
OnUnitActiveSec=5min

[Install]
WantedBy=timers.target
```

**Дейност на учениците:**
- Създаване на скриптовете
- Конфигуриране на услугата
- Тестване и отстраняване на грешки
- Проверка на логовете

## 8. ПРИЛОЖЕНИЯ

### ПРИЛОЖЕНИЕ 1: SYSTEMCTL CHEAT SHEET

| Команда | Описание |
|---------|----------|
| `systemctl status SERVICE` | Показва статус на услуга |
| `systemctl start SERVICE` | Стартира услуга |
| `systemctl stop SERVICE` | Спира услуга |
| `systemctl restart SERVICE` | Рестартира услуга |
| `systemctl reload SERVICE` | Презарежда конфигурацията |
| `systemctl enable SERVICE` | Активира автостарт |
| `systemctl disable SERVICE` | Деактивира автостарт |
| `systemctl daemon-reload` | Презарежда systemd конфигурацията |
| `systemctl list-units` | Списък на всички units |
| `systemctl list-timers` | Списък на всички timers |
