### **3. ДЕМОНСТРАЦИЯ И ПРАКТИКА (55 минути)**

#### **Част 1: Подготовка на средата (10 минути)**

**Nested Virtualization концепция:**
```
┌─────────────────────────────────────────────────────────────┐
│                 PHYSICAL HOST (Вашият компютър)             │
│  ┌───────────────────────────────────────────────────────┐  │
│  │            VMware Workstation Pro 17.x                │  │
│  │  ┌─────────────────────────────────────────────────┐  │  │
│  │  │              ESXi Virtual Machine               │  │  │
│  │  │   ┌─────────────┐    ┌─────────────┐           │  │  │
│  │  │   │   VM #1     │    │   VM #2     │           │  │  │
│  │  │   │  (Ubuntu)   │    │  (Windows)  │           │  │  │
│  │  │   └─────────────┘    └─────────────┘           │  │  │
│  │  └─────────────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

**Системни изисквания за nested ESXi:**
```
Минимални изисквания (препоръчителни):
├── CPU: 4 vCPU (с VT-x enabled)
├── RAM: 8 GB (минимум 4 GB)
├── Storage: 40 GB (thin provisioned)
├── Network: 1 vNIC (NAT или VMnet)
└── ISO: VMware ESXi 8.0 Installer

Важно: Активирайте "Virtualize Intel VT-x/EPT" 
в настройките на VM!
```

**Практическа задача 1.1: Създаване на ESXi VM**

```
Стъпки в VMware Workstation:

1. File → New Virtual Machine → Custom (advanced)

2. Hardware compatibility: Workstation 17.x

3. Guest OS: VMware ESXi 8

4. Name: ESXi_Lab_[вашето_име]
   Location: C:\VMs\ESXi_Lab

5. Processors: 4 CPUs
   ☑ Virtualize Intel VT-x/EPT or AMD-V/RVI

6. Memory: 8192 MB (8 GB)

7. Network: NAT

8. SCSI Controller: LSI Logic SAS

9. Virtual Disk: 
   - Create new virtual disk
   - 40 GB
   - Store as single file

10. Finish
```

```
След рестарта ще видите DCUI екрана:

╔══════════════════════════════════════════════════════════════╗
║                    VMware ESXi 8.0.0                         ║

**Практическа задача 3.2: Конфигуриране на мрежа**

```
1. Натиснете F2 (Customize System)

2. Въведете root парола: Esxi@Lab2025!

3. Configure Management Network
   │
   ├── Network Adapters
   │   └── Изберете vmnic0
   │
   ├── IPv4 Configuration
   │   ├── Set static IPv4 address
   │   ├── IPv4 Address: 192.168.xxx.100
   │   │   (използвайте адрес от вашата мрежа)
   │   ├── Subnet Mask: 255.255.255.0
   │   └── Default Gateway: 192.168.xxx.1
   │
   └── DNS Configuration
       ├── Primary DNS: 8.8.8.8
       └── Hostname: esxi-lab-[вашето_име]

4. Натиснете Esc → Y (Apply changes and restart)
```

```
В vSphere Host Client:

1. Navigator → Virtual Machines → Create/Register VM

2. Select creation type:
   ○ Create a new virtual machine
   [Next]

3. Select a name and guest OS:
   Name: Ubuntu_Test_VM
   Compatibility: ESXi 8.0 virtual machine
   Guest OS family: Linux
   Guest OS version: Ubuntu Linux (64-bit)
   [Next]

4. Select storage:
   ○ datastore1 (default)
   [Next]

5. Customize settings:
   ┌─────────────────────────────────────────┐
   │ CPU:        2                           │
   │ Memory:     2048 MB                     │
   │ Hard disk:  20 GB (Thin Provision)      │

```

   CD/DVD Drive → Datastore ISO file
   Browse → iso/ubuntu-24.04-live-server-amd64.iso

6. Save

### ПРИЛОЖЕНИЕ 2: ЧЕСТО СРЕЩАНИ ПРОБЛЕМИ И РЕШЕНИЯ

```
╔══════════════════════════════════════════════════════════════╗
║                     TROUBLESHOOTING GUIDE                    ║
╚══════════════════════════════════════════════════════════════╝

ПРОБЛЕМ 1: "This host does not support virtualizing real mode"
──────────────────────────────────────────────────────────────
Причина: VT-x не е активиран
Решение:
1. Изключете VM
2. VM Settings → Processors
3. ☑ Virtualize Intel VT-x/EPT
4. Ако не помага → BIOS → Enable VT-x

ПРОБЛЕМ 2: ESXi не вижда мрежов адаптер
──────────────────────────────────────────────────────────────
Причина: Несъвместим виртуален NIC
Решение:
1. VM Settings → Network Adapter
2. Изберете: VMXNET 3 или E1000E
3. Рестартирайте VM

ПРОБЛЕМ 3: Няма достъп до web interface
──────────────────────────────────────────────────────────────
Причина: Firewall блокира или грешен IP
Решение:
1. Проверете IP в DCUI екрана
2. ping [ESXi_IP] от host
3. Ако ping работи → проверете firewall
4. Ако ping не работи → проверете VMnet настройки

ПРОБЛЕМ 4: "Permission denied" при login
──────────────────────────────────────────────────────────────
Причина: Грешна парола
Решение:
1. Внимавайте за Caps Lock
2. Паролата е case-sensitive
3. Опитайте през DCUI (F2)

ПРОБЛЕМ 5: VM не стартира - "Not enough memory"
──────────────────────────────────────────────────────────────
Причина: Недостатъчно RAM на ESXi host
Решение:
1. Намалете RAM на вътрешната VM
2. Или увеличете RAM на ESXi VM
3. Workstation → Settings → Memory → 12288 MB

ПРОБЛЕМ 6: Бавна производителност
──────────────────────────────────────────────────────────────
Причина: Nested virtualization overhead
Решение:
1. Увеличете vCPU и RAM
2. Използвайте SSD за storage
3. Намалете броя на вътрешните VM

ПРОБЛЕМ 7: Cannot find ESXi installation media
──────────────────────────────────────────────────────────────
Причина: ISO не е свързан
Решение:
1. VM Settings → CD/DVD
2. ☑ Connected
3. Use ISO image file → Browse → ESXi ISO
