### **3. НОВ МАТЕРИАЛ (45 минути)**

#### **Част 1: Еволюция към многоядрени процесори (10 мин)**

**Историческо развитие:**
```
2005: Intel Pentium D - първи dual-core
2006: Intel Core 2 Duo - истински dual-core
2008: Intel Core i7 - quad-core + HT
2011: AMD FX-8150 - 8 ядра
2017: AMD Ryzen - модерна многоядрена ера
2021: Intel Alder Lake - хибридна архитектура
2025: 32+ ядра в desktop процесори
```

**Причини за многоядрени дизайни:**
1. Физически ограничения на честотата
2. Power wall - консумация при високи честоти
3. Нарастващ паралелизъм в софтуера
4. По-добра енергийна ефективност

#### **Част 2: Intel Core архитектура (15 мин)**

**Alder Lake/Raptor Lake (12th/13th/14th gen):**

```
Хибридна архитектура:
┌──────────────────────────────────┐
│         Intel Core i9-14900K      │
├──────────────────────────────────┤
│   8 P-cores (Performance)         │
│   - Hyper-Threading (16 threads)  │
│   - High IPC                      │
│   - 5.8 GHz boost                 │
├──────────────────────────────────┤
│   16 E-cores (Efficiency)         │
│   - No HT (16 threads)            │
│   - Lower power                   │
│   - 4.4 GHz boost                 │
├──────────────────────────────────┤
│   Total: 24 cores, 32 threads     │
│   L3 Cache: 36MB                  │
│   Process: Intel 7 (10nm)         │
└──────────────────────────────────┘
```

**Thread Director:**
- Hardware scheduler
- OS integration (Windows 11)
- Оптимално разпределение на задачи
- QoS (Quality of Service)

**Демонстрация в Task Manager:**
```powershell
# PowerShell команда за процесорна информация
Get-WmiObject Win32_Processor | Select-Object Name, NumberOfCores, NumberOfLogicalProcessors

# Виждане на P и E cores
wmic cpu get NumberOfCores, NumberOfEnabledCore, NumberOfLogicalProcessors
```

#### **Част 3: AMD Ryzen архитектура (15 мин)**

**Zen 4 Architecture (Ryzen 7000/9000):**

```
Chiplet Design:
┌─────────────────────────────────────┐
│       AMD Ryzen 9 7950X             │
├─────────────────────────────────────┤
│  CCD 1 (Chiplet)    CCD 2 (Chiplet) │
│  ┌──────────┐       ┌──────────┐    │
│  │ 8 cores  │       │ 8 cores  │    │
│  │ 32MB L3  │       │ 32MB L3  │    │
│  └──────────┘       └──────────┘    │
├─────────────────────────────────────┤
│         I/O Die (6nm)                │
│  - Memory Controller                 │
│  - PCIe 5.0 Controller              │
│  - USB, SATA                        │
├─────────────────────────────────────┤
│  Total: 16 cores, 32 threads (SMT)  │
│  L3 Cache: 64MB                     │
│  Process: TSMC 5nm (CCD)            │
└─────────────────────────────────────┘
```

**Infinity Fabric:**
- Свързва chiplets
- Скорост зависи от RAM честота
- FCLK = MCLK/2 (оптимално)

**3D V-Cache технология:**
- AMD Ryzen 7 7800X3D
- +64MB L3 cache (96MB total)
- Отлично за gaming

#### **Част 4: Сравнителен анализ (5 мин)**

**Intel vs AMD:**

| Характеристика | Intel Core | AMD Ryzen |
|----------------|------------|-----------|
| Архитектура | Monolithic/Hybrid | Chiplet |
| Cores тип | P-cores + E-cores | Uniform cores |
| Threading | HT на P-cores | SMT на всички |
| Cache | Shared L3 | L3 per CCD |
| Platform | LGA1700 | AM5 |
| DDR Support | DDR4/DDR5 | DDR5 only |
| PCIe | 5.0 | 5.0 |
| Power | До 253W | До 170W |

### ПРИЛОЖЕНИЕ 2: PROCESS AFFINITY CALCULATOR

```python
# Python скрипт за изчисляване на affinity маска
def calculate_affinity(cores):
    """Изчислява hex маска за дадени ядра"""
    mask = 0
    for core in cores:
        mask |= (1 << core)
    return hex(mask)

# Примери:
# Cores 0-3: calculate_affinity([0,1,2,3]) = 0xF
# Cores 0-7: calculate_affinity(range(8)) = 0xFF
# Even cores: calculate_affinity([0,2,4,6]) = 0x55
```
