#### **ЧАС 1: CLOUD SERVICE MODELS И SECURITY IMPLICATIONS (40 мин)**

**Infrastructure as a Service (IaaS) Security:**

**Customer Responsibility Scope:**
В IaaS model, customer assumes significant security responsibility for operating system level и above.

**Security Control Matrix:**
```
IaaS Security Responsibility Distribution:
Cloud Provider Responsibilities:
├── Physical security (data centers, hardware)
├── Network infrastructure (underlying networking)
├── Hypervisor security (virtualization layer)
├── Host operating system (where applicable)
└── Service availability (uptime, redundancy)

Customer Responsibilities:
├── Guest operating system security (patching, hardening)
├── Application security (code, configuration)
├── Identity и access management (user accounts, permissions)
├── Data protection (encryption, backup, access controls)
├── Network controls (firewalls, segmentation, monitoring)
└── Compliance adherence (regulatory requirements)
```

**IaaS Security Implementation:**

**Virtual Machine Hardening:**
- **OS Baseline Configuration:** CIS benchmarks, security hardening guides
- **Patch Management:** Automated patching strategies, vulnerability assessment
- **Endpoint Protection:** Anti-malware, host-based intrusion detection
- **Configuration Management:** Infrastructure as Code, desired state management

**Network Security Architecture:**
```
Cloud Network Security Design:
Virtual Private Cloud (VPC) Structure:
├── Public Subnet: Internet-facing resources (load balancers, NAT gateways)
├── Private Subnet: Application tier (web servers, API gateways)
├── Database Subnet: Data tier (databases, sensitive processing)
└── Management Subnet: Administrative access (bastion hosts, monitoring)

Security Controls:
├── Network Access Control Lists (NACLs): Subnet-level traffic filtering
├── Security Groups: Instance-level firewall rules
├── VPN Connections: Secure site-to-site connectivity
├── Web Application Firewalls: Application-layer protection
└── DDoS Protection: Distributed denial of service mitigation
```

**Platform as a Service (PaaS) Security:**

**Shared Security Model:**
PaaS abstracts infrastructure management while maintaining application-level security responsibility.

**Developer Security Responsibilities:**
- **Application Code Security:** Secure coding practices, vulnerability remediation
- **Data Security:** Encryption implementation, access control design
- **Identity Integration:** Authentication и authorization implementation
- **API Security:** Secure API design, rate limiting, input validation

**PaaS Security Advantages:**
- **Automated Patching:** Platform provider handles OS и middleware updates
- **Built-in Security Services:** Integrated authentication, monitoring, compliance tools
- **Scalability Security:** Automatic scaling с security controls maintained
- **Compliance Certifications:** Provider-managed compliance framework adherence

Lambda Drift Detection Function:
import boto3
import json
import os

def lambda_handler(event, context):
    cloudformation = boto3.client('cloudformation')
    sns = boto3.client('sns')
    
    # Get all stacks
    paginator = cloudformation.get_paginator('list_stacks')
    
    drift_results = []
    
    for page in paginator.paginate(StackStatusFilter=['CREATE_COMPLETE', 'UPDATE_COMPLETE']):
        for stack in page['StackSummaries']:
            stack_name = stack['StackName']
            
            # Trigger drift detection
            response = cloudformation.detect_stack_drift(StackName=stack_name)
            detection_id = response['StackDriftDetectionId']
            
            # Wait for detection to complete
            waiter = cloudformation.get_waiter('stack_drift_detection_complete')
            waiter.wait(StackName=stack_name, StackDriftDetectionId=detection_id)
            
            # Get drift results
            drift_status = cloudformation.describe_stack_drift_detection_status(
                StackDriftDetectionId=detection_id
            )
            
            if drift_status['StackDriftStatus'] == 'DRIFTED':
                drift_results.append({
                    'StackName': stack_name,
                    'DriftStatus': drift_status['StackDriftStatus'],
                    'DriftedStackResourceCount': drift_status.get('DriftedStackResourceCount', 0)
                })
    
    # Send notification if drift detected
    if drift_results:
        message = {
            'DriftedStacks': drift_results,
            'DetectionTime': context.aws_request_id
        }
        
        sns.publish(
            TopicArn=os.environ['SNS_TOPIC_ARN'],
            Message=json.dumps(message),
            Subject='Infrastructure Drift Detected'
        )
    
    return {
        'statusCode': 200,
        'body': json.dumps(f'Checked drift for {len(drift_results)} stacks')
    }
