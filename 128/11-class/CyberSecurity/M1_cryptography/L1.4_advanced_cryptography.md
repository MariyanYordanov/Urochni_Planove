#### **Час 16: Post-Quantum Cryptography Preview (40 мин)**

**Теоретична част (25 мин):**

**🔬 Quantum Computing Threat Assessment (10 мин)**

**Quantum Computing Fundamentals:**

**Classical vs Quantum Computing:**
- **Classical bits** - 0 или 1, deterministic processing
- **Quantum qubits** - superposition на 0 и 1 simultaneously
- **Quantum parallelism** - process multiple possibilities simultaneously
- **Quantum entanglement** - correlated quantum states
- **Decoherence challenges** - quantum states fragile, require extreme conditions

**Shor's Algorithm Impact:**
- **Integer factorization** - efficiently factor large integers
- **Discrete logarithms** - solve discrete log problems quickly
- **RSA vulnerability** - breaks RSA encryption и signatures
- **ECC vulnerability** - breaks elliptic curve cryptography
- **Timeline estimates** - cryptographically-relevant quantum computers by 2030-2040

**Grover's Algorithm Impact:**
- **Symmetric key search** - quadratic speedup in key search
- **AES impact** - AES-128 → 64-bit effective security, AES-256 → 128-bit
- **Hash functions** - reduced security но still practical
- **Mitigation** - double key sizes за equivalent security

**Current Quantum Computing Status:**
- **IBM Quantum** - 100+ qubit systems, limited coherence time
- **Google Sycamore** - demonstrated quantum supremacy за specific problems
- **Research progress** - rapid advancement във quantum error correction
- **Government investment** - massive funding от US, China, EU за quantum research

**🛡️ Post-Quantum Algorithm Categories (8 мин)**

**NIST Post-Quantum Cryptography Standardization:**

**Selected Algorithms (2022):**

**Public Key Encryption:**
- **CRYSTALS-Kyber** - lattice-based encryption, good performance
- **Key sizes** - larger than current algorithms, но manageable
- **Security basis** - learning with errors (LWE) problem
- **Applications** - key encapsulation mechanism (KEM) за hybrid encryption

**Digital Signatures:**
- **CRYSTALS-Dilithium** - lattice-based signatures, efficient verification
- **FALCON** - lattice-based signatures, compact signatures
- **SPHINCS+** - hash-based signatures, conservative security approach

**Mathematical Foundations:**

**Lattice-Based Cryptography:**
- **Lattice problems** - shortest vector problem (SVP), closest vector problem (CVP)
- **Learning with Errors (LWE)** - computational assumption за security
- **Advantages** - good performance, well-studied mathematical foundation
- **Challenges** - larger key sizes, implementation complexity

**Hash-Based Signatures:**
- **One-time signatures** - secure signatures based только на hash functions
- **Merkle trees** - enable multiple signatures від single public key
- **Advantages** - provably secure, conservative approach
- **Limitations** - stateful signatures, limited signature count

**Code-Based Cryptography:**
- **Error-correcting codes** - security based на decoding random linear codes
- **Historical foundation** - oldest post-quantum approach (McEliece 1978)
- **Advantages** - fast encryption/decryption operations
- **Challenges** - very large key sizes, implementation complexity

**Multivariate Cryptography:**
- **Multivariate polynomials** - solving systems на polynomial equations
- **Compact signatures** - smaller signature sizes than other approaches
- **Research status** - fewer candidates survived NIST evaluation
- **Specialized applications** - may be useful за specific use cases

**Technical Migration Challenges:**

**Key Size Impact:**
- **Current RSA-2048** - 256 bytes public key, 1024 bytes private key

- **Network overhead** - increased bandwidth requirements za certificates
- **Storage impact** - larger certificate stores, key management complexity
- **Performance degradation** - slower cryptographic operations

**Implementation Complexity:**
- **Algorithm diversity** - multiple post-quantum algorithms vs single RSA/ECC
- **Hybrid approaches** - combine classical и post-quantum za transition period
- **Crypto agility** - systems must support algorithm upgrades
- **Testing requirements** - extensive validation на new implementations
- **Side-channel resistance** - new algorithms need protection against timing attacks
