**Теоретична част (22 мин):**

**Information Governance Framework (8 мин)**

**Данните като бизнес актив:**
Информацията представлява значителна бизнес стойност, която изисква структуриран подход за защита, подобен на физическите активи.

**Управление на жизнения цикъл на информацията:**
- **Creation** - процеси за генериране, придобиване и въвеждане на данни
- **Processing** - анализ, трансформация, комбиниране с други данни
- **Distribution** - дейности по споделяне, предаване и публикуване
- **Storage** - архивиране, backup, дългосрочно съхранение
- **Disposal** - сигурно изтриване, процедури в края на жизнения цикъл

**Компоненти на управлението:**
- **Data ownership** - присвояване на отговорност за управление на данни
- **Policy framework** - установяване на изисквания за обработка на данни
- **Control implementation** - технически и процедурни защитни мерки
- **Monitoring и compliance** - непрекъснат надзор и одит
- **Continuous improvement** - адаптиране на политиките въз основа на опита

**Регулаторни задължения:**
- **GDPR** - изисквания за защита на лични данни
- **PCI DSS** - стандарти за сигурност на данни от платежни карти
- **HIPAA** - защита на здравна информация (американски стандарт с глобално влияние)
- **SOX** - изисквания за целостта на финансовите данни
- **Отраслови специфики** - секторни регулации и стандарти

**Classification Schemes и нива (7 мин)**

**Традиционна правителствена класификация:**
- **Top Secret** - изключително тежки щети на националната сигурност
- **Secret** - сериозни щети на националната сигурност
- **Confidential** - щети на националната сигурност
- **Restricted** - нежелани ефекти при разкриване
- **Unclassified** - не е необходима специална защита

**Бизнес класификация на данните:**
```
Level 4 - HIGHLY CONFIDENTIAL
Examples: Trade secrets, M&A information, executive compensation
Protection: Highest security controls, minimal access
Handling: Encrypted storage, secure transmission only
Retention: Legal requirements, secure disposal

Level 3 - CONFIDENTIAL
Examples: Customer data, financial records, strategic plans
Protection: Strong security controls, role-based access
Handling: Encryption required, audit trails mandatory
Retention: Business requirements, compliance obligations

Level 2 - INTERNAL
Examples: Employee handbooks, internal procedures, project data
Protection: Standard security controls, employee access
Handling: Normal business processes, internal sharing OK
Retention: Business value-based decisions

Level 1 - PUBLIC
Examples: Marketing materials, public announcements, published research
Protection: Basic integrity controls, public availability
Handling: No special restrictions, normal distribution
Retention: Business или legal requirements only
```

**Критерии за класификация на данните:**
- **Business impact** - потенциални щети от неоторизирано разкриване
- **Legal requirements** - регулаторни задължения за конкретни типове данни
- **Competitive advantage** - стратегическа стойност за организацията
- **Personal privacy** - индивидуални права и очаквания за поверителност
- **Operational importance** - критичност за ежедневните бизнес операции

#### **Час 2: Data Loss Prevention (DLP) Technologies (40 мин)**

**Теоретична част (24 мин):**

**DLP архитектура и компоненти (10 мин)**

**Предназначение на Data Loss Prevention:**
DLP решенията наблюдават, засичат и предотвратяват неоторизирано предаване, достъп или използване на чувствителни данни в организацията.

**Модели за внедряване на DLP:**

**Network-Based DLP:**
- **Gateway deployment** - наблюдава целия мрежов трафик, напускащ организацията
- **Protocol support** - имейл, уеб, FTP, незабавни съобщения
- **Content inspection** - задълбочена инспекция на пакети за чувствителни данни
- **Policy enforcement** - блокиране, карантина или криптиране на подозрителни предавания
- **Incident logging** - изчерпателна одитна следа за всички нарушения на политиките

**Endpoint DLP:**
- **Agent-based** - софтуер, инсталиран на работни станции и сървъри на потребителите
- **File monitoring** - проследяване на файлови операции, копиране, изтриване
- **Application control** - наблюдение на използването на данни в конкретни приложения
- **Removable media** - контрол на USB устройства, записване на CD/DVD
- **Screen capture prevention** - блокиране на screenshots на чувствителна информация

**Storage DLP:**
- **Data discovery** - сканиране на файлови системи и бази данни за чувствително съдържание
- **Content classification** - автоматично категоризиране на откритите данни
- **Access monitoring** - проследяване кой осъществява достъп до чувствителни файлове
- **Encryption enforcement** - автоматично криптиране на чувствителни данни в покой
- **Compliance reporting** - генериране на отчети за регулаторни изисквания

**Cloud DLP:**
- **SaaS integration** - наблюдение на облачни приложения (Office 365, Google Workspace)
- **API connectivity** - интеграция с облачни услуги за съхранение
- **Shadow IT discovery** - идентифициране на неоторизирано използване на облачни услуги
- **Data governance** - прилагане на политики в хибридни среди

**Технологии за засичане в DLP (8 мин)**

**Техники за анализ на съдържание:**

**Pattern Matching:**
- **Regular expressions** - засичане на ЕГН, кредитни карти, национални идентификатори
- **Keyword dictionaries** - търсене на конкретни термини или фрази
- **Data formats** - идентифициране на структурирани шаблони за данни
- **Custom patterns** - специфични за организацията формати на чувствителни данни

*Български примери:*
```
EGN (ЕГН) Pattern: \d{10}
Bulgarian IBAN: BG\d{2}[A-Z]{4}\d{14}
Bulgarian Mobile: (\+359|0)[8-9]\d{8}
```

**Machine Learning класификация:**
- **Document classification** - категоризиране на документи по вид на съдържание
- **Language detection** - идентифициране на езика на документа
- **Sentiment analysis** - засичане на поверителни комуникации
- **Anomaly detection** - идентифициране на необичайни модели на достъп до данни
- **User behavior analytics** - засичане на индикатори за вътрешни заплахи

**Техники за „отпечатъци" (Fingerprinting):**
- **Exact data matching** - съпоставяне с известни чувствителни бази данни
- **Partial document matching** - идентифициране на откъси от поверителни документи
- **Statistical fingerprinting** - анализ на статистическите свойства на документа
- **Indexed document matching** - търсене в репозитории с документи

**Контекстуален анализ:**
- **Metadata examination** - свойства на файлове, дати на създаване, автори
- **Communication context** - получатели на имейли, съдържание на съобщения
- **User role analysis** - сравняване на достъп спрямо служебните задължения
- **Time-based patterns** - засичане на необичайно време или честота на достъп

**Рамка на DLP политики (6 мин)**

**Компоненти на политиката:**

**Интеграция с класификацията на данните:**
```
Policy Structure:
IF (Data Classification = "HIGHLY CONFIDENTIAL")
AND (Action = "Email External")
THEN (Block + Alert Security Team)

IF (Data Classification = "CONFIDENTIAL")
AND (Action = "USB Transfer")
AND (User Role ≠ "Authorized Data Handler")
THEN (Block + Notify Manager)
```

**Действия по политиката:**
- **Block** - пълно предотвратяване на действието
- **Quarantine** - изолиране на данните за преглед
- **Encrypt** - автоматично криптиране на предаването
- **Notify** - уведомяване на потребители или администратори
- **Log** - записване на инцидента за одитни цели
- **Redirect** - пренасочване към защитен комуникационен канал

**Нива на прилагане на политиките:**
```
Monitor Mode: Log activities, no blocking
- Purpose: Baseline establishment, policy tuning
- Duration: Initial deployment phase
- Risk: High (no active protection)

Educate Mode: Warn users, allow override
- Purpose: User awareness, gradual enforcement
- Duration: Training period
- Risk: Medium (user discretion)

Enforce Mode: Block violations, limited overrides
- Purpose: Active protection, business justification required
- Duration: Normal operations
- Risk: Low (strict enforcement)

Lockdown Mode: Block all sensitive data movement
- Purpose: Emergency response, incident containment
- Duration: Crisis situations only
- Risk: Very Low (maximum protection)
```

**Съображения за потребителско изживяване:**
- **False positive minimization** - настройване на политики за намаляване на легитимното блокиране
- **Override mechanisms** - процедури за бизнес обосновка
- **User education** - ясни насоки за приемливо поведение
- **Performance impact** - минимизиране на забавянето на системата
- **Transparent operations** - потребителите разбират обосновката на защитата

**Активна фаза (Hot Data):**
- **Frequent access** - ежедневна оперативна употреба
- **High performance** - SSD съхранение, бързи мрежи
- **Immediate availability** - нулево време за възстановяване
- **Current backups** - почасови или ежедневни инкременти

**Референтна фаза (Warm Data):**
- **Occasional access** - седмична или месечна справка
- **Standard performance** - традиционно дисково съхранение
- **Quick availability** - възстановяване за минути до часове
- **Regular backups** - ежедневни или седмични цикли
- **Limited functionality** - само четене или ограничено редактиране

- **Достъп:** Изследователи, правителствени служители, архивен персонал
- **Compliance:** Изисквания за сигурност на българската държава, защита на данни в ЕС

**ПРЕДИЗВИКАТЕЛСТВО ЗА ПРОЕКТИРАНЕ НА АРХИТЕКТУРА:**

**Фаза 1: Анализ на изискванията за сигурност (5 мин)**

**Съпоставяне на класификация и защита на данните:**
```
Classification Level | Storage Requirements | Access Controls | Encryption Requirements
-------------------|---------------------|-----------------|----------------------
State Secret | Air-gapped systems | Multi-person authorization | Hardware encryption + PKI
Confidential | Secure network zone | Role-based + MFA | Full disk + database encryption
Internal | Standard network | Department-based access | Database encryption
Public | Web-accessible | Authentication optional | Transport encryption only
```

**Оценка на заплахите:**
```
External Threats:
- Nation-state cyber attacks
- Historical document theft
- Unauthorized research access
- Data exfiltration attempts

Mitigation Strategy:
□ Network segmentation по classification level
□ Multi-factor authentication за all access
□ Comprehensive audit logging
□ Regular penetration testing
□ Incident response procedures

Internal Threats:
- Insider data theft
- Unauthorized document access
- Accidental disclosure
- Privilege misuse

Mitigation Strategy:
□ Separation на duties implementation
□ Regular access reviews
□ Behavioral monitoring
□ Document watermarking
□ Secure disposal procedures
```

**Фаза 2: Проектиране на техническата архитектура (6 мин)**

**Оформление на инфраструктурата за съхранение:**
```
Air-Gapped Network (State Secret):
- Dedicated servers за classified materials
- Physical access controls
- Hardware security modules
- Manual data transfer procedures
- Isolated backup systems

Secure Network Zone (Confidential):
- DMZ deployment за controlled access
- Database encryption със HSM key management
- VPN access за authorized remote users
- Intrusion detection systems
- Secure backup to separate facility

Standard Network (Internal/Public):
- Standard corporate network integration
- Database encryption със software key management
- Web portal за public document access
- Standard backup procedures
- Load balancing за high availability
```

**Стратегия за криптиране:**
```
State Secret Data:
- Algorithm: AES-256-GCM
- Key Management: Hardware Security Module (HSM)
- Key Rotation: Annual със multi-person ceremony
- Backup: Geographically separated HSMs
- Access: Hardware tokens, biometric authentication

Confidential Data:
- Algorithm: AES-256-TDE (Transparent Database Encryption)
- Key Management: Enterprise key management system
- Key Rotation: Quarterly automated rotation
- Backup: Encrypted backups със separate keys
- Access: Smart cards, multi-factor authentication

Internal/Public Data:
- Algorithm: AES-128-TDE (performance optimized)
- Key Management: Database-integrated key store
- Key Rotation: Annual automated rotation
- Backup: Standard encrypted backups
- Access: Username/password + MFA
```
