### **3. НОВ МАТЕРИАЛ (35 минути)**

#### **Част 1: BIOS vs UEFI (10 мин)**

**BIOS (Basic Input/Output System):**
```
Характеристики на Legacy BIOS:
├── Създаден: 1981 (IBM PC)
├── Размер: 1 MB максимум
├── Режим: 16-bit Real Mode
├── Интерфейс: Текстов (синьо-сив)
├── Boot: MBR (Master Boot Record)
├── Дискове: До 2.2 TB
├── Секции: До 4 primary partitions
└── Сигурност: Ограничена
```

**UEFI (Unified Extensible Firmware Interface):**
```
Характеристики на UEFI:
├── Стандарт: UEFI Forum (Intel инициатива, 2005+)
├── Размер: Няма ограничение
├── Режим: 32/64-bit Protected Mode
├── Интерфейс: Графичен (мишка, цветове)
├── Boot: GPT (GUID Partition Table)
├── Дискове: До 9.4 ZB (zettabytes)
├── Секции: До 128 partitions
├── Мрежа: Мрежов boot (PXE подобрен)
└── Сигурност: Secure Boot, TPM интеграция
```

**Сравнителна таблица:**

| Характеристика | BIOS | UEFI |
|---------------|------|------|
| Boot режим | 16-bit | 32/64-bit |
| Интерфейс | Текстов | Графичен |
| Partition scheme | MBR | GPT |
| Макс. размер на диск | 2.2 TB | 9.4 ZB |
| Boot скорост | По-бавен | По-бърз |
| Secure Boot | Не | Да |
| Мрежов boot | Ограничен | Пълен |
| Драйвери | Вграден код | EFI модули |

#### **Част 2: POST процес (8 мин)**

**Power-On Self-Test последователност:**
```
1. Power Good сигнал (PSU → дънна платка)
   └── Ако няма → системата не стартира

2. CPU Reset вектор → UEFI код
   └── CPU започва изпълнение на firmware

3. CPU и Cache тест
   └── Проверка на регистри и кеш

4. Памет тест (RAM)
   └── Memory training
   └── Detection на модули
   └── Тест за грешки

5. Периферни устройства
   └── GPU инициализация
   └── Storage контролери
   └── USB контролери

6. Boot устройство търсене
   └── Според boot priority
   └── Зареждане на bootloader

7. Предаване на контрола на OS
```

**POST кодове (Debug LED/Speaker):**
```
Типични POST кодове (варират по производител):
00 - Не е открит CPU
55 - Памет не е открита
d6 - Няма видео сигнал
A2 - IDE/SATA откриване
AA - Системата е готова
```

#### **Част 3: UEFI секции и настройки (12 мин)**

**Типична UEFI структура:**
```
UEFI Setup Utility
│
├── Main
│   ├── System Date/Time
│   ├── BIOS Version
│   ├── CPU Information
│   ├── Memory Information
│   └── SATA Information
│
├── Ai Tweaker / OC Tweaker / Extreme Tweaker
│   ├── Ai Overclock Tuner
│   │   ├── Auto
│   │   ├── XMP I / XMP II
│   │   └── Manual
│   ├── BCLK Frequency
│   ├── CPU Core Ratio
│   ├── DRAM Frequency
│   └── DRAM Voltage
│
├── Advanced
│   ├── CPU Configuration
│   │   ├── Intel Virtualization Technology [Enabled/Disabled]
│   │   ├── Hyper-Threading [Enabled/Disabled]
│   │   └── CPU Power Management
│   ├── SATA Configuration
│   │   └── SATA Mode [AHCI/RAID]
│   ├── USB Configuration
│   ├── Onboard Devices Configuration
│   │   ├── HD Audio Controller
│   │   ├── Intel LAN Controller
│   │   └── RGB LED Lighting
│   └── APM Configuration
│       └── ErP Ready [Enabled/Disabled]
│
├── Boot
│   ├── Boot Option Priorities
│   │   ├── Boot Option #1
│   │   ├── Boot Option #2
│   │   └── Boot Option #3
│   ├── CSM (Compatibility Support Module)
│   │   └── Launch CSM [Enabled/Disabled]
│   ├── Secure Boot
│   │   ├── Secure Boot State
│   │   └── Key Management
│   └── Fast Boot [Enabled/Disabled]
│
├── Security / Tool
│   ├── Administrator Password
│   ├── User Password
│   ├── Secure Boot Control
│   ├── TPM Device Selection
│   └── ASUS EZ Flash 3 / Q-Flash
│
└── Exit
    ├── Save Changes & Reset
    ├── Discard Changes & Exit
    ├── Load Optimized Defaults
    └── Launch EFI Shell
```

**Важни настройки за обяснение:**

**1. XMP/EXPO (Extreme Memory Profile):**
```
Какво е XMP:
- Intel технология за RAM overclocking
- Записва профили в SPD на модула
- Позволява автоматично активиране на по-високи честоти

AMD EXPO (Extended Profiles for Overclocking):
- AMD еквивалент за DDR5
- Оптимизирани профили за Ryzen процесори

Пример:
DDR5-4800 (JEDEC стандарт) → DDR5-6000 (XMP профил)
```

**2. Virtualization Technology:**
```
Intel VT-x / AMD-V:
├── Позволява hardware-assisted виртуализация
├── Необходимо за: VMware, VirtualBox, Hyper-V, WSL2
├── Default: Понякога е Disabled!
└── Местоположение: Advanced → CPU Configuration

Intel VT-d / AMD-Vi (IOMMU):
├── I/O Memory Management Unit
├── Позволява passthrough на PCIe устройства към VM
└── Необходимо за GPU passthrough
```

**3. Secure Boot:**
```
Функция:
├── Проверява дигитални подписи на boot файлове
├── Предотвратява зареждане на неподписан код
├── Изисква се за Windows 11

Режими:
├── Standard - само Microsoft keys
├── Custom - позволява добавяне на собствени keys
└── Disabled - без проверка (за Linux/специални случаи)
```

#### **Част 4: Практически сценарии (5 мин)**

**Сценарий 1: Компютърът не вижда новия SSD**
```
Решение:
1. Boot → Storage Configuration
2. Провери SATA Mode: AHCI (не IDE)
3. Advanced → NVMe Configuration
4. Провери дали M.2 слотът не споделя lanes с SATA
```

**Сценарий 2: Искаме да инсталираме Linux от USB**
```
Стъпки:
1. Boot → Boot Option Priorities
2. Постави USB на първо място
3. Boot → CSM → Enable (ако е Legacy installer)
4. Boot → Secure Boot → Disable (временно)
5. Save & Exit
```

**Сценарий 3: RAM работи на по-ниска честота**
```
Решение:
1. Ai Tweaker → Ai Overclock Tuner → XMP I или XMP II
2. Провери DRAM Frequency
3. Save & Exit
4. Провери с CPU-Z / Task Manager
```

### ПРИЛОЖЕНИЕ 3: BIOS RECOVERY ПРОЦЕДУРИ

**Метод 1: Load Defaults**
```
1. Влезте в UEFI
2. Exit → Load Optimized Defaults
3. Save & Exit
```

**Метод 2: CMOS Clear**
```
1. Изключете захранването (и PSU превключвателя)
2. Извадете CMOS батерията
3. Изчакайте 5-10 минути
4. Поставете батерията
5. Включете системата
```

**Метод 3: Clear CMOS Jumper**
```
1. Изключете захранването
2. Намерете CLR_CMOS jumper на платката
3. Преместете jumper от 1-2 на 2-3
4. Изчакайте 10 секунди
5. Върнете jumper на 1-2
6. Включете системата
```

**Метод 4: BIOS Flashback (ако се поддържа)**
```
1. Изтеглете BIOS файл от производителя
2. Преименувайте на специфично име (например ASUS: CAP файл)
3. Копирайте на FAT32 USB
4. Поставете в специфичен USB порт
5. Натиснете BIOS Flashback бутон
6. Изчакайте LED индикатора
```
