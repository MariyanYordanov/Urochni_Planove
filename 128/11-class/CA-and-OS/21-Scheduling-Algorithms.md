### **3. НОВ МАТЕРИАЛ (40 минути)**

#### **Част 1: Цели и критерии (8 мин)**

**Цели на CPU Scheduling:**
```
1. CPU Utilization - максимално използване на CPU
   Идеал: 100%, Реално: 40-90%

2. Throughput - брой завършени процеси за време
   По-висок = по-добър

3. Turnaround Time - време от submission до completion
   По-нисък = по-добър
   Turnaround = Completion - Arrival

4. Waiting Time - време в Ready опашката
   По-нисък = по-добър
   Waiting = Turnaround - Burst Time

5. Response Time - време до първия отговор
   Важно за интерактивни системи
```

**Дилемата:**
```
Не може всичко да е оптимално едновременно!

Пример:
- Кратките задачи първи → По-добро avg waiting time
- Но дългите задачи чакат прекалено → "Starvation"
```

#### **Част 2: FCFS - First-Come, First-Served (8 мин)**

**Принцип:**
```
Процесите се изпълняват в реда на пристигане.
Като опашка в магазин - първи дошъл, първи обслужен.

Non-preemptive: веднъж започнал, процесът работи до край.
```

**Пример:**
```
Процес  Arrival  Burst Time
P1      0        24
P2      0        3
P3      0        3

Gantt диаграма (FCFS):
|------P1------|--P2--|--P3--|
0              24     27     30

Изчисления:
P1: Waiting = 0,  Turnaround = 24
P2: Waiting = 24, Turnaround = 27
P3: Waiting = 27, Turnaround = 30

Average Waiting Time = (0 + 24 + 27) / 3 = 17 ms
```

**Convoy Effect:**
```
Ако дълъг процес е пръв, всички кратки чакат дълго!
Решение: подреди по burst time (SJF)
```

#### **Част 3: SJF - Shortest Job First (8 мин)**

**Принцип:**
```
Изпълнява се процесът с най-кратък burst time.
Оптимален за минимизиране на average waiting time.
```

**Същият пример с SJF:**
```
Процес  Arrival  Burst Time
P1      0        24
P2      0        3
P3      0        3

Gantt диаграма (SJF):
|--P2--|--P3--|------P1------|
0      3      6              30

Изчисления:
P2: Waiting = 0, Turnaround = 3
P3: Waiting = 3, Turnaround = 6
P1: Waiting = 6, Turnaround = 30

Average Waiting Time = (6 + 0 + 3) / 3 = 3 ms

Сравнение: FCFS = 17 ms, SJF = 3 ms !
```

**Проблем:**
```
❌ Starvation - дълги процеси може никога да не се изпълнят
❌ Трябва да знаем burst time предварително (невъзможно!)
    Решение: prediction базирано на история
```

#### **Част 4: Round Robin (8 мин)**

**Принцип:**
```
Всеки процес получава фиксиран time quantum (напр. 10ms).
След изтичане - процесът се връща в края на опашката.
Preemptive - справедлив за всички.
```

**Пример с Time Quantum = 4:**
```
Процес  Burst Time
P1      24
P2      3
P3      3

Gantt диаграма (RR, q=4):
|P1|P2|P3|P1|P1|P1|P1|P1|
0  4  7 10 14 18 22 26 30

P1: 0-4, 10-14, 14-18, 18-22, 22-26, 26-30
P2: 4-7 (завършва, burst=3 < quantum=4)
P3: 7-10 (завършва)

Waiting times:
P1: 30-24 = 6 (turnaround) - 24 (burst) = 6... 
    Реално: общо чакане = (10-4) + 0 = 6
P2: 4-0 = 4, но burst=3, waiting = 4
P3: 7-0 = 7, но burst=3, waiting = 4
```

**Time Quantum избор:**
```
Малък quantum (1-5 ms):
├── По-добър response time
├── Много context switches
└── По-голям overhead

Голям quantum (100+ ms):
├── По-малко overhead
├── Подобен на FCFS
└── По-лош response time

Типичен: 10-100 ms
```

#### **Част 5: Priority Scheduling (8 мин)**

**Принцип:**
```
Всеки процес има приоритет (число).
Изпълнява се процесът с най-висок приоритет.
По-ниско число = по-висок приоритет (обикновено).
```

**Пример:**
```
Процес  Burst  Priority
P1      10     3
P2      1      1 (най-висок)
P3      2      4
P4      1      5
P5      5      2

Gantt (Priority, non-preemptive):
|P2|--P5--|---P1----|P3|P4|
0  1      6        16 18 19

Order: P2(1) → P5(2) → P1(3) → P3(4) → P4(5)
```

**Starvation и решение:**
```
❌ Starvation: Нископриоритетни процеси чакат вечно

✅ Aging: Постепенно увеличаване на приоритета
   с времето на чакане
   
   Пример: На всеки 1 секунда чакане, 
           приоритетът се увеличава с 1
```

### **4. ПРАКТИЧЕСКИ УПРАЖНЕНИЯ (25 минути)**

#### **Задача 1: FCFS изчисления (8 мин)**

```
Дадени процеси:
Процес  Arrival  Burst
P1      0        5
P2      1        3
P3      2        8
P4      3        6

1. Начертайте Gantt диаграма за FCFS
2. Изчислете Waiting Time за всеки процес
3. Изчислете средно Waiting Time
4. Изчислете средно Turnaround Time
```

**Решение:**
```
Gantt: |--P1--|--P2--|----P3----|---P4---|
       0      5      8         16       22

        Arrival Burst Completion Turnaround Waiting
P1      0       5     5          5          0
P2      1       3     8          7          4
P3      2       8     16         14         6
P4      3       6     22         19         13

Avg Waiting = (0+4+6+13)/4 = 5.75 ms
Avg Turnaround = (5+7+14+19)/4 = 11.25 ms
```

#### **Задача 2: SJF изчисления (8 мин)**

```
Същите процеси с SJF (non-preemptive):

Процес  Arrival  Burst
P1      0        5
P2      1        3
P3      2        8
P4      3        6

Внимание: В момент 0 само P1 е пристигнал!
```

**Решение:**
```
t=0: Само P1 → изпълнява се P1
t=5: P1 завършва, налични: P2(3), P3(8), P4(6)
     Най-кратък: P2(3) → изпълнява се P2
t=8: P2 завършва, налични: P3(8), P4(6)
     Най-кратък: P4(6) → изпълнява се P4
t=14: P4 завършва, налични: P3(8)
      → изпълнява се P3
t=22: P3 завършва

Gantt: |--P1--|--P2--|---P4---|----P3----|
       0      5      8        14        22

        Arrival Burst Completion Turnaround Waiting
P1      0       5     5          5          0
P2      1       3     8          7          4
P4      3       6     14         11         5
P3      2       8     22         20         12

Avg Waiting = (0+4+5+12)/4 = 5.25 ms
```

#### **Задача 3: Round Robin симулация (9 мин)**

```
Процеси:
P1: Burst = 10
P2: Burst = 4
P3: Burst = 6

Time Quantum = 3

Симулирайте изпълнението стъпка по стъпка.
```

**Решение:**
```
Ready Queue: [P1, P2, P3]

t=0-3:   P1 изпълнява (остават 7), Queue: [P2, P3, P1]
t=3-6:   P2 изпълнява (остава 1), Queue: [P3, P1, P2]
t=6-9:   P3 изпълнява (остават 3), Queue: [P1, P2, P3]
t=9-12:  P1 изпълнява (остават 4), Queue: [P2, P3, P1]
t=12-13: P2 завършва (burst=1), Queue: [P3, P1]
t=13-16: P3 завършва (burst=3), Queue: [P1]
t=16-19: P1 изпълнява (остава 1), Queue: [P1]
t=19-20: P1 завършва

Gantt:
|P1|P2|P3|P1|P2|P3|P1|P1|
0  3  6  9 12 13 16 19 20
```

### **5. ОБОБЩЕНИЕ (8 минути)**

**Сравнителна таблица:**

| Алгоритъм | Preemptive | Предимства | Недостатъци |
|-----------|------------|------------|-------------|
| FCFS | Не | Прост | Convoy effect |
| SJF | Не/Да | Оптимален avg wait | Starvation, prediction |
| Priority | Не/Да | Гъвкав | Starvation |
| Round Robin | Да | Справедлив, добър response | Повече context switches |

**Ключови точки:**
```
1. Няма "най-добър" алгоритъм за всички случаи
2. Изборът зависи от типа система
3. Реалните ОС използват комбинации (Multilevel Queue)
4. Preemptive алгоритмите са по-справедливи
```
