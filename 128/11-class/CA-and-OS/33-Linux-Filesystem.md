#### **ЗАДАЧА 2: Управление на права (20 минути)**

**Сценарий:** Имате web приложение с различни нива на достъп

```bash
# 1. Създайте потребители и групи (с sudo права):
sudo groupadd developers
sudo groupadd testers
sudo useradd -m -G developers alice
sudo useradd -m -G testers bob

# 2. Създайте проектна структура:
sudo mkdir -p /opt/webapp/{public,private,logs}

# 3. Задайте собственост:
sudo chown -R alice:developers /opt/webapp

# 4. Конфигурирайте права:
# Public - всички могат да четат
sudo chmod 755 /opt/webapp/public

# Private - само developers имат пълен достъп
sudo chmod 770 /opt/webapp/private

# Logs - developers пишат, testers четат
sudo chmod 750 /opt/webapp/logs
sudo chgrp testers /opt/webapp/logs

# 5. Създайте тестови файлове:
sudo -u alice echo "Public content" > /opt/webapp/public/index.html
sudo -u alice echo "Private data" > /opt/webapp/private/secrets.txt
sudo -u alice echo "Log entry" > /opt/webapp/logs/app.log

# 6. Тествайте достъпа:
sudo -u bob cat /opt/webapp/public/index.html  # Трябва да работи
sudo -u bob cat /opt/webapp/private/secrets.txt  # Трябва да се провали
sudo -u bob cat /opt/webapp/logs/app.log  # Трябва да работи
```

**Анализ на правата:**
```bash
# Проверка с различни команди
ls -la /opt/webapp/
stat /opt/webapp/private/
getfacl /opt/webapp/logs/
```

#### **ЗАДАЧА 3: Специални права и символни връзки (15 минути)**

**Работа със SUID, SGID и Sticky bit:**

```bash
# 1. Създайте директория за споделена работа:
mkdir ~/shared
chmod 777 ~/shared

# 2. Добавете Sticky bit (само собственикът може да трие):
chmod +t ~/shared
# или
chmod 1777 ~/shared

# 3. Създайте програма със SUID (пример):
echo '#!/bin/bash
echo "Running as: $(whoami)"
echo "Real user: $SUDO_USER"' > ~/suid_test.sh
chmod 755 ~/suid_test.sh
chmod u+s ~/suid_test.sh  # SUID bit
ls -l ~/suid_test.sh  # Ще видите 's' вместо 'x'

# 4. SGID на директория:
mkdir ~/team_folder
chmod g+s ~/team_folder  # Новите файлове наследяват групата

# 5. Символни връзки:
ln -s /etc/passwd ~/passwd_link
ln -s /var/log ~/logs_link
ls -la ~/ | grep "^l"  # Показва символните връзки

# 6. Твърди връзки:
echo "Original file" > ~/original.txt
ln ~/original.txt ~/hardlink.txt
ls -li ~/original.txt ~/hardlink.txt  # Същият inode
```

**Сценарий 2: Backup скрипт**
```bash
# Създайте backup скрипт
cat << 'EOF' > ~/backup.sh
#!/bin/bash
BACKUP_DIR="/tmp/backups"
SOURCE_DIR="$HOME/project"
DATE=$(date +%Y%m%d_%H%M%S)

# Създаване на backup директория
mkdir -p "$BACKUP_DIR"

# Създаване на архив
tar -czf "$BACKUP_DIR/backup_$DATE.tar.gz" "$SOURCE_DIR" 2>/dev/null

# Запазване само последните 3 backup-а
ls -t "$BACKUP_DIR"/backup_*.tar.gz | tail -n +4 | xargs -r rm

echo "Backup completed: backup_$DATE.tar.gz"
EOF

chmod 750 ~/backup.sh  # Само собственикът и групата
./backup.sh
```

**Сценарий 3: Мониторинг на промени**
```bash
# Скрипт за проверка на промени
cat << 'EOF' > ~/monitor.sh
#!/bin/bash
WATCH_DIR="$HOME/project"
HASH_FILE="/tmp/file_hashes.txt"

# Генериране на хешове
find "$WATCH_DIR" -type f -exec md5sum {} \; > "$HASH_FILE.new"

# Сравнение с предишни
if [ -f "$HASH_FILE" ]; then
    diff "$HASH_FILE" "$HASH_FILE.new" | grep "^>" | cut -d' ' -f3-
    if [ $? -eq 0 ]; then
        echo "Changes detected!"
    fi
fi

mv "$HASH_FILE.new" "$HASH_FILE"
EOF

chmod 700 ~/monitor.sh  # Само собственикът
```

## 8. ПРИЛОЖЕНИЯ

### ПРИЛОЖЕНИЕ 1: СПРАВОЧНИК ПРАВА

**Символно представяне:**
```
r (read)    = 4
w (write)   = 2  
x (execute) = 1

Примери:
rwx = 7 (4+2+1)
rw- = 6 (4+2)
r-x = 5 (4+1)
r-- = 4
-wx = 3 (2+1)
-w- = 2
--x = 1
--- = 0
```

**Специални битове:**
```
SUID = 4000  (u+s)
SGID = 2000  (g+s)
Sticky = 1000 (+t)

Пример: chmod 4755 = -rwsr-xr-x
```

### ПРИЛОЖЕНИЕ 2: КОМАНДИ CHEAT SHEET

```bash
# Файлови операции
ls -la              # Подробен списък
pwd                 # Текуща директория
cd /path            # Смяна на директория
mkdir dir           # Създаване на директория
rmdir dir           # Изтриване на празна директория
rm -rf dir          # Изтриване на директория с файлове
cp -r src dst       # Копиране
mv src dst          # Преместване/преименуване
ln -s target link   # Символна връзка
find / -name file   # Търсене на файл

# Права и собственост
chmod 755 file      # Промяна на права (октално)
chmod u+x file      # Добавяне на execute за user
chmod g-w file      # Премахване на write за group
chmod o=r file      # Задаване само read за others
chown user file     # Промяна на собственик
chgrp group file    # Промяна на група
chown user:group file # Двете едновременно
umask 022           # Маска по подразбиране

# Потребители и групи
useradd user        # Създаване на потребител
userdel user        # Изтриване на потребител
usermod -aG group user # Добавяне към група
groupadd group      # Създаване на група
groups user         # Показва групите на потребител
id user             # Подробна информация

# Информация
stat file           # Детайлна информация
file file           # Тип на файла
du -sh dir          # Размер на директория
df -h               # Свободно място
```

### ПРИЛОЖЕНИЕ 3: СТРУКТУРА НА LINUX ФАЙЛОВА СИСТЕМА

```
├── bin/        # Основни команди

├── dev/        # Устройства

├── home/       # Домашни директории
├── lib/        # Системни библиотеки
├── media/      # Монтирани медии
├── mnt/        # Временни mount точки
├── opt/        # Допълнителен софтуер
├── proc/       # Процесна информация
├── root/       # Домашна на root
├── sbin/       # Системни команди
├── srv/        # Данни за услуги
├── sys/        # Системна информация
├── tmp/        # Временни файлове
├── usr/        # Потребителски програми
└── var/        # Променливи данни
```
