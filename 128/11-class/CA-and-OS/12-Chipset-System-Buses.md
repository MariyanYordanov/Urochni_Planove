**Класическа архитектура (1990-2008):**
```
┌──────────────────────────────────────────────────────────┐
│                         CPU                               │
│                    (Pentium 4, Core 2)                    │
└────────────────────────┬─────────────────────────────────┘

                         │ 6.4-12.8 GB/s
                    ┌────┴────┐
                    │ NORTH   │ Memory Controller Hub (MCH)
                    │ BRIDGE  │
                    └────┬────┘
        ┌────────────────┼────────────────┐
        │                │                │
   ┌────┴────┐     ┌────┴────┐     ┌─────┴─────┐
   │   RAM   │     │   AGP/  │     │    DMI    │
   │  DDR2   │     │  PCIe   │     │  Link     │
   └─────────┘     │  x16    │     └─────┬─────┘
                   └─────────┘           │
                                   ┌─────┴─────┐
                                   │  SOUTH    │ I/O Controller Hub
                                   │  BRIDGE   │

                    ┌────────────────────┼────────────────────┐

               ┌────┴───┐ ┌───┴───┐ ┌────┴────┐ ┌───┴───┐ ┌───┴───┐
               │  SATA  │ │  USB  │ │  Audio  │ │  LAN  │ │ PCIe  │
               │        │ │       │ │         │ │       │ │  x1   │
               └────────┘ └───────┘ └─────────┘ └───────┘ └───────┘
```

**Проблеми с класическата архитектура:**
- FSB споделен между всички компоненти
- North Bridge като bottleneck
- Висока латентност CPU ↔ RAM
- Допълнително захранване и охлаждане

**Съвременна архитектура (2008+):**
```
┌───────────────────────────────────────────────────────────────────┐
│                              CPU                                   │
│  ┌─────────────┬────────────────┬───────────────┐                 │
│  │ CPU Cores   │ Memory         │ PCIe          │                 │
│  │ L1/L2/L3    │ Controller     │ Controller    │                 │
│  │ Cache       │ (Integrated)   │ (Integrated)  │                 │
│  └─────────────┴───────┬────────┴───────┬───────┘                 │
└────────────────────────│────────────────│─────────────────────────┘
                         │                │
                         ▼                ▼
                   ┌─────────┐      ┌───────────┐
                   │  DDR5   │      │ PCIe 5.0  │
                   │  RAM    │      │  x16 GPU  │
                   └─────────┘      └───────────┘
                         │
                         │ DMI 4.0 (Intel) / 
                         │ Infinity Fabric (AMD)
                         │ ~8 GB/s
                         ▼
              ┌────────────────────┐
              │        PCH         │ Platform Controller Hub
              │   (Z790 / X670)    │
              └──────────┬─────────┘

     │         │         │         │         │         │
┌────┴───┐ ┌───┴───┐ ┌───┴───┐ ┌───┴───┐ ┌───┴───┐ ┌───┴───┐
```

#### **Част 3: Системни шини и интерфейси (12 мин)**

**DMI (Direct Media Interface) - Intel:**
```
Версия    Bandwidth   Lanes   Използване
DMI 2.0   2 GB/s      x4      Sandy Bridge era
DMI 3.0   4 GB/s      x4      Skylake - Rocket Lake
DMI 4.0   8 GB/s      x8      Alder Lake+
```

**Infinity Fabric - AMD:**
```
Особености:
├── Scalable interconnect
├── Използва се и за CPU-to-CPU комуникация
├── Настройваеми честоти
├── Синхронизира се с RAM честотата
└── По-висока latency от Intel, но по-гъвкав
```

**PCIe поколения сравнение:**
```
Поколение  Per-Lane    x1        x4        x16
PCIe 3.0   1 GB/s      1 GB/s    4 GB/s    16 GB/s
PCIe 4.0   2 GB/s      2 GB/s    8 GB/s    32 GB/s
PCIe 5.0   4 GB/s      4 GB/s    16 GB/s   64 GB/s
PCIe 6.0   8 GB/s      8 GB/s    32 GB/s   128 GB/s*
*Очаквано 2024-2025
```

**Как се изчислява bandwidth:**
```
Формула: Bandwidth = Lanes × Per-Lane Speed × Encoding Efficiency

Пример PCIe 4.0 x16:
= 16 lanes × 16 GT/s × 128b/130b encoding
= 16 × 2 GB/s × 0.985
≈ 31.5 GB/s (практически)
```

#### **Част 4: Lane разпределение и bottlenecks (8 мин)**

**Типично разпределение в Z790 система:**
```
CPU Direct Lanes (20 PCIe 5.0):
├── GPU: x16 PCIe 5.0 (или x8+x8 за SLI/CrossFire)
└── M.2 NVMe: x4 PCIe 5.0

PCH Lanes (28 PCIe 4.0/3.0):
├── M.2 #2: x4 PCIe 4.0
├── M.2 #3: x4 PCIe 4.0
├── SATA: 6 порта (споделени)
├── USB 3.2: 10 порта
├── LAN: x1 PCIe 3.0
├── WiFi: x1 PCIe 3.0
└── Допълнителни слотове: x4 + x1
```

**Идентифициране на bottleneck:**
```
Сценарий: Копиране от SATA SSD на NVMe SSD
├── SATA SSD: 560 MB/s (максимум)
├── NVMe SSD: 7000 MB/s (възможно)
├── DMI 4.0: 8000 MB/s (достатъчно)
└── Bottleneck: SATA интерфейсът

Сценарий: Два NVMe SSD през PCH, едновременен достъп
├── NVMe #1: 7000 MB/s
├── NVMe #2: 7000 MB/s
├── Общо необходимо: 14000 MB/s
├── DMI 4.0: 8000 MB/s
└── Bottleneck: DMI връзката!
```
