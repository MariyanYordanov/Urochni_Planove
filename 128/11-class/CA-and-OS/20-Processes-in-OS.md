### **3. НОВ МАТЕРИАЛ (40 минути)**

#### **Част 1: Какво е процес? (10 мин)**

**Определение:**
```
Процес = Програма в изпълнение

Програма: Пасивен код на диска (файл .exe, .py, .sh)
Процес:   Активна инстанция с:
          ├── Код (text section)
          ├── Данни (data section)
          ├── Стек (stack)
          ├── Heap (динамична памет)
          └── Контекст (регистри, PC)
```

**Аналогия:**
```
Рецепта (Програма) vs Готвене (Процес)

Рецепта:
- Лежи в книга (на диска)
- Не прави нищо сама
- Може да се използва многократно

Готвене:
- Активна дейност (в RAM)
- Използва ресурси (съставки = памет)
- Има състояние (етап на приготвяне)
- Може да се паузира и продължи
```

**Process Control Block (PCB):**
```
┌─────────────────────────────────┐
│      PROCESS CONTROL BLOCK      │
├─────────────────────────────────┤
│ Process ID (PID): 1234          │
│ Parent PID (PPID): 1            │
│ Process State: Running          │
│ Program Counter: 0x7fff5678     │
│ CPU Registers: [saved values]   │
│ Memory Limits: 0x1000-0x9000    │
│ Open Files: [fd0, fd1, fd2...]  │
│ Priority: 20                    │
│ CPU Time Used: 00:00:05.234     │
│ Owner: user1                    │
└─────────────────────────────────┘
```

#### **Част 2: Състояния на процес (12 мин)**

**Диаграма на състоянията:**
```
                    ┌─────────┐
                    │   NEW   │
                    │(Създаден│
                    └────┬────┘
                         │ admitted
                         ▼
                    ┌─────────┐
         ┌─────────│  READY  │◄────────────┐
         │         │(Готов)  │             │
         │         └────┬────┘             │
         │              │ scheduler        │ I/O or event
         │              │ dispatch         │ completion
         │              ▼                  │
         │         ┌─────────┐             │
         │         │ RUNNING │─────────────┤
         │         │(Изпълн.)│             │
         │         └────┬────┘             │
         │              │                  │
         │    ┌─────────┼─────────┐       │
         │    │         │         │       │
         │interrupt  exit    I/O or      │
         │    │         │    event wait  │
         │    │         ▼         │       │
         │    │    ┌─────────┐    │       │
         │    │    │TERMINATED   │       │
         │    │    │(Завършен)   │       │
         │    │    └─────────┘    │       │
         │    │                   ▼       │
         │    │              ┌─────────┐  │
         └────┘              │ WAITING │──┘
                             │(Чакащ)  │
                             └─────────┘
```

**Описание на състоянията:**

| Състояние | Описание | Пример |
|-----------|----------|--------|
| NEW | Процесът се създава | Стартиране на програма |
| READY | Готов за изпълнение, чака CPU | В опашката за scheduler |
| RUNNING | Изпълнява се на CPU | Активно обработва данни |
| WAITING | Чака I/O или събитие | Чете от диск, чака мрежа |
| TERMINATED | Завършил изпълнение | Програмата е приключила |

**Преходи:**
```
NEW → READY:      Процесът е приет от ОС
READY → RUNNING:  Scheduler избира процеса
RUNNING → READY:  Прекъсване (interrupt), time slice изтекъл
RUNNING → WAITING: Заявка за I/O операция
WAITING → READY:  I/O операцията завършва
RUNNING → TERMINATED: Процесът завършва (exit)
```

#### **Част 3: Процес vs Нишка (8 мин)**

**Сравнение:**
```
ПРОЦЕС (Process)              НИШКА (Thread)
├── Отделно адресно           ├── Споделено адресно
│   пространство              │   пространство
├── Собствени ресурси         ├── Споделени ресурси
├── По-тежък за създаване     ├── По-лек за създаване
├── Комуникация чрез IPC      ├── Директен достъп до паметта
└── По-изолиран               └── По-бърза комуникация

         ПРОЦЕС
    ┌──────────────────┐
    │    ┌──────────┐  │
    │    │ Thread 1 │  │
    │    └──────────┘  │
    │    ┌──────────┐  │
    │    │ Thread 2 │  │
    │    └──────────┘  │
    │    ┌──────────┐  │
    │    │ Thread 3 │  │
    │    └──────────┘  │
    │                  │
    │  Споделена памет │
    └──────────────────┘
```

**Пример - браузър:**
```
Chrome.exe (Процес)
├── Main Thread (UI)
├── Renderer Thread (Tab 1)
├── Renderer Thread (Tab 2)
├── Network Thread
├── GPU Thread
└── Plugin Threads...

Всеки tab може да е отделен процес (Chrome process isolation)!
```

#### **Част 4: Context Switching (10 мин)**

**Какво е Context Switch:**
```
Процесът на запазване на състоянието на един процес
и зареждане на състоянието на друг.

Стъпки:
1. Запази регистрите на текущия процес в PCB
2. Актуализирай PCB (състояние, CPU time)
3. Премести текущия процес в съответната опашка
4. Избери следващ процес (scheduler)
5. Актуализирай PCB на новия процес
6. Зареди контекста на новия процес
7. Продължи изпълнението

Време за context switch: ~1-10 микросекунди
```

**Визуализация:**
```
Време →
────────────────────────────────────────────────────►

P1: ████████░░░░░░░░████████░░░░░░░░████████
P2: ░░░░░░░░████████░░░░░░░░████████░░░░░░░░

    │      │        │      │        │
    └──CS──┘        └──CS──┘        └──CS──┘
    
CS = Context Switch (overhead)
```

## 8. ПРИЛОЖЕНИЯ

### ПРИЛОЖЕНИЕ 1: LINUX КОМАНДИ СПРАВОЧНИК

```bash
# Процеси
ps aux              # Всички процеси
ps -ef              # Друг формат
ps -u username      # Процеси на потребител
pstree              # Дървовидно
pgrep name          # Търсене по име

# Мониторинг
top                 # Интерактивен монитор
htop                # По-добър монитор (ако е инсталиран)
atop                # Разширен мониторинг

# Управление
kill PID            # Изпрати сигнал (default: TERM)
kill -9 PID         # Force kill (SIGKILL)
killall name        # Kill по име
nice -n 10 command  # Стартирай с приоритет
renice 10 -p PID    # Промени приоритет

# Информация
/proc/PID/          # Директория с информация
cat /proc/PID/status   # Статус
cat /proc/PID/cmdline  # Команда
```

### ПРИЛОЖЕНИЕ 2: WINDOWS КОМАНДИ

```powershell
# PowerShell команди
Get-Process                    # Списък процеси
Get-Process -Name chrome       # По име
Get-Process | Sort CPU -Desc   # Сортирани по CPU
Stop-Process -Id PID           # Спри процес
Start-Process notepad          # Стартирай

# CMD команди
tasklist                       # Списък
tasklist /FI "PID eq 1234"     # Филтър
taskkill /PID 1234             # Спри
taskkill /IM notepad.exe       # По име
```
