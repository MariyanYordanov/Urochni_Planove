## 5. ХОД НА УРОКА

### **НОВ МАТЕРИАЛ**

#### **Йерархична структура:**
```
ДЪРВОВИДНА ОРГАНИЗАЦИЯ:

       /  (root)
       │
   ┌───┴───┬───────┬───────┐
  home   etc    var      usr
   │      │      │        │
  user1  passwd  log     bin
   │             │
 Documents    syslog

ПРЕДИМСТВА:
✓ Логическа организация
✓ Лесна навигация
✓ Групиране на свързани файлове
✓ Контрол на достъпа по директории
```

#### **Пътища:**
```
АБСОЛЮТЕН ПЪТ - започва от root:
Linux:   /home/user/Documents/file.txt
Windows: C:\Users\User\Documents\file.txt

ОТНОСИТЕЛЕН ПЪТ - спрямо текуща директория:
./file.txt       (текуща директория)
../folder/file   (родителска → folder)
folder/file      (поддиректория)

СПЕЦИАЛНИ:
.  = текуща директория
.. = родителска директория
~  = home (Linux)
```

#### **Inodes (Linux):**
```
INODE съдържа:
├── Тип на файла
├── Права за достъп
├── Собственик и група
├── Размер
├── Timestamps (atime, mtime, ctime)
├── Брой твърди връзки
└── Указатели към data blocks

Файловото име НЕ е в inode!
Директорията = таблица: име → inode номер

ls -i file.txt    # Показва inode номер
df -i             # Използвани inodes
```
