           │                 │
           ├─────────────────┤
           │    L2 Cache     │  256 KB-1 MB, ~4 ns
           │                 │
           ├─────────────────┤
           │    L3 Cache     │  8-64 MB, ~10 ns

#### **Част 2: Locality of Reference (8 мин)**

```
TEMPORAL LOCALITY (времева):
Ако достъпих данни сега, вероятно ще ги достъпя пак скоро.
Пример: Променлива в цикъл

for (i = 0; i < 1000; i++) {
    sum += array[i];  // 'sum' се използва 1000 пъти
}

SPATIAL LOCALITY (пространствена):
Ако достъпих адрес X, вероятно ще достъпя X+1, X+2...
Пример: Обхождане на масив

Затова кешът зарежда цели "cache lines" (64 bytes)
Ако четеш array[0], кешът зарежда array[0..15]
```

#### **Част 3: CPU Cache работа (12 мин)**

```
CPU иска данни от адрес 0x1234:

1. Провери L1 Cache
   ├── HIT → Върни данните (~1 ns)
   └── MISS → Провери L2

2. Провери L2 Cache  
   ├── HIT → Копирай в L1, върни данните (~4 ns)
   └── MISS → Провери L3

3. Провери L3 Cache
   ├── HIT → Копирай в L2→L1, върни (~10 ns)
   └── MISS → Чети от RAM

4. Чети от RAM (~100 ns)
   └── Копирай в L3→L2→L1, върни данните
```

**Cache Hit Rate:**
```
Hit Rate = Cache Hits / (Cache Hits + Cache Misses) × 100%

Пример:
1000 достъпа до памет
950 cache hits
50 cache misses

Hit Rate = 950/1000 = 95%

Average Access Time = Hit Rate × Cache Time + Miss Rate × RAM Time
= 0.95 × 1ns + 0.05 × 100ns
= 0.95 + 5 = 5.95 ns

Сравни с: 100 ns (без кеш) → 17× по-бързо!
```

#### **Част 4: Storage - HDD vs SSD vs NVMe (12 мин)**

**HDD (Hard Disk Drive):**
```
Механичен диск:
├── Въртящи се плочи (5400-7200 RPM)
├── Движеща се глава за четене/запис
├── Seek time: 5-10 ms (много бавно!)
├── Sequential read: ~150-200 MB/s
├── Random read: ~0.5-1 MB/s (4K IOPS: ~100)
├── Капацитет: до 20+ TB
└── Цена: ~0.02 лв/GB

Предимства: Евтин, голям капацитет
Недостатъци: Бавен, механични части, чувствителен към удари
```

**SATA SSD:**
```
Flash памет, SATA интерфейс:
├── Без движещи се части
├── Latency: ~50-100 μs
├── Sequential read: ~550 MB/s (SATA limit)
├── Random read: ~50-100 MB/s (4K IOPS: ~50,000)
├── Капацитет: до 8 TB
└── Цена: ~0.07-0.10 лв/GB

Предимства: 50-100× по-бърз от HDD за random I/O
Недостатъци: Ограничен от SATA интерфейса
```

**NVMe SSD:**
```
Flash памет, PCIe интерфейс:
├── Директна връзка към CPU/PCH
├── Latency: ~10-20 μs
├── Sequential read: 3,500-7,500 MB/s (PCIe 4.0)
├── Random read: ~300-500 MB/s (4K IOPS: ~500,000-1,000,000)
├── Капацитет: до 8 TB
└── Цена: ~0.10-0.15 лв/GB

Предимства: Най-бърз consumer storage
```

**IOPS (Input/Output Operations Per Second):**
```
Колко операции може да направи устройството за секунда.
Критично важен за:
- Бази данни
- Виртуални машини
- Операционна система

HDD:     ~100-200 IOPS
SATA SSD: ~50,000-100,000 IOPS
NVMe:    ~500,000-1,000,000 IOPS
```

## 7. ПРИЛОЖЕНИЯ

### ФОРМУЛИ

```
Hit Rate = Hits / (Hits + Misses) × 100%
Miss Rate = 1 - Hit Rate

Average Access Time = Hit_Rate × Cache_Time + Miss_Rate × Memory_Time

Speedup = Time_without_cache / Time_with_cache
```
