### **ЧАС 1: Smart Card архитектура и технологии (40 мин)**

#### **Теоретична част (25 мин):**

**💳 Smart Card основи (10 мин)**
- **Дефиниция:** пластикова карта с embedded микропроцесор или memory chip
- **Историческа справка:**
  - 1968: Helmut Gröttrup патентова идеята
  - 1974: Roland Moreno създава първата practical smart card
  - 1984: Първите телефонни карти във Франция
  - 1990s: Банковите EMV стандарти

**Типове Smart Cards:**
- **Memory Cards:** само storage, без processing power
- **Microprocessor Cards:** CPU + RAM + ROM + EEPROM
- **Dual Interface Cards:** contact + contactless functionality
- **Hybrid Cards:** multiple чипове за различни приложения

**🔌 Contact vs. Contactless технологии (8 мин)**

**Contact Smart Cards:**
- **Interface:** физически контакт със card reader
- **Стандарт:** ISO 7816 (8 pins connector)
- **Power:** получава power от reader
- **Data rate:** до 9.6 Kbps (ISO 7816-3)
- **Security:** физически достъп до комуникацията е труден
- **Използване:** банкови карти, SIM карти, digital certificates

**Contactless Smart Cards:**
- **Interface:** Radio Frequency (RF) комуникация
- **Честоти:** 13.56 MHz (ISO 14443), 125 kHz (proximity)
- **Range:** 0-10 cm за secure applications
- **Data rate:** до 848 Kbps (ISO 14443)
- **Power:** RF energy harvesting от reader
- **Използване:** access cards, transportation, payments

**⚡ Power Management и Communication (7 мин)**
- **Power Delivery:** inductively coupled RF energy
- **Antenna Design:** coil around card perimeter  
- **Modulation:** amplitude shift keying (ASK)
- **Anti-collision:** multiple карти в RF field simultaneously
- **Sleep Modes:** power conservation when idle

*Практически пример:*
Работничка карта в офис сграда:
- Morning: badge in at reception (записва време на влизане)
- Lunch: payment в cafeteria (дебитира lunch account)
- Afternoon: access to secure lab (проверява permissions)
- Evening: badge out (записва overtime hours)

### **ЧАС 2: RFID системи и протоколи (40 мин)**

#### **Теоретична част (22 мин):**

**📡 RFID Technology Overview (8 мин)**
- **RFID = Radio Frequency Identification**
- **Components:** RFID tag + RFID reader + backend system
- **Frequency bands:**
  - **LF (Low Frequency):** 125-134 kHz, short range (~10cm)
  - **HF (High Frequency):** 13.56 MHz, medium range (~1m)  
  - **UHF (Ultra High Frequency):** 860-960 MHz, long range (~12m)
  - **Microwave:** 2.4 GHz+, specialized applications

**Tag типове:**
- **Passive tags:** no internal battery, powered by reader RF energy
- **Active tags:** internal battery, longer range, more expensive
- **Semi-passive tags:** battery for chip power, RF for communication

**🏷️ RFID vs. NFC разлика (6 мин)**

**RFID (Radio Frequency Identification):**
- **Предназначение:** identification и tracking на objects
- **Range:** varies by frequency (cm до meters)
- **Standards:** ISO 18000 series, EPC Global
- **Applications:** supply chain, inventory, animal tracking
- **Security:** generally limited, focus on identification
- **Cost:** €0.05-5 per tag depending on type

**NFC (Near Field Communication):**
- **Предназначение:** secure communication between devices
- **Range:** <4cm (deliberately limited)
- **Standards:** ISO 14443, NFC Forum specifications
- **Applications:** payments, access control, device pairing
- **Security:** encryption, mutual authentication
- **Cost:** €1-10 per device integration

**🔐 RFID Security Challenges (8 мин)**

**Основни vulnerabilities:**
- **Eavesdropping:** RF signals може да се intercept
- **Replay attacks:** записване и повторно предаване на signals
- **Cloning:** copying RFID data към нова карта
- **Jamming:** interference със RF communication
- **Physical tampering:** removing/modifying tags

**Attack scenarios:**
- **Скрито четене:** powerful reader sniffs карти в backpacks/wallets
- **Man-in-the-middle:** intercepting communication between card и legitimate reader
- **Social engineering:** using cloned карта за unauthorized access
- **Inventory tracking:** следене на individuals through personal RFID items

**Defense mechanisms:**
- **Encryption:** protecting data transmission
- **Authentication:** verifying карта legitimacy
- **Rolling codes:** changing codes per use
- **Faraday cage wallets:** RF shielding за personal items
- **Kill commands:** permanently disabling tags when no longer needed

### **ЧАС 3: Криптографски защита и security protocols (40 мин)**

#### **Теоретична част (24 мин):**

**🔐 Cryptographic Protection Methods (10 мин)**

**Symmetric Key Systems:**
- **DES/3DES:** older encryption standard (56/112-bit keys)
- **AES:** Advanced Encryption Standard (128/192/256-bit keys)  
- **Key management:** shared secret между card и system
- **Performance:** fast encryption/decryption на limited hardware
- **Challenge:** secure key distribution и storage

**Asymmetric (Public Key) Systems:**
- **RSA:** widely used, но slow за real-time applications
- **ECC (Elliptic Curve):** smaller keys, better performance
- **Digital signatures:** proving карта authenticity
- **Certificate chains:** PKI infrastructure за trust

**🔑 Authentication Protocols (8 мин)**

**Challenge-Response Authentication:**
1. Reader sends random challenge към карта
2. Card encrypts challenge със secret key
3. Card returns encrypted response
4. Reader verifies response matches expected value
5. Access granted if verification succeeds

**Mutual Authentication:**
- Both карта и reader prove identity to each other
- Prevents cloned readers от stealing credentials
- Essential за high-security applications

**Rolling Code Systems:**
- Authentication codes change with each use
- Previous codes become invalid
- Prevents replay attacks
- Common в automotive key fobs

**🛡️ Advanced Security Features (6 мин)**

**Secure Elements:**
- Tamper-resistant hardware protection
- Hardware-based crypto operations
- Secure key storage
- Common в payment cards, mobile wallets

**Biometric Integration:**
- Fingerprint verification на smart card
- Match-on-card vs. match-on-host architectures  
- Privacy protection (biometric data never leaves card)
- Higher security с convenience

**Anti-counterfeiting:**
- Unique chip identifiers (UIDs)
- Hardware security modules
- Certificate-based authentication
- Impossible-to-clone physical characteristics

*Case Study: EMV Payment Card Security*
- Dynamic cryptograms за each transaction
- PIN verification на card (offline)
- Risk analysis based на transaction patterns  
- Fraud reduction: 70%+ decrease в counterfeit card fraud

- **Anti-theft:** item-level tagging
- **Customer analytics:** shopping behavior tracking
- **Loyalty programs:** integrated rewards systems

**⚠️ Common Vulnerabilities и Attacks (10 мин)**

**Technical Attacks:**
- **RFID skimming:** unauthorized reading на cards in proximity
- **Relay attacks:** extending communication range illegally
- **Jamming attacks:** disabling RFID systems със RF interference
- **Side-channel attacks:** analyzing power consumption за crypto keys

**Physical Attacks:**
- **Card cloning:** copying magnetic stripe or chip data
- **Social engineering:** convincing employees да share access cards
- **Tailgating:** following authorized person through secured door
- **Lost/stolen cards:** immediate revocation challenges

**System-Level Vulnerabilities:**
- **Database security:** centralized credential storage risks
- **Network security:** communication between readers и servers
- **Software updates:** firmware vulnerabilities в field devices
- **Backup systems:** emergency access bypass mechanisms

*Recent attack examples:*
- **Hotel key card hacking:** Onity locks vulnerable да portable device
- **University access cards:** cloning attacks using €50 equipment
- **Payment card skimming:** fake readers на ATMs и POS terminals

*Document всички smart cards и RFID devices you regularly use:*

**Payment Cards:**
- Credit/debit cards: technology type, security features
- Mobile payment systems: Apple Pay, Google Pay, Samsung Pay
- Gift cards, loyalty cards: what data they store
- Transportation cards: metro, bus, train passes

**Access Cards:**
- Student ID: what it provides access to
- Workplace badges: access levels and restrictions  
- Gym/club memberships: personal data storage
- Hotel key cards: temporary access mechanisms

**Analysis format за каждa карта:**
```
Card: [Description]
Technology: [Contact/Contactless/Magnetic stripe]
Data Storage: [What information is stored]  
Authentication: [PIN/Biometric/None required]
Security Features: [Encryption/Holograms/etc]
Risk Level: [Low/Medium/High] - justify rating
```

**Part B: Vulnerability Assessment (15т)**
*Identify potential security risks:*

**Technical Vulnerabilities:**
- Which cards могат да be read without your knowledge?
- What happens if карта is lost or stolen?
- Are any cards susceptible към cloning attacks?
- How quickly можете да disable compromised cards?

**Behavioral Risks:**
- Do you use same PIN за multiple cards?
- How do you store cards? (wallet type, location)
- Do you check statements regularly за unauthorized use?
- What information do you share about card usage?

**Environmental Factors:**
- Where do you use cards most frequently?
- Are those locations generally secure?
- Do you use cards в high-risk environments?

**Part C: Security Improvement Plan (10т)**
*Develop personal security enhancement strategy:*

**Immediate Actions (this week):**
- Settings changes на existing cards/apps
- Behavior modifications for card usage
- Purchase на security accessories if needed

**Short-term Improvements (1-3 months):**
- Card replacements or upgrades
- New security practices adoption
- Enhanced monitoring procedures

**Long-term Strategy (6+ months):**
- Technology upgrade planning
- Comprehensive security review schedule

**Format:** 3-4 page report със photos (privacy-safe) + personal action plan
**Assessment criteria:** Practical application на course concepts, thorough analysis, realistic recommendations
