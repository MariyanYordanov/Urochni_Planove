### **ЧАС 1: OSI Model и TCP/IP Security Fundamentals (40 мин)**

#### **Теоретична част (25 мин):**

**🌐 OSI Model Security Perspective (12 мин)**

**Layer-by-Layer Security Analysis:**

**Layer 1 - Physical Layer:**
- **Security concerns:** cable tapping, fiber optic eavesdropping
- **Protective measures:** secured cable runs, conduit protection
- **Attack examples:** wire shark tapping, electromagnetic interference
- **Detection methods:** physical access controls, cable integrity monitoring

**Layer 2 - Data Link Layer:**
- **Security protocols:** 802.1X port-based authentication
- **Vulnerabilities:** ARP poisoning, MAC flooding, VLAN hopping
- **Protection:** MAC address filtering, port security, private VLANs
- **Switching security:** secure switch configuration, trunk protection

**Layer 3 - Network Layer:**
- **IP security:** IPSec, packet filtering, routing security
- **Vulnerabilities:** IP spoofing, routing attacks, ICMP attacks
- **Protection:** anti-spoofing filters, routing protocol authentication
- **Network segmentation:** subnetting за security isolation

**Layer 4 - Transport Layer:**
- **TCP/UDP security:** port-based filtering, connection state monitoring
- **Vulnerabilities:** TCP hijacking, UDP flooding, port scanning
- **Protection:** stateful firewalls, rate limiting, connection monitoring
- **Security protocols:** SSL/TLS termination points

**Layers 5-7 - Session, Presentation, Application:**
- **Application security:** deep packet inspection, application firewalls
- **Data encryption:** end-to-end encryption, protocol security
- **User authentication:** application-level access controls
- **Content filtering:** malware detection, data loss prevention

**📦 TCP/IP Protocol Suite Security (13 мин)**

**Internet Protocol (IP) Security Issues:**
- **IPv4 limitations:** address exhaustion leading to NAT complications
- **IPv6 security:** larger address space, IPSec mandatory, new attack vectors
- **IP fragmentation:** attacks using fragmented packets
- **Source routing:** potential for bypassing security controls

**Transmission Control Protocol (TCP):**
- **Connection establishment:** three-way handshake vulnerabilities
- **Sequence number attacks:** TCP sequence prediction attacks
- **Connection hijacking:** taking over established TCP connections
- **SYN flood attacks:** overwhelming target with connection requests

**User Datagram Protocol (UDP):**
- **Connectionless nature:** no built-in reliability or security
- **UDP flooding:** overwhelming target with UDP traffic
- **DNS security:** DNS cache poisoning, DNS tunneling
- **DHCP security:** rogue DHCP servers, DHCP spoofing

**Internet Control Message Protocol (ICMP):**
- **Network diagnostics:** ping, traceroute functionality
- **Security implications:** network reconnaissance, covert channels
- **ICMP attacks:** ping of death, ICMP tunneling
- **Protection strategies:** ICMP filtering, rate limiting

### **ЧАС 2: Network Topology и Security Architecture (40 мин)**

#### **Теоретична част (22 мин):**

**🏗️ Network Topology Security Impact (12 мин)**

**Traditional Network Topologies:**

**Bus Topology:**
- **Security characteristics:** single collision domain, shared medium
- **Vulnerabilities:** easy eavesdropping, single point of failure
- **Legacy considerations:** older Ethernet networks, coaxial cable systems
- **Security controls:** limited options, physical security critical

**Star Topology:**
- **Security advantages:** centralized control, isolated connections
- **Hub vs Switch:** collision domains, security implications
- **Single point of failure:** central hub/switch vulnerability
- **Security controls:** port-based security, traffic monitoring

**Ring Topology:**
- **Token-based access:** deterministic access control
- **Fault tolerance:** dual-ring configurations
- **Security considerations:** token manipulation, traffic flow analysis
- **Modern applications:** FDDI, SONET/SDH networks

**Mesh Topology:**
- **Full mesh:** complete connectivity, maximum redundancy
- **Partial mesh:** selective connectivity based на requirements
- **Security benefits:** multiple paths, fault tolerance
- **Complexity challenges:** configuration management, cost considerations

**🌐 Modern Network Architectures (10 мин)**

**Hierarchical Network Design:**

**Access Layer:**
- **Function:** end-device connectivity, user access control
- **Security features:** port security, 802.1X authentication, VLAN assignment
- **Common devices:** access switches, wireless access points
- **Security concerns:** physical access, device configuration

**Distribution Layer:**
- **Function:** routing between VLANs, policy enforcement
- **Security features:** access control lists, VLAN routing, QoS
- **Common devices:** multilayer switches, distribution routers
- **Security role:** network segmentation, traffic filtering

**Core Layer:**
- **Function:** high-speed connectivity, redundant paths
- **Security considerations:** availability, performance impact
- **Common devices:** high-performance routers, core switches
- **Design principles:** minimal processing, maximum throughput

**Software-Defined Networking (SDN):**
- **Centralized control:** programmable network management
- **Security benefits:** dynamic policy enforcement, traffic visibility
- **OpenFlow protocol:** communication between controller и switches
- **Security challenges:** controller security, southbound API protection

**Network Function Virtualization (NFV):**
- **Virtual appliances:** software-based network functions
- **Security services:** virtual firewalls, virtual load balancers
- **Deployment flexibility:** on-demand security service deployment
- **Management complexity:** orchestration, lifecycle management

### **ЧАС 3: Common Network Protocols и Security Vulnerabilities (40 мин)**

#### **Теоретична част (24 мин):**

**📡 Critical Network Protocols Analysis (15 мин)**

**Address Resolution Protocol (ARP):**
- **Function:** MAC to IP address mapping
- **Security weakness:** no authentication в ARP responses
- **ARP poisoning attack:** sending fake ARP responses to redirect traffic
- **Man-in-the-middle setup:** intercepting communications между devices
- **Protection methods:** static ARP entries, ARP inspection, network monitoring

**Dynamic Host Configuration Protocol (DHCP):**
- **Function:** automatic IP address assignment
- **Rogue DHCP servers:** providing malicious network configuration
- **DHCP starvation:** exhausting available IP addresses
- **Information disclosure:** DHCP options revealing network information
- **Security measures:** DHCP snooping, authorized server lists, monitoring

**Domain Name System (DNS):**
- **Function:** domain name to IP address resolution
- **DNS cache poisoning:** injecting false DNS records
- **DNS tunneling:** using DNS за covert communication channels
- **Fast flux:** rapidly changing IP addresses за malicious domains
- **Protection strategies:** DNS filtering, secure DNS (DoH/DoT), monitoring

**Simple Network Management Protocol (SNMP):**
- **Function:** network device monitoring и management
- **Version vulnerabilities:** SNMPv1/v2c use community strings
- **Information disclosure:** device configuration, network topology
- **Default credentials:** commonly unchanged default community strings
- **Security evolution:** SNMPv3 with authentication и encryption

**Network Time Protocol (NTP):**
- **Function:** time synchronization across networks
- **Security importance:** accurate timestamps за logging, certificates
- **NTP amplification:** using NTP servers за DDoS attacks
- **Time-based attacks:** manipulating time để bypass security controls
- **Securing NTP:** authenticated NTP, internal time sources

**🔓 Protocol Vulnerability Categories (9 мин)**

**Authentication Weaknesses:**
- **Clear text protocols:** Telnet, HTTP, FTP sending credentials unencrypted
- **Weak authentication:** easily guessable passwords, default credentials
- **Replay attacks:** capturing и reusing authentication credentials
- **Protection:** encrypted protocols (SSH, HTTPS, SFTP), strong authentication

**Information Disclosure:**
- **Banner grabbing:** services revealing version information
- **Error messages:** detailed error information helping attackers
- **Network reconnaissance:** protocols revealing network topology
- **Mitigation:** service hardening, information hiding, monitoring

**Traffic Interception:**
- **Packet sniffing:** capturing network traffic в broadcast domains
- **Man-in-the-middle:** intercepting communications між parties
- **SSL/TLS weaknesses:** weak ciphers, certificate validation issues
- **Protection:** encryption, certificate pinning, secure protocols

**Denial of Service (DoS):**
- **Protocol flooding:** overwhelming services със excessive requests
- **Resource exhaustion:** consuming all available connections или memory
- **Amplification attacks:** using protocols to amplify attack traffic
- **DDoS mitigation:** rate limiting, traffic filtering, redundancy

### **ЧАС 4: Network Attack Vectors и Defense Mechanisms (40 мин)**

#### **Теоретична част (20 мин):**

**🎯 Common Network Attack Types (12 мин)**

**Reconnaissance Attacks:**
- **Network scanning:** identifying live hosts, open ports, services
- **OS fingerprinting:** determining operating systems и versions
- **Service enumeration:** gathering detailed service information
- **Vulnerability scanning:** identifying known security weaknesses
- **Social engineering:** gathering information through human interaction

**Access Attacks:**
- **Password attacks:** brute force, dictionary, rainbow tables
- **Protocol exploitation:** leveraging protocol weaknesses
- **Man-in-the-middle:** intercepting communications between parties
- **Session hijacking:** taking over established network sessions
- **Privilege escalation:** gaining higher access levels

**DoS/DDoS Attacks:**
- **Volumetric attacks:** overwhelming network bandwidth
- **Protocol attacks:** exploiting protocol weaknesses
- **Application attacks:** targeting specific applications или services
- **Distributed attacks:** using multiple sources за amplification
- **Reflection attacks:** using third-party servers за attack amplification

**Advanced Persistent Threats (APT):**
- **Initial compromise:** gaining initial network access
- **Lateral movement:** spreading through network
- **Persistence:** maintaining long-term access
- **Data exfiltration:** stealing sensitive information
- **Command и control:** maintaining communication с attackers

**🛡️ Network Defense Strategies (8 мин)**

**Defense in Depth:**
- **Perimeter security:** firewalls, intrusion detection
- **Network segmentation:** VLANs, subnets, access controls
- **Endpoint protection:** antivirus, host-based firewalls
- **Application security:** secure coding, input validation
- **Data protection:** encryption, access controls, backup

**Network Access Control (NAC):**
- **Device identification:** identifying connected devices
- **Policy enforcement:** applying appropriate access policies
- **Quarantine capabilities:** isolating non-compliant devices
- **Guest network access:** providing limited access to visitors
- **Monitoring и reporting:** tracking device compliance

**Intrusion Prevention:**
- **Signature-based detection:** identifying known attack patterns
- **Anomaly detection:** identifying unusual network behavior
- **Real-time response:** blocking malicious traffic automatically
- **Threat intelligence:** incorporating latest threat information
- **Incident response:** coordinated response to detected threats
