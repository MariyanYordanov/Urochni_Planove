### **ЧАС 1: OSI Model и TCP/IP Security Fundamentals (40 мин)**

#### **Теоретична част (25 мин):**

**OSI Model Security Perspective (12 мин)**

**Layer-by-Layer Security Analysis:**

**Layer 1 - Physical Layer:**
- **Security concerns:** прехващане на кабели, подслушване на оптични влакна
- **Protective measures:** защитени кабелни трасета, защита с тръби
- **Attack examples:** прихващане чрез wire shark, електромагнитни смущения
- **Detection methods:** контрол на физически достъп, мониторинг на целостта на кабелите

**Layer 2 - Data Link Layer:**
- **Security protocols:** 802.1X удостоверяване на база порт
- **Vulnerabilities:** ARP poisoning, MAC flooding, VLAN hopping
- **Protection:** MAC address filtering, port security, private VLANs
- **Switching security:** сигурна конфигурация на switch, защита на trunk

**Layer 3 - Network Layer:**
- **IP security:** IPSec, packet filtering, routing security
- **Vulnerabilities:** IP spoofing, атаки срещу routing, ICMP атаки
- **Protection:** anti-spoofing филтри, удостоверяване на routing протоколи
- **Network segmentation:** subnetting за security изолация

**Layer 4 - Transport Layer:**
- **TCP/UDP security:** port-базирано филтриране, мониторинг на connection state
- **Vulnerabilities:** TCP hijacking, UDP flooding, port scanning
- **Protection:** stateful firewalls, rate limiting, мониторинг на връзките
- **Security protocols:** SSL/TLS точки на прекратяване

**Layers 5-7 - Session, Presentation, Application:**
- **Application security:** deep packet inspection, application firewalls
- **Data encryption:** end-to-end криптиране, protocol security
- **User authentication:** контрол на достъп на ниво приложение
- **Content filtering:** открИване на malware, предотвратяване на изтичане на данни

**TCP/IP Protocol Suite Security (13 мин)**

**Internet Protocol (IP) Security Issues:**
- **IPv4 limitations:** изчерпване на адресното пространство води до усложнения с NAT
- **IPv6 security:** по-голямо адресно пространство, задължителен IPSec, нови attack vectors
- **IP fragmentation:** атаки с фрагментирани пакети
- **Source routing:** възможност за заобикаляне на security контроли

**Transmission Control Protocol (TCP):**
- **Connection establishment:** уязвимости при three-way handshake
- **Sequence number attacks:** атаки чрез предсказване на TCP sequence числа
- **Connection hijacking:** поемане на установени TCP връзки
- **SYN flood attacks:** претоварване на цел с connection заявки

**User Datagram Protocol (UDP):**
- **Connectionless nature:** липса на вградена надеждност или security
- **UDP flooding:** претоварване на цел с UDP трафик
- **DNS security:** DNS cache poisoning, DNS tunneling
- **DHCP security:** rogue DHCP сървъри, DHCP spoofing

**Internet Control Message Protocol (ICMP):**
- **Network diagnostics:** ping, traceroute функционалност
- **Security implications:** мрежово разузнаване, скрити канали
- **ICMP attacks:** ping of death, ICMP tunneling
- **Protection strategies:** ICMP филтриране, rate limiting

### **ЧАС 2: Network Topology и Security Architecture (40 мин)**

#### **Теоретична част (22 мин):**

**Network Topology Security Impact (12 мин)**

**Traditional Network Topologies:**

**Bus Topology:**
- **Security characteristics:** единичен collision domain, споделена среда
- **Vulnerabilities:** лесно подслушване, единична точка на отказ
- **Legacy considerations:** по-стари Ethernet мрежи, коаксиални кабелни системи
- **Security controls:** ограничени възможности, физическата security е критична

**Star Topology:**
- **Security advantages:** централизиран контрол, изолирани връзки
- **Hub vs Switch:** collision domains, security последствия
- **Single point of failure:** уязвимост на централния hub/switch
- **Security controls:** port-базирана security, мониторинг на трафика

**Ring Topology:**
- **Token-based access:** детерминистичен контрол на достъпа
- **Fault tolerance:** конфигурации с двоен пръстен
- **Security considerations:** манипулация на token, анализ на трафиковия поток
- **Modern applications:** FDDI, SONET/SDH мрежи

**Mesh Topology:**
- **Full mesh:** пълна свързаност, максимална резервираност
- **Partial mesh:** избирателна свързаност съобразно изискванията
- **Security benefits:** множество пътища, устойчивост на откази
- **Complexity challenges:** управление на конфигурацията, разходни съображения

**Modern Network Architectures (10 мин)**

**Hierarchical Network Design:**

**Access Layer:**
- **Function:** свързване на крайни устройства, контрол на потребителски достъп
- **Security features:** port security, 802.1X удостоверяване, VLAN разпределение
- **Common devices:** access switches, wireless access points
- **Security concerns:** физически достъп, конфигурация на устройствата

**Distribution Layer:**
- **Function:** routing между VLANs, прилагане на политики
- **Security features:** access control lists, VLAN routing, QoS
- **Common devices:** multilayer switches, distribution routers
- **Security role:** network segmentation, филтриране на трафика

**Core Layer:**
- **Function:** високоскоростна свързаност, резервирани пътища
- **Security considerations:** наличност, влияние върху производителността
- **Common devices:** високопроизводителни routers, core switches
- **Design principles:** минимална обработка, максимална пропускателна способност

**Software-Defined Networking (SDN):**
- **Centralized control:** програмируемо управление на мрежата
- **Security benefits:** динамично прилагане на политики, видимост на трафика
- **OpenFlow protocol:** комуникация между контролера и switches
- **Security challenges:** security на контролера, защита на southbound API

**Network Function Virtualization (NFV):**
- **Virtual appliances:** мрежови функции на software основа
- **Security services:** виртуални firewalls, виртуални load balancers
- **Deployment flexibility:** разгръщане на security услуги при поискване
- **Management complexity:** оркестрация, управление на жизнения цикъл

### **ЧАС 3: Common Network Protocols и Security Vulnerabilities (40 мин)**

#### **Теоретична част (24 мин):**

**Critical Network Protocols Analysis (15 мин)**

**Address Resolution Protocol (ARP):**
- **Function:** съпоставяне на MAC с IP адрес
- **Security weakness:** липса на удостоверяване в ARP отговорите
- **ARP poisoning attack:** изпращане на фалшиви ARP отговори за пренасочване на трафика
- **Man-in-the-middle setup:** прихващане на комуникации между устройства
- **Protection methods:** статични ARP записи, ARP inspection, мрежов мониторинг

**Dynamic Host Configuration Protocol (DHCP):**
- **Function:** автоматично разпределяне на IP адреси
- **Rogue DHCP servers:** предоставяне на злонамерена мрежова конфигурация
- **DHCP starvation:** изчерпване на наличните IP адреси
- **Information disclosure:** DHCP опции, разкриващи мрежова информация
- **Security measures:** DHCP snooping, списъци с разрешени сървъри, мониторинг

**Domain Name System (DNS):**
- **Function:** преобразуване на домейн имена в IP адреси
- **DNS cache poisoning:** инжектиране на неверни DNS записи
- **DNS tunneling:** използване на DNS за скрити комуникационни канали
- **Fast flux:** бързо сменящи се IP адреси за злонамерени домейни
- **Protection strategies:** DNS филтриране, secure DNS (DoH/DoT), мониторинг

**Simple Network Management Protocol (SNMP):**
- **Function:** мониторинг и управление на мрежови устройства
- **Version vulnerabilities:** SNMPv1/v2c използват community strings
- **Information disclosure:** конфигурация на устройствата, мрежова топология
- **Default credentials:** нечесто сменяни default community strings
- **Security evolution:** SNMPv3 с удостоверяване и криптиране

**Network Time Protocol (NTP):**
- **Function:** синхронизация на времето в мрежите
- **Security importance:** точни timestamps за логове, сертификати
- **NTP amplification:** използване на NTP сървъри за DDoS атаки
- **Time-based attacks:** манипулиране на времето за заобикаляне на security контроли
- **Securing NTP:** удостоверен NTP, вътрешни time sources

**Protocol Vulnerability Categories (9 мин)**

**Authentication Weaknesses:**
- **Clear text protocols:** Telnet, HTTP, FTP изпращат credentials некриптирани
- **Weak authentication:** лесно познаеми пароли, default credentials
- **Replay attacks:** прихващане и повторно използване на credentials
- **Protection:** криптирани протоколи (SSH, HTTPS, SFTP), силно удостоверяване

**Information Disclosure:**
- **Banner grabbing:** услуги, разкриващи информация за версията
- **Error messages:** подробни съобщения за грешки, помагащи на атакуващите
- **Network reconnaissance:** протоколи, разкриващи мрежова топология
- **Mitigation:** укрепване на услугите, скриване на информация, мониторинг

**Traffic Interception:**
- **Packet sniffing:** прихващане на мрежов трафик в broadcast домейни
- **Man-in-the-middle:** прихващане на комуникации между страни
- **SSL/TLS weaknesses:** слаби cipher-и, проблеми с валидацията на сертификати
- **Protection:** криптиране, certificate pinning, сигурни протоколи

**Denial of Service (DoS):**
- **Protocol flooding:** претоварване на услуги с прекомерен брой заявки
- **Resource exhaustion:** изчерпване на всички налични връзки или памет
- **Amplification attacks:** използване на протоколи за усилване на атакуващия трафик
- **DDoS mitigation:** rate limiting, филтриране на трафика, резервираност

### **ЧАС 4: Network Attack Vectors и Defense Mechanisms (40 мин)**

#### **Теоретична част (20 мин):**

**Common Network Attack Types (12 мин)**

**Reconnaissance Attacks:**
- **Network scanning:** идентифициране на активни хостове, отворени портове, услуги
- **OS fingerprinting:** определяне на операционни системи и версии
- **Service enumeration:** събиране на подробна информация за услугите
- **Vulnerability scanning:** идентифициране на известни security слабости
- **Social engineering:** събиране на информация чрез взаимодействие с хора

**Access Attacks:**
- **Password attacks:** brute force, речникови атаки, rainbow tables
- **Protocol exploitation:** използване на слабости на протоколите
- **Man-in-the-middle:** прихващане на комуникации между страни
- **Session hijacking:** поемане на установени мрежови сесии
- **Privilege escalation:** получаване на по-високо ниво на достъп

**DoS/DDoS Attacks:**
- **Volumetric attacks:** претоварване на мрежовата честотна лента
- **Protocol attacks:** използване на слабости на протоколите
- **Application attacks:** насочване срещу конкретни приложения или услуги
- **Distributed attacks:** използване на множество източници за усилване
- **Reflection attacks:** използване на трети сървъри за усилване на атаката

**Advanced Persistent Threats (APT):**
- **Initial compromise:** получаване на начален мрежов достъп
- **Lateral movement:** разпространяване из мрежата
- **Persistence:** поддържане на дългосрочен достъп

Implementation Priority: Immediate

Usage Context: Name resolution, internal и external queries
