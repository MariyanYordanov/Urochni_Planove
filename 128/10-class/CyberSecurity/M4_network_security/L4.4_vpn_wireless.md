### **ЧАС 1: VPN FUNDAMENTALS И ARCHITECTURE (40 мин)**

#### **Теоретична част (22 мин):**

**🌐 VPN Concept и Business Drivers (8 мин)**

**Virtual Private Network Definition:**
VPN създава secure, encrypted connection over public network (като Internet) за provide private communication между remote locations или users и corporate network.

**Business Justifications за VPN:**
- **Remote workforce support** - secure access за employees working от home
- **Cost reduction** - avoid expensive dedicated network lines
- **Geographic expansion** - connect offices across different locations  
- **BYOD enablement** - secure personal device access за business resources
- **Compliance requirements** - meet regulatory needs за secure data transmission

**VPN Security Benefits:**
- **Data encryption** - protect информация от eavesdropping
- **Identity verification** - ensure only authorized users gain access
- **Traffic tunneling** - hide network traffic from potential attackers
- **Geographic flexibility** - secure access от anywhere в world
- **Network segmentation** - isolate VPN traffic от general internet

**🏗️ VPN Deployment Models (7 мин)**

**Site-to-Site VPN (Branch Office Connectivity):**
- **Purpose:** Connect multiple office locations через secure tunnels
- **Implementation:** VPN gateways на each location establish permanent connections
- **Use cases:** Branch offices, partner organizations, cloud data centers
- **Advantages:** Transparent connectivity, centralized management
- **Considerations:** Bandwidth requirements, redundancy planning

*Example Scenario:* София head office connected to Пловдив branch office със permanent encrypted tunnel

**Remote Access VPN (Individual User Connectivity):**
- **Purpose:** Individual users connect to corporate network от remote locations
- **Implementation:** VPN client software на user devices connects to corporate VPN server
- **Use cases:** Work from home, traveling employees, contractors
- **Advantages:** Flexible access, user mobility, controlled permissions
- **Considerations:** Client management, user training, support complexity

**Client-to-Site VPN (Application-Specific Access):**
- **Purpose:** Specific applications или services access through VPN
- **Implementation:** Application-level VPN integration или proxy solutions
- **Use cases:** Cloud applications, specific server access, limited connectivity
- **Advantages:** Granular control, minimal client footprint, application security
- **Considerations:** Application compatibility, performance overhead

**🔧 VPN Technologies Comparison (7 мин)**

**IPSec (Internet Protocol Security):**
- **Layer:** Network layer (Layer 3) encryption
- **Protocols:** ESP (Encapsulating Security Payload), AH (Authentication Header)
- **Modes:** Tunnel mode (entire packet encrypted), Transport mode (payload only)
- **Advantages:** Strong security, wide vendor support, mature technology
- **Disadvantages:** Complex configuration, NAT traversal issues, firewall compatibility

**SSL/TLS VPN (Secure Sockets Layer/Transport Layer Security):**
- **Layer:** Application layer encryption using HTTPS
- **Access methods:** Web browser-based, thin client, full client
- **Advantages:** No client software required, firewall friendly, easy deployment
- **Disadvantages:** Limited application support, potential performance issues
- **Use cases:** Web applications, email access, file sharing

**OpenVPN (Open Source VPN):**
- **Technology:** SSL/TLS based с custom protocol
- **Advantages:** Cross-platform support, strong encryption, active community
- **Configuration:** Certificate-based authentication, flexible routing
- **Disadvantages:** Requires client installation, not native OS integration
- **Use cases:** Small businesses, personal use, mixed environments

**WireGuard (Modern VPN Protocol):**
- **Technology:** Modern cryptography, minimal codebase
- **Advantages:** High performance, simple configuration, built-in roaming
- **Features:** Always-on connectivity, minimal battery impact
- **Considerations:** Newer technology, limited enterprise features
- **Use cases:** Mobile devices, high-performance requirements

### **ЧАС 2: VPN PROTOCOLS И CONFIGURATION (40 мин)**

#### **Теоретична част (18 мин):**

**🔒 IPSec Deep Dive (9 мин)**

**IPSec Protocol Suite Components:**

**Authentication Header (AH):**
- **Function:** Provides authentication и integrity verification
- **Protection:** Source authentication, data integrity, anti-replay protection
- **No encryption:** Data remains readable, только verified
- **Use cases:** Scenarios where encryption is не required или prohibited
- **Limitations:** Cannot protect против traffic analysis

**Encapsulating Security Payload (ESP):**
- **Function:** Provides encryption, authentication, и integrity
- **Protection:** Data confidentiality + all AH protections
- **Encryption:** Data becomes unreadable to unauthorized parties
- **Use cases:** Most common IPSec implementation за full protection
- **Flexibility:** Can be used with или без authentication

**IPSec Modes:**

**Transport Mode:**
- **Scope:** Encrypts только payload на IP packet, leaves headers intact
- **Use case:** Host-to-host communications
- **Advantages:** Minimal overhead, preserves IP routing information
- **Disadvantages:** Source и destination IPs are visible
- **Best for:** Application-level security, host-based VPNs

**Tunnel Mode:**
- **Scope:** Encrypts entire original IP packet, adds new IP header
- **Use case:** Gateway-to-gateway communications (site-to-site VPNs)
- **Advantages:** Complete traffic hiding, flexible routing
- **Disadvantages:** Higher overhead, more complex configuration
- **Best for:** Network-level security, remote access VPNs

**IPSec Key Management:**
- **IKE (Internet Key Exchange):** Automated key negotiation protocol
- **Manual keys:** Pre-shared keys configured manually (не recommended за production)
- **Certificate-based:** Public key infrastructure за scalable authentication
- **Perfect Forward Secrecy:** Ensures session keys cannot be compromised retroactively

**🌐 SSL/TLS VPN Implementation (9 мин)**

**SSL VPN Access Methods:**

**Clientless (Browser-Based) Access:**
- **Technology:** Standard web browser с HTTPS connection
- **Applications:** Web-based applications, file shares, email
- **Advantages:** No software installation, universal compatibility
- **Limitations:** Limited application support, potential security gaps
- **User experience:** Familiar browser interface, easy troubleshooting

**Thin Client (Java/ActiveX) Access:**
- **Technology:** Browser plugins за extend functionality
- **Applications:** Network applications, database access, specialized software
- **Advantages:** Broader application support than clientless
- **Limitations:** Browser plugin security concerns, compatibility issues
- **Deployment:** Automatic plugin download, но may require administrator rights

**Full Client (Network Extension) Access:**
- **Technology:** Complete VPN client software installation
- **Applications:** All network applications, complete network transparency
- **Advantages:** Full functionality, maximum compatibility
- **Limitations:** Client installation и management overhead
- **Use cases:** Power users, complex application requirements

**SSL VPN Security Features:**

**Application Control:**
- **Policy enforcement:** Restrict access to specific applications и resources
- **Content inspection:** Scan files и data for security threats
- **Data loss prevention:** Monitor и control information leaving organization
- **Session recording:** Maintain audit trails на user activities

**Endpoint Compliance:**
- **Antivirus checking:** Verify endpoint has current antivirus protection
- **Patch level validation:** Ensure operating system и applications are updated
- **Personal firewall:** Confirm endpoint firewall is active и configured
- **Device registration:** Control which devices can establish VPN connections

### **ЧАС 3: WIRELESS SECURITY FUNDAMENTALS (40 мин)**

#### **Теоретична част (25 мин):**

**📡 Wi-Fi Security Evolution (10 мин)**

**Historical Progression на Wireless Security:**

**WEP (Wired Equivalent Privacy) - 1990s:**
- **Security level:** Extremely weak, easily broken
- **Encryption:** 40-bit или 104-bit RC4 encryption
- **Vulnerabilities:** Static keys, weak authentication, IV attacks
- **Current status:** Should never be used, но still found на legacy devices
- **Attack time:** Can be broken в minutes с standard tools

**WPA (Wi-Fi Protected Access) - 2003:**
- **Improvements:** Dynamic keys, stronger authentication
- **Encryption:** TKIP (Temporal Key Integrity Protocol)
- **Authentication:** PSK (Pre-Shared Key) или Enterprise (802.1X)
- **Vulnerabilities:** TKIP attacks possible, но significantly harder than WEP
- **Status:** Legacy standard, not recommended за new deployments

**WPA2 (Wi-Fi Protected Access 2) - 2004:**
- **Encryption:** AES-CCMP (Advanced Encryption Standard)
- **Security level:** Strong protection when properly configured
- **Authentication modes:** Personal (PSK) и Enterprise (802.1X/RADIUS)
- **Vulnerabilities:** KRACK attack (2017), WPS PIN attacks
- **Status:** Still widely used, acceptable за most applications

**WPA3 (Wi-Fi Protected Access 3) - 2018:**
- **Improvements:** Enhanced password protection, forward secrecy
- **SAE (Simultaneous Authentication на Equals):** Replaces PSK authentication
- **Enhanced Open:** Protection на public Wi-Fi networks
- **Enterprise improvements:** 192-bit encryption за high-security environments
- **Status:** Latest standard, recommended за all new deployments

**🏢 Enterprise vs Personal Wi-Fi Security (8 мин)**

**Personal Mode (PSK - Pre-Shared Key):**
- **Authentication:** Single password shared by all users
- **Use cases:** Home networks, small offices, guest networks
- **Advantages:** Simple setup, no additional infrastructure required
- **Disadvantages:** Password sharing, no individual user tracking
- **Security considerations:** Password strength critical, regular changes needed

**Enterprise Mode (802.1X Authentication):**
- **Authentication:** Individual credentials за each user
- **Infrastructure:** RADIUS server required за authentication
- **Certificate management:** PKI infrastructure за device certificates
- **Benefits:** Individual accountability, centralized management, stronger security
- **Complexity:** Requires specialized knowledge и infrastructure

**802.1X Authentication Process:**
1. **Supplicant (client)** requests network access
2. **Authenticator (access point)** forwards request to authentication server
3. **Authentication server (RADIUS)** verifies credentials
4. **Dynamic key generation** - unique keys за each session
5. **Network access granted** с session-specific encryption

**Enterprise Authentication Methods:**
- **EAP-TLS:** Certificate-based authentication (strongest security)
- **PEAP-MSCHAPv2:** Password-based с certificate validation
- **EAP-TTLS:** Flexible inner authentication methods
- **EAP-FAST:** Cisco proprietary, certificate-less deployment

**🚨 Common Wireless Threats (7 мин)**

**Rogue Access Points:**
- **Threat:** Unauthorized access points на corporate network
- **Risks:** Network access bypass, data interception
- **Detection:** Wireless scanning, network monitoring, physical surveys
- **Prevention:** Access point management, employee education
- **Response:** Immediate removal, security incident investigation

**Evil Twin Attacks:**
- **Technique:** Fake access point mimicking legitimate network
- **Goal:** Credential harvesting, traffic interception
- **Indicators:** Multiple networks със same name, unusual authentication prompts
- **Protection:** Certificate validation, VPN usage, network verification
- **User education:** Verify network authenticity before connecting

**Wi-Fi Deauthentication Attacks:**
- **Method:** Sending deauth frames to disconnect users
- **Impact:** Denial на service, forcing reconnection
- **Tools:** Aircrack-ng suite, specialized hardware
- **Protection:** Management frame protection (802.11w), monitoring systems
- **Business impact:** Productivity loss, potential security incidents

**WPS (Wi-Fi Protected Setup) Vulnerabilities:**
- **PIN attack:** Brute force attack против WPS PIN
- **Attack time:** WPA2 network can be compromised в hours
- **Vulnerability:** Mathematical flaw в WPS implementation
- **Mitigation:** Disable WPS на all access points
- **Current status:** WPS should be disabled на enterprise networks

**Wireless Eavesdropping:**
- **Technique:** Passive monitoring на wireless communications
- **Targets:** Unencrypted traffic, weak encryption protocols
- **Tools:** Wireless packet capture tools, protocol analyzers
- **Protection:** Strong encryption, VPN usage, HTTPS везде
- **Legal considerations:** Monitoring others' communications may be illegal

**📱 Mobile Threat Landscape (10 мин)**

**Unique Mobile Security Challenges:**

**Device Mobility:**
- **Physical security:** Easy to lose, steal, или leave unattended
- **Network diversity:** Connects to multiple untrusted networks
- **Location privacy:** GPS tracking, location-based services
- **Public access:** Used в public spaces где observation is possible

**Operating System Vulnerabilities:**
- **Fragmented updates:** Android devices с delayed security patches
- **Jailbreaking/Rooting:** Users bypass built-in security controls
- **App store risks:** Malicious apps в official и third-party stores
- **Legacy support:** Older devices без current security updates

**Application Security Risks:**
- **Over-privileged apps:** Applications requesting unnecessary permissions
- **Data leakage:** Apps transmitting sensitive information inappropriately
- **Insecure storage:** Credentials и data stored в plaintext
- **Network communications:** Unencrypted data transmission

**📋 BYOD (Bring Your Own Device) Policies (8 мин)**

**BYOD Business Drivers:**
- **Cost reduction:** Employees purchase their own devices
- **User satisfaction:** People prefer using familiar devices
- **Productivity increase:** Access to business data anytime, anywhere
- **Technology adoption:** Faster adoption на new technologies

**BYOD Security Challenges:**

**Device Management:**
- **Inventory tracking:** Unknown devices accessing corporate resources
- **Configuration control:** Inability to enforce security settings
- **Update management:** Ensuring devices have current security patches
- **End-of-life handling:** Secure data removal когато devices are retired

**Data Protection:**
- **Data segregation:** Separating personal и business information
- **Data loss prevention:** Controlling how sensitive data is shared
- **Remote wipe capability:** Ability to remove business data remotely
- **Backup и recovery:** Ensuring business data is protected

**Compliance Considerations:**
- **Regulatory requirements:** Meeting industry-specific data protection standards
- **Audit trails:** Maintaining records на data access и usage
- **Privacy concerns:** Balancing business security със employee privacy
- **Legal jurisdiction:** Understanding data protection laws в different regions

**BYOD Implementation Strategies:**

**Full Access Model:**
- **Description:** Personal devices have complete access to corporate resources
- **Benefits:** Maximum flexibility и user satisfaction
- **Risks:** Highest security exposure, difficult data control
- **Use cases:** Small organizations със high trust levels

**Limited Access Model:**
- **Description:** Personal devices can access только specific applications/data
- **Benefits:** Balanced security и usability
- **Implementation:** Application wrapping, secure containers
- **Use cases:** Medium organizations със defined security requirements

**Secure Container Model:**
- **Description:** Business data kept в separate, managed container на device
- **Benefits:** Strong data segregation, centralized management
- **Challenges:** User experience complexity, dual-app management
- **Use cases:** Large organizations със strict compliance requirements

- **Device enrollment:** Automated setup и configuration

### **ЧАС 5: PUBLIC WI-FI SAFETY И ZERO TRUST ACCESS (40 мин)**

#### **Теоретична част (22 мин):**

**☕ Public Wi-Fi Security Risks (10 мин)**

**Common Public Wi-Fi Environments:**
- **Coffee shops и restaurants** - customer convenience networks
- **Hotels и conference centers** - guest accommodation services  
- **Airports и transportation hubs** - traveler connectivity needs
- **Shopping centers и retail** - customer engagement tools
- **Educational institutions** - visitor и guest access networks

**Attack Vectors в Public Wi-Fi:**

**Man-in-the-Middle (MITM) Attacks:**
- **Technique:** Attacker intercepts communications между user и legitimate service
- **Implementation:** Rogue access points, ARP poisoning, DNS manipulation
- **Targets:** Login credentials, email communications, financial transactions
- **Indicators:** SSL certificate warnings, unusual network behavior
- **Protection:** VPN usage, certificate validation, HTTPS enforcement

**Packet Sniffing и Eavesdropping:**
- **Method:** Passive monitoring на network traffic
- **Vulnerable data:** Unencrypted communications, HTTP traffic, email
- **Tools:** Wireshark, tcpdump, specialized hardware
- **Business impact:** Credential theft, confidential information exposure
- **Mitigation:** End-to-end encryption, VPN tunnels

**Rogue Hotspot Creation:**
- **Technique:** Creating fake access points със legitimate-sounding names
- **Examples:** "Airport_WiFi_Free", "Hotel_Guest_Network"
- **User deception:** People connect thinking it's legitimate service
- **Data collection:** All traffic passes through attacker's system
- **Prevention:** Network authentication, VPN usage, corporate policies

**Malware Distribution:**
- **Vector:** Compromise других users на same network
- **Methods:** Network worms, file sharing exploits, social engineering
- **Payloads:** Keyloggers, ransomware, information stealers
- **Lateral movement:** Attacking multiple devices на shared network
- **Defense:** Updated antivirus, network segmentation, firewalls

**🛡️ Public Wi-Fi Protection Strategies (12 мин)**

**Technical Protection Measures:**

**VPN Usage Best Practices:**
- **Always-on VPN:** Automatic connection when joining public networks
- **Kill switch:** Disconnect internet если VPN fails
- **DNS protection:** Prevent DNS hijacking through VPN provider
- **Provider selection:** Choose reputable VPN services със no-logs policies
- **Performance considerations:** Select servers близо geographic location

**Application Security:**
- **HTTPS enforcement:** Browser settings requiring encrypted connections
- **Email encryption:** S/MIME или PGP за sensitive communications
- **Messaging apps:** Use end-to-end encrypted messaging (Signal, WhatsApp)
- **File sharing:** Secure cloud storage със client-side encryption
- **Banking/financial:** Use dedicated apps, avoid web browsers за sensitive transactions

**Device Configuration:**
- **Disable auto-connect:** Prevent automatic connection to remembered networks
- **Network forgetting:** Remove public networks після use
- **File sharing disabled:** Turn off Windows file sharing, AirDrop
- **Firewall enabled:** Host-based firewall blocking incoming connections
- **Wi-Fi hotspot security:** Secure personal hotspot quando sharing connection

**Behavioral Security Practices:**

**Network Verification:**
- **Ask staff:** Confirm network name и password с establishment
- **Official networks only:** Avoid networks не affiliated с business
- **Certificate validation:** Don't ignore SSL certificate warnings
- **Login page scrutiny:** Verify authentic captive portal pages
- **Connection monitoring:** Watch за unusual prompts или behavior

**Data Handling Guidelines:**
- **Sensitive data restriction:** No banking, confidential work в public
- **Offline capability:** Download files за offline work când possible
- **Cloud synchronization:** Delay sync activities until secure network
- **Communication timing:** Schedule sensitive calls/meetings за secure locations
- **Physical security:** Be aware на shoulder surfing, device theft

**🔐 Zero Trust Network Architecture (10 мин)**

**Traditional Network Security Model (Perimeter-Based):**
- **Concept:** Strong perimeter security със trusted internal network
- **Assumption:** Internal users и devices are trustworthy
- **Implementation:** Firewalls, VPNs, network access control
- **Limitations:** Insider threats, lateral movement, remote work challenges
- **Modern inadequacy:** Digital transformation, cloud adoption, mobile workforce

**Zero Trust Principles:**
- **Never trust, always verify** - continuous authentication и authorization
- **Least privilege access** - minimum required permissions за each user/device
- **Assume breach** - design под assumption that network is compromised
- **Verify explicitly** - use all available data points за access decisions
- **Continuous monitoring** - ongoing assessment на security posture

**Zero Trust Implementation Components:**

**Identity и Access Management (IAM):**
- **Multi-factor authentication (MFA)** - verify user identity със multiple factors
- **Single sign-on (SSO)** - centralized authentication за all applications
- **Privileged access management (PAM)** - control administrative access
- **Identity governance** - lifecycle management на user accounts и permissions

**Device Trust Assessment:**
- **Device registration** - known, managed devices only
- **Compliance checking** - verify security policies are enforced
- **Risk scoring** - assess device security posture
- **Conditional access** - grant access based на device trust level

**Network Micro-Segmentation:**
- **Application-level segmentation** - isolate individual applications
- **User-based segmentation** - network access based на user identity
- **Device-type segmentation** - different access за different device categories
- **Dynamic segmentation** - real-time adjustment based на risk factors
