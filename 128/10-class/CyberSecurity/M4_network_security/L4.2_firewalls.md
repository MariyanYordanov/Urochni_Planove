### **ЧАС 1: FIREWALL FUNDAMENTALS И АРХИТЕКТУРИ (40 мин)**

#### **Теоретична част (20 мин):**

**Какво е Firewall? (5 мин)**

**Definition и Purpose:**
Firewall е мрежово security устройство или software, което контролира входящия и изходящия мрежов трафик въз основа на предварително зададени security правила. Действа като бариера между доверената вътрешна мрежа и недоверени external мрежи (като Internet).

**Primary Functions:**
- **Traffic filtering** - позволява или блокира traffic based на rules
- **Access control** - определя кой може да access specific network resources
- **Logging и monitoring** - записва network activities за analysis
- **NAT (Network Address Translation)** - скрива internal network structure

**Firewall Архитектури (8 мин)**

**Packet Filtering Firewalls (Stateless):**
- **Механизъм:** Разглежда отделни пакети независимо от контекста на връзката
- **Decision factors:** Source/destination IP, port numbers, protocol type
- **Advantages:** Бърза обработка, ниско използване на ресурси
- **Disadvantages:** Липса на awareness за connection state, уязвимост към сложни атаки
- **Use cases:** Основно периметрово филтриране, среди с висока пропускателна способност

*Пример: Router ACL правило блокиращо всички TCP connections към port 23 (Telnet)*

**Stateful Inspection Firewalls:**
- **Механизъм:** Проследява connection state и прилага правила въз основа на контекста на връзката
- **State tracking:** TCP handshakes, установени връзки, информация за сесията
- **Dynamic rule application:** Автоматично позволява обратен трафик за установени връзки
- **Security improvement:** Предотвратява атаки за предсказване на TCP sequence
- **Performance:** Умерено използване на ресурси, отличен баланс security/производителност

*Пример: Firewall позволява outbound HTTPS connection и automatically allows corresponding inbound response traffic*

**Application Layer Firewalls (Proxy Firewalls):**
- **Deep packet inspection:** Анализира протоколи на ниво приложение (HTTP, FTP, SMTP)
- **Content filtering:** Разглежда действителното съдържание на данните, не само headers
- **Protocol validation:** Осигурява, че трафикът отговаря на стандартите за протоколи
- **Advanced threat detection:** Идентифицира атаки на application layer
- **Resource intensive:** Значителни изисквания за CPU и памет

*Пример: Web Application Firewall (WAF) detecting SQL injection attempts in HTTP requests*

**Next-Generation Firewalls (NGFW):**
- **Integrated security features:** Traditional firewall + IPS + application awareness
- **Application identification:** Разпознава приложения независимо от порта/протокола
- **User identity integration:** Свързва мрежовата активност с конкретни потребители
- **Threat intelligence:** Актуализации в реално време за известни злонамерени IP/домейни
- **SSL inspection:** Декриптира и анализира криптиран трафик

*Пример: NGFW detecting cryptocurrency mining traffic disguised as HTTPS communication*

**Evolution Summary (2 мин)**
*1980s:* Packet filtering → *1990s:* Stateful inspection → *2000s:* Application awareness → *2010s+:* Integrated threat prevention

### **ЧАС 2: NETWORK SEGMENTATION СТРАТЕГИИ (40 мин)**

#### **Теоретична част (22 мин):**

**Network Segmentation Principles (8 мин)**

**Fundamental Concept:**
Network segmentation е практиката на разделяне на мрежата на по-малки, изолирани подмрежи за подобряване на security, производителността и съответствието.

**Core Benefits:**
- **Lateral movement prevention** - атакуващите не могат да се разпространят свободно
- **Improved performance** - намалени broadcast домейни, оптимизиран трафик
- **Enhanced monitoring** - по-лесен анализ на трафика и засичане на аномалии
- **Compliance isolation** - разделяне на регулираните данни от общите мрежови ресурси
- **Incident containment** - security пробивите остават локализирани

**Segmentation Approaches:**

**Physical Segmentation:**
- **Separate hardware** за различни мрежови зони
- **Dedicated switches/routers** за всеки security домейн
- **Air-gapped networks** за най-високи security изисквания
- **Advantages:** Пълна изолация, без споделени уязвимости
- **Disadvantages:** Висока цена, сложност на управлението

**Logical Segmentation (VLANs):**
- **Virtual LANs** на обща физическа инфраструктура
- **IEEE 802.1Q standard** за VLAN tagging
- **Software-defined boundaries** между мрежовите сегменти
- **Cost-effective** решение за повечето организации
- **Flexible reconfiguration** без физически промени

**VLAN Security Implementation (7 мин)**

**VLAN Security Benefits:**
- **Broadcast domain isolation** - ограничава влиянието на broadcast storms
- **Access control** - ограничава комуникацията между VLANs
- **Traffic monitoring** - по-лесен анализ на конкретни мрежови сегменти
- **Policy enforcement** - различни security политики за различни VLANs

**VLAN Security Vulnerabilities:**

**VLAN Hopping Attacks:**
- **Switch spoofing** - атакуващият имитира switch поведение за достъп до множество VLANs
- **Double tagging** - експлоатация на слабости на native VLAN
- **Prevention:** Деактивиране на неизползвани портове, конфигуриране на port security, избягване на default VLANs

**VLAN Configuration Best Practices:**
- **Change default VLAN** (VLAN 1) за native трафик
- **Disable unused ports** и присвояване към dummy VLAN
- **Configure port security** - ограничаване на MAC адреси
- **Implement VLAN ACLs** за контрол на inter-VLAN комуникацията
- **Use 802.1X** за динамично присвояване на VLAN на базата на потребителско удостоверяване

**DMZ Design и Perimeter Security (7 мин)**

**DMZ (Demilitarized Zone) Concept:**
DMZ е неутрална зона между вътрешната доверена мрежа и external недоверена мрежа (Internet). Служи като буферна зона, където се поставят публично достъпни услуги.

**DMZ Architecture Components:**

**Three-Legged Firewall Design:**
- **External interface** - свързва се с Internet
- **Internal interface** - свързва се с LAN
- **DMZ interface** - хоства публични сървъри (web, email, DNS)
- **Traffic control** - строги правила между всички зони

**Dual Firewall Design:**
- **Perimeter firewall** - защита, обърната към Internet
- **Internal firewall** - допълнителна защита за LAN
- **DMZ между firewalls** - изолирана сървърна зона
- **Defense-in-depth** - множество security слоеве

**Common DMZ Services:**
- **Web servers** - хостване на публичен уебсайт
- **Email servers** - SMTP/POP3/IMAP услуги
- **DNS servers** - публично разрешаване на домейн имена
- **FTP servers** - услуги за пренос на файлове
- **VPN concentrators** - крайни точки за отдалечен достъп

**DMZ Security Principles:**
- **Minimal trust** - DMZ сървърите имат ограничен LAN достъп
- **Service isolation** - всяка услуга на отделен DMZ сегмент
- **Regular patching** - поддържане на актуални security обновления
- **Monitoring и logging** - цялостно проследяване на активността
- **Backup strategies** - отделна backup инфраструктура

### **ЧАС 3: FIREWALL TYPES И ТЕХНОЛОГИИ (40 мин)**

#### **Теоретична част (25 мин):**

**Software vs Hardware Firewalls (8 мин)**

**Software Firewalls (Host-based):**

**Windows Defender Firewall:**
- **Built-in protection** на Windows операционни системи
- **Profile-based configuration** - Domain, Private, Public мрежи
- **Application control** - правила allow/deny за отделни програми
- **Advanced configuration** чрез Group Policy, PowerShell
- **Integration** с Windows security екосистема

*Configuration Example:*
```powershell
# Block specific application
New-NetFirewallRule -DisplayName "Block_Torrent" -Direction Outbound -Program "C:\Program Files\Torrent\utorrent.exe" -Action Block

# Allow application on specific port
New-NetFirewallRule -DisplayName "Allow_App_Port8080" -Direction Inbound -Protocol TCP -LocalPort 8080 -Action Allow
```

**Linux iptables:**
- **Kernel-level packet filtering** в Linux системи
- **Flexible rule structure** - INPUT, OUTPUT, FORWARD chains
- **Stateful connection tracking** с conntrack модул
- **Performance efficiency** - обработка в kernel пространство
- **Scriptable configuration** за автоматизирано разгръщане

*Configuration Example:*
```bash
# Block all incoming connections by default
iptables -P INPUT DROP

# Allow established connections
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# Allow SSH from specific subnet
iptables -A INPUT -s 192.168.1.0/24 -p tcp --dport 22 -j ACCEPT
```

**Third-Party Software Firewalls:**
- **ZoneAlarm, Comodo** - решения, насочени към потребители
- **Symantec Endpoint Protection** - enterprise endpoint security
- **Application-aware filtering** - прецизен контрол на мрежовия достъп на програмите
- **User-friendly interfaces** за нетехнически потребители

**Hardware Firewalls (Network-based):**

**Dedicated Firewall Appliances:**
- **Cisco ASA Series** - enterprise-grade security appliances
- **Fortinet FortiGate** - unified threat management
- **Palo Alto Networks** - next-generation firewall лидери
- **SonicWall** - решения, насочени към средния пазар
- **pfSense** - open-source firewall платформа

**Advantages на Hardware Firewalls:**
- **Centralized protection** за цялата мрежа
- **High throughput** - специализиран хардуер за обработка на пакети
- **Network-level visibility** - цялостен анализ на трафика
- **Independent operation** - не зависи от security на host OS

**Disadvantages:**
- **Single point of failure** - влияние върху цялата мрежа при отказ
- **Higher cost** - специализиран хардуер и лицензи
- **Configuration complexity** - изисква мрежова експертиза
- **Limited host-specific control** - не може да филтрира поведението на приложенията

**Next-Generation Firewall Features (9 мин)**

**Deep Packet Inspection (DPI):**
- **Content analysis** - разглежда payload на пакета, не само headers
- **Protocol reconstruction** - разбира комуникациите на application layer
- **Malware detection** - идентифицира злонамерено съдържание в трафика
- **Data loss prevention** - предотвратява ексфилтрация на чувствителни данни

**Application Awareness:**
- **Application identification** - разпознава приложения независимо от порта/протокола
- **Application control** - позволява/блокира конкретни приложения
- **Bandwidth management** - приоритизира критичните бизнес приложения
- **User activity correlation** - свързва мрежовата активност с конкретни потребители

*Пример: Блокиране на Facebook трафика при позволяване на LinkedIn, независимо от използваните портове*

**Intrusion Prevention System (IPS) Integration:**
- **Signature-based detection** - познати attack patterns
- **Anomaly-based detection** - необичайно поведение на трафика
- **Real-time blocking** - незабавен отговор на заплахи
- **Automated updates** - непрекъснато обновяване на threat signatures

**SSL/TLS Inspection:**
- **Certificate analysis** - проверка на автентичността на SSL сертификати
- **Encrypted traffic decryption** - инспектиране на HTTPS съдържание
- **Man-in-the-middle inspection** - контролирано декриптиране за security анализ
- **Privacy considerations** - баланс между security и поверителност на потребителите

**Threat Intelligence Integration:**
- **Global threat feeds** - списъци с злонамерени IP/домейни в реално време
- **Reputation scoring** - оценка на нивото на риска на дестинациите
- **Automated blocking** - незабавен отговор на известни заплахи
- **Contextual analysis** - комбиниране на множество threat indicators

### **ЧАС 4: ADVANCED SEGMENTATION И MICRO-SEGMENTATION (40 мин)**

#### **Теоретична част (24 мин):**

**Micro-Segmentation Concept (8 мин)**

**Traditional Network Security Model:**
- **Perimeter-based** - силен external firewall, доверена вътрешна мрежа
- **Castle-and-moat approach** - твърда обвивка, мека вътрешност
- **Limitations:** Lateral movement при пробив, вътрешни заплахи

**Micro-Segmentation Approach:**
- **Granular security controls** - сегментиране на мрежата на много малки зони
- **Workload-level protection** - security на ниво отделно приложение/сървър
- **Zero-trust principle** - никога не се доверявай, винаги проверявай

### **ЧАС 5: FIREWALL CONFIGURATION И MANAGEMENT (40 мин)**

#### **Теоретична част (20 мин):**

**Firewall Rule Management Best Practices (10 мин)**

**Rule Creation Principles:**

**Principle of Least Privilege:**
- **Default deny** - блокиране на целия трафик по подразбиране
- **Explicit allow** - позволяване само на необходимите комуникации
- **Specific sources** - избягване на широки "any" source адреси
- **Limited time windows** - временни правила с автоматично изтичане
- **Regular review** - периодична оценка на необходимостта от правила

**Rule Organization Strategies:**
- **Logical grouping** - организиране на правилата по функция/отдел
- **Rule documentation** - ясно описание и обосновка за всяко правило
- **Naming conventions** - последователни, описателни имена на правила
- **Priority ordering** - най-специфичните правила първи, общите последни
- **Change management** - контролиран процес за модифициране на правила

**Common Configuration Mistakes:**
- **Any-Any rules** - прекалено разрешителен достъп (Source: Any, Destination: Any, Service: Any)
- **Unused rules** - стари правила, непремахнати, създаващи безпорядък
- **Overlapping rules** - конфликтни политики, причиняващи непредсказуемо поведение
- **Insufficient logging** - липсваща одитна следа за security събития
- **Default configurations** - непромяна на настройките по подразбиране на производителя

**Firewall Rule Analysis Framework (10 мин)**

**Rule Effectiveness Criteria:**

**Security Effectiveness:**
- **Threat coverage** - правилата адресират релевантни attack vectors
- **False positive rate** - легитимният трафик не се блокира ненужно
- **Response time** - колко бързо правилата засичат и отговарят на заплахи
- **Compliance alignment** - правилата подкрепят регулаторните изисквания

**Operational Effectiveness:**
- **Performance impact** - обработката на правилата не влошава скоростта на мрежата
- **Management complexity** - правилата са поддържаеми и разбираеми
- **Change responsiveness** - правилата могат да се адаптират към бизнес промени
- **Documentation quality** - ясно обяснение и обосновка

**Rule Optimization Techniques:**
- **Rule consolidation** - обединяване на подобни правила за намаляване на сложността
- **Object groups** - създаване на многократно използваеми групи за общи адреси/услуги
- **Time-based rules** - ограничаване на достъпа на базата на времеви прозорци
- **Bandwidth limiting** - контрол на потреблението на ресурси за конкретен трафик
- **Hit count analysis** - идентифициране на неизползвани или рядко използвани правила
