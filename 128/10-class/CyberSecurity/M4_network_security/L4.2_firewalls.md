### **ЧАС 1: FIREWALL FUNDAMENTALS И АРХИТЕКТУРИ (40 мин)**

#### **Теоретична част (20 мин):**

**🔥 Какво е Firewall? (5 мин)**

**Definition и Purpose:**
Firewall е network security device или software който контролира incoming и outgoing network traffic based на predetermined security rules. Действа като barrier between trusted internal network и untrusted external networks (като Internet).

**Primary Functions:**
- **Traffic filtering** - позволява или блокира traffic based на rules
- **Access control** - определя кой може да access specific network resources  
- **Logging и monitoring** - записва network activities за analysis
- **NAT (Network Address Translation)** - скрива internal network structure

**🏗️ Firewall Архитектури (8 мин)**

**Packet Filtering Firewalls (Stateless):**
- **Механизъм:** Examine individual packets независимо от connection context
- **Decision factors:** Source/destination IP, port numbers, protocol type
- **Advantages:** Fast processing, low resource usage
- **Disadvantages:** No connection state awareness, vulnerable to sophisticated attacks
- **Use cases:** Basic perimeter filtering, high-throughput environments

*Пример: Router ACL правило блокиращо всички TCP connections към port 23 (Telnet)*

**Stateful Inspection Firewalls:**
- **Механизъм:** Track connection state и enforce rules based на connection context
- **State tracking:** TCP handshakes, established connections, session information
- **Dynamic rule application:** automatically allowing return traffic за established connections
- **Security improvement:** prevents TCP sequence prediction attacks
- **Performance:** moderate resource usage, excellent security/performance balance

*Пример: Firewall позволява outbound HTTPS connection и automatically allows corresponding inbound response traffic*

**Application Layer Firewalls (Proxy Firewalls):**
- **Deep packet inspection:** analyze application-specific protocols (HTTP, FTP, SMTP)
- **Content filtering:** examine actual data content, not just headers
- **Protocol validation:** ensure traffic conforms to protocol standards
- **Advanced threat detection:** identify application-layer attacks
- **Resource intensive:** significant CPU и memory requirements

*Пример: Web Application Firewall (WAF) detecting SQL injection attempts in HTTP requests*

**Next-Generation Firewalls (NGFW):**
- **Integrated security features:** traditional firewall + IPS + application awareness
- **Application identification:** recognize applications regardless of port/protocol
- **User identity integration:** link network activity to specific users
- **Threat intelligence:** real-time updates за known malicious IPs/domains
- **SSL inspection:** decrypt и analyze encrypted traffic

*Пример: NGFW detecting cryptocurrency mining traffic disguised as HTTPS communication*

**🏆 Evolution Summary (2 мин)**
*1980s:* Packet filtering → *1990s:* Stateful inspection → *2000s:* Application awareness → *2010s+:* Integrated threat prevention

### **ЧАС 2: NETWORK SEGMENTATION СТРАТЕГИИ (40 мин)**

#### **Теоретична част (22 мин):**

**🌐 Network Segmentation Principles (8 мин)**

**Fundamental Concept:**
Network segmentation е practice на dividing network в smaller, isolated sub-networks за improving security, performance, и compliance.

**Core Benefits:**
- **Lateral movement prevention** - attackers не могат да се разпространят свободно
- **Improved performance** - reduced broadcast domains, optimized traffic flow
- **Enhanced monitoring** - easier traffic analysis и anomaly detection
- **Compliance isolation** - separate regulated data от general network resources
- **Incident containment** - security breaches остават localized

**Segmentation Approaches:**

**Physical Segmentation:**
- **Separate hardware** за different network zones
- **Dedicated switches/routers** за each security domain
- **Air-gapped networks** за highest security requirements
- **Advantages:** Complete isolation, no shared vulnerabilities
- **Disadvantages:** High cost, management complexity

**Logical Segmentation (VLANs):**
- **Virtual LANs** на same physical infrastructure
- **IEEE 802.1Q standard** за VLAN tagging
- **Software-defined boundaries** между network segments
- **Cost-effective** solution за most organizations
- **Flexible reconfiguration** without physical changes

**📊 VLAN Security Implementation (7 мин)**

**VLAN Security Benefits:**
- **Broadcast domain isolation** - limits broadcast storm impact
- **Access control** - restrict communication between VLANs
- **Traffic monitoring** - easier analysis на specific network segments
- **Policy enforcement** - different security policies за different VLANs

**VLAN Security Vulnerabilities:**

**VLAN Hopping Attacks:**
- **Switch spoofing** - attacker mimics switch behavior to access multiple VLANs
- **Double tagging** - exploitation на native VLAN weaknesses
- **Prevention:** disable unused ports, configure port security, avoid default VLANs

**VLAN Configuration Best Practices:**
- **Change default VLAN** (VLAN 1) за native traffic
- **Disable unused ports** и assign to dummy VLAN
- **Configure port security** - MAC address limiting
- **Implement VLAN ACLs** за inter-VLAN communication control
- **Use 802.1X** за dynamic VLAN assignment based на user authentication

**🏰 DMZ Design и Perimeter Security (7 мин)**

**DMZ (Demilitarized Zone) Concept:**
DMZ е neutral zone между internal trusted network и external untrusted network (Internet). Служи като buffer zone където се поставят publicly accessible services.

**DMZ Architecture Components:**

**Three-Legged Firewall Design:**
- **External interface** - connects to Internet
- **Internal interface** - connects to LAN
- **DMZ interface** - hosts public servers (web, email, DNS)
- **Traffic control** - strict rules между all zones

**Dual Firewall Design:**
- **Perimeter firewall** - Internet-facing protection
- **Internal firewall** - additional protection за LAN
- **DMZ между firewalls** - isolated server zone
- **Defense-in-depth** - multiple security layers

**Common DMZ Services:**
- **Web servers** - public website hosting
- **Email servers** - SMTP/POP3/IMAP services
- **DNS servers** - public domain name resolution
- **FTP servers** - file transfer services
- **VPN concentrators** - remote access endpoints

**DMZ Security Principles:**
- **Minimal trust** - DMZ servers have limited LAN access
- **Service isolation** - each service на separate DMZ segment
- **Regular patching** - maintain current security updates
- **Monitoring и logging** - comprehensive activity tracking
- **Backup strategies** - separate backup infrastructure

### **ЧАС 3: FIREWALL TYPES И ТЕХНОЛОГИИ (40 мин)**

#### **Теоретична част (25 мин):**

**🔧 Software vs Hardware Firewalls (8 мин)**

**Software Firewalls (Host-based):**

**Windows Defender Firewall:**
- **Built-in protection** на Windows operating systems
- **Profile-based configuration** - Domain, Private, Public networks
- **Application control** - per-program allow/deny rules
- **Advanced configuration** чрез Group Policy, PowerShell
- **Integration** с Windows security ecosystem

*Configuration Example:*
```powershell
# Block specific application
New-NetFirewallRule -DisplayName "Block_Torrent" -Direction Outbound -Program "C:\Program Files\Torrent\utorrent.exe" -Action Block

# Allow application on specific port
New-NetFirewallRule -DisplayName "Allow_App_Port8080" -Direction Inbound -Protocol TCP -LocalPort 8080 -Action Allow
```

**Linux iptables:**
- **Kernel-level packet filtering** в Linux systems
- **Flexible rule structure** - INPUT, OUTPUT, FORWARD chains
- **Stateful connection tracking** с conntrack module
- **Performance efficiency** - kernel-space processing
- **Scriptable configuration** за automated deployment

*Configuration Example:*
```bash
# Block all incoming connections by default
iptables -P INPUT DROP

# Allow established connections
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# Allow SSH from specific subnet
iptables -A INPUT -s 192.168.1.0/24 -p tcp --dport 22 -j ACCEPT
```

**Third-Party Software Firewalls:**
- **ZoneAlarm, Comodo** - consumer-focused solutions
- **Symantec Endpoint Protection** - enterprise endpoint security
- **Application-aware filtering** - granular control over program network access
- **User-friendly interfaces** за non-technical users

**Hardware Firewalls (Network-based):**

**Dedicated Firewall Appliances:**
- **Cisco ASA Series** - enterprise-grade security appliances
- **Fortinet FortiGate** - unified threat management
- **Palo Alto Networks** - next-generation firewall leaders
- **SonicWall** - mid-market focused solutions
- **pfSense** - open-source firewall platform

**Advantages на Hardware Firewalls:**
- **Centralized protection** за entire network
- **High throughput** - specialized hardware за packet processing
- **Network-level visibility** - comprehensive traffic analysis
- **Independent operation** - не depends на host OS security

**Disadvantages:**
- **Single point of failure** - network-wide impact при failure
- **Higher cost** - specialized hardware и licensing
- **Configuration complexity** - requires network expertise
- **Limited host-specific control** - cannot filter application behavior

**🛡️ Next-Generation Firewall Features (9 мин)**

**Deep Packet Inspection (DPI):**
- **Content analysis** - examine packet payload, не само headers
- **Protocol reconstruction** - understand application layer communications
- **Malware detection** - identify malicious content в traffic
- **Data loss prevention** - prevent sensitive data exfiltration

**Application Awareness:**
- **Application identification** - recognize apps regardless на port/protocol
- **Application control** - allow/block specific applications
- **Bandwidth management** - prioritize critical business applications
- **User activity correlation** - link network activity to specific users

*Example: Blocking Facebook traffic while allowing LinkedIn, regardless of ports used*

**Intrusion Prevention System (IPS) Integration:**
- **Signature-based detection** - known attack patterns
- **Anomaly-based detection** - unusual traffic behaviors
- **Real-time blocking** - immediate threat response
- **Automated updates** - continuously updated threat signatures

**SSL/TLS Inspection:**
- **Certificate analysis** - verify SSL certificate authenticity
- **Encrypted traffic decryption** - inspect HTTPS content
- **Man-in-the-middle inspection** - controlled decryption за security analysis
- **Privacy considerations** - balance security и user privacy

**Threat Intelligence Integration:**
- **Global threat feeds** - real-time malicious IP/domain lists
- **Reputation scoring** - assess risk level на destinations
- **Automated blocking** - immediate response to known threats
- **Contextual analysis** - combine multiple threat indicators

**🔬 Micro-Segmentation Concept (8 мин)**

**Traditional Network Security Model:**
- **Perimeter-based** - strong external firewall, trusted internal network
- **Castle-and-moat approach** - hard exterior, soft interior
- **Limitations:** lateral movement upon breach, insider threats

**Micro-Segmentation Approach:**
- **Granular security controls** - segment network в many small zones
- **Workload-level protection** - individual application/server security
- **Zero-trust principle** - never trust, always verify
- **Software-defined networking** - programmatic rule enforcement

**Implementation Technologies:**

**Software-Defined Perimeters (SDP):**
- **Identity-based access** - authenticate before network access
- **Encrypted tunnels** - secure communications между endpoints
- **Least privilege access** - minimum required network permissions
- **Dynamic provisioning** - real-time access control adjustments

**Network Access Control (NAC):**
- **Device identification** - recognize и categorize connecting devices
- **Compliance checking** - verify security policy adherence
- **Automated quarantine** - isolate non-compliant devices
- **Guest network isolation** - separate visitor access

**🏢 Enterprise Segmentation Strategies (8 мин)**

**Business Function Segmentation:**
- **Department-based isolation** - HR, Finance, Engineering separate zones
- **Project-based segmentation** - isolate sensitive project resources
- **Contractor access control** - temporary, limited network access

**Data Classification Segmentation:**
- **Public data zone** - freely accessible information
- **Internal data zone** - employee-only resources
- **Confidential data zone** - restricted access, logging required

**Compliance-Driven Segmentation:**
- **PCI DSS zone** - payment card data isolation
- **HIPAA zone** - healthcare information protection

**Zero Trust Principles:**

- **Assume breach** - design под assumption that perimeter is compromised

**Zero Trust Implementation Components:**

### **ЧАС 5: FIREWALL CONFIGURATION И MANAGEMENT (40 мин)**

#### **Теоретична част (20 мин):**

**⚙️ Firewall Rule Management Best Practices (10 мин)**

**Rule Creation Principles:**

**Principle of Least Privilege:**
- **Default deny** - block all traffic by default
- **Explicit allow** - permit only necessary communications
- **Specific sources** - avoid broad "any" source addresses
- **Limited time windows** - temporary rules с automatic expiration
- **Regular review** - periodic assessment на rule necessity

**Rule Organization Strategies:**
- **Logical grouping** - organize rules by function/department
- **Rule documentation** - clear description и justification за каждо rule
- **Naming conventions** - consistent, descriptive rule names
- **Priority ordering** - most specific rules first, general rules last
- **Change management** - controlled rule modification process

**Common Configuration Mistakes:**
- **Any-Any rules** - overly permissive access (Source: Any, Destination: Any, Service: Any)
- **Unused rules** - old rules не removed, creating clutter
- **Overlapping rules** - conflicting policies causing unpredictable behavior
- **Insufficient logging** - missing audit trail за security events
- **Default configurations** - не changing vendor default settings

**🔍 Firewall Rule Analysis Framework (10 мин)**

**Rule Effectiveness Criteria:**

**Security Effectiveness:**
- **Threat coverage** - rules address relevant attack vectors
- **False positive rate** - legitimate traffic не blocked unnecessarily
- **Response time** - how quickly rules detect и respond to threats
- **Compliance alignment** - rules support regulatory requirements

**Operational Effectiveness:**
- **Performance impact** - rule processing не degrading network speed
- **Management complexity** - rules are maintainable и understandable
- **Change responsiveness** - rules can adapt to business changes
- **Documentation quality** - clear explanation и justification

**Rule Optimization Techniques:**
- **Rule consolidation** - combine similar rules to reduce complexity
- **Object groups** - create reusable groups за common addresses/services
- **Time-based rules** - restrict access based на time windows
- **Bandwidth limiting** - control resource consumption za specific traffic
- **Hit count analysis** - identify unused или rarely used rules
