## Учебно съдържание (40 минути)

### 1. Какво е Wireshark и защо го ползваме (5 мин)

**Wireshark** е безплатен, open source network protocol analyzer (packet sniffer). Позволява улавяне и анализ на мрежов трафик в реално време или от записани файлове (.pcap / .pcapng).

**Защо го ползваме:**
- Диагностика на мрежови проблеми
- Анализ на мрежова сигурност и засичане на атаки
- Обучение — виждаме как протоколите работят на практика
- Forensics — анализ на инциденти

**Официален сайт:** https://www.wireshark.org
**Лиценз:** GNU GPL (open source, безплатен)
**Поддържани платформи:** Windows, Linux, macOS

> Wireshark е стандартен инструмент за мрежови инженери, security анализатори и penetration testers.

---

### 2. Инсталация и първи стартиране (5 мин)

**Инсталация на Windows:**
1. Отвори https://www.wireshark.org/download.html
2. Свали „Windows Installer (64-bit)"
3. Стартирай инсталатора — остави настройките по подразбиране
4. При питане за **Npcap** — задължително инсталирай (необходим за packet capture)
5. Рестартирай, ако се изисква

**Interface overview след стартиране:**
- **Capture interfaces list** — списък с мрежови интерфейси и активността им
- **Menu bar** — File, Capture, Analyze, Statistics...
- **Display filter bar** — въвеждане на филтри
- **Packet list pane** — списък с уловените пакети
- **Packet details pane** — разгънати слоеве на избрания пакет
- **Packet bytes pane** — raw данните в hex / ASCII

---

### 3. Избор на capture interface и стартиране на запис (5 мин)

**Стъпки:**
1. Стартирай Wireshark
2. На началния екран виж списъка с interfaces — активните показват вълнообразна линия (активен трафик)
3. Избери интерфейса, свързан с мрежата (обикновено `Ethernet` или `Wi-Fi`)
4. Двойно кликване върху интерфейса стартира capture
5. За спиране: бутон **Stop** (червен квадрат) или `Ctrl+E`
6. За запис в файл: File → Save As → формат `.pcapng`

**Забележка за Linux:** Wireshark може да изисква root права или добавяне на потребителя в групата `wireshark`:
```bash
sudo usermod -aG wireshark $USER
```

---

### 4. Основни display filters (10 мин)

Display filters филтрират показаните пакети без да спират capture. Въвеждат се в лентата под toolbar-а. При правилен синтаксис — зелен фон; при грешен — червен.

| Filter | Описание |
|--------|----------|
| `http` | Показва само HTTP трафик |
| `tcp` | Показва само TCP пакети |
| `udp` | Показва само UDP пакети |
| `dns` | Показва само DNS заявки и отговори |
| `ip.addr == 192.168.1.1` | Трафик от или към конкретен IP адрес |
| `ip.src == 192.168.1.1` | Само трафик от конкретен IP |
| `ip.dst == 192.168.1.1` | Само трафик към конкретен IP |
| `tcp.port == 80` | TCP трафик на порт 80 (HTTP) |
| `tcp.port == 443` | TCP трафик на порт 443 (HTTPS) |
| `not arp` | Скрива ARP пакети |

**Комбиниране на филтри:**
```
http and ip.addr == 192.168.1.5
tcp.port == 80 or tcp.port == 443
dns and ip.src == 192.168.1.1
```

**Follow TCP Stream:** Десен клик върху TCP пакет → Follow → TCP Stream — показва цялата комуникация като текст.

### 5. Четене на packet details (10 мин)

При кликване върху пакет в Packet list pane, Packet details pane показва разгънатата структура по слоеве:

**Слой 2 — Ethernet Frame:**
```
Ethernet II
  Destination: aa:bb:cc:dd:ee:ff  (MAC адрес на получател)
  Source:      11:22:33:44:55:66  (MAC адрес на изпращач)
  Type: IPv4 (0x0800)
```

**Слой 3 — IP Header:**
```
Internet Protocol Version 4
  Source Address: 192.168.1.5
  Destination Address: 142.250.74.46
  Protocol: TCP (6)
  Time to Live: 64
```

**Слой 4 — TCP Segment:**
```
Transmission Control Protocol
  Source Port: 54321
  Destination Port: 80
  Sequence Number: 1
  Flags: SYN, ACK
```

**Payload / Application Layer:**
```
Hypertext Transfer Protocol
  GET /index.html HTTP/1.1\r\n
  Host: example.com\r\n
```

> Кликни на стрелката вляво от всеки слой за разгъване/сгъване на детайлите.

### 6. Практическо упражнение: Capture на собствен трафик и намиране на DNS заявка (5 мин)

**Задача:** Улови DNS заявка от собствения си компютър и прочети нейното съдържание.

**Стъпки:**
1. Стартирай Wireshark и избери активния мрежови интерфейс
2. Стартирай capture
3. Отвори Command Prompt / Terminal и изпълни:
   ```
   nslookup google.com
   ```
   или отвори браузъра и посети произволен сайт
4. Спри capture след 10-15 секунди
5. Въведи display filter: `dns`
6. Намери DNS Query пакет (Type: A, Name: google.com)
7. Намери съответния DNS Response пакет
8. В Packet details разгъни секцията „Domain Name System" и прочети:
   - Queried name
   - Returned IP address(es)

**Очакван резултат:** Виждаш DNS заявката за google.com и отговора с IP адреса на Google.

## Методически указания

**Организация на часа:**
- Урокът е уводен и практически — всеки ученик работи на собствен компютър
- Учителят демонстрира всяка стъпка на проектора преди учениците да я изпълнят
- При технически затруднения (напр. Npcap не е инсталиран) — работа по двойки

**Диференциация:**
- По-бързите ученици могат да опитат допълнителни филтри или Follow TCP Stream
- По-бавните ученици се насочват към основната задача — само DNS capture

**Типични проблеми:**
- „No interfaces found" — Npcap не е инсталиран или липсват права
- Capture показва само собствен трафик (switch среда) — нормално, не е грешка
- HTTPS трафикът е криптиран и payload-ът не се чете — обяснява защо HTTPS е важен

**Домашна работа / подготовка за следващи уроци:**
- Разгледай `resources/Wireshark/workshop-part-01` у дома
- Опитай се да уловиш трафик от домашната си мрежа и да намериш HTTP заявка
