### **ЧАС 1: INTRUSION DETECTION SYSTEMS (IDS) FUNDAMENTALS (40 мин)**

#### **Теоретична част (22 мин):**

**IDS Concept и Architecture (8 мин)**

**Definition и Purpose:**
Intrusion Detection System (IDS) е security технология, която мониторира мрежовия трафик или системните дейности за злонамерени активности или нарушения на политики и генерира alerts когато такива дейности бъдат засечени.

**Core Functions:**
- **Traffic monitoring** - непрекъснат анализ на мрежовите комуникации
- **Pattern recognition** - идентифициране на познати attack signatures
- **Anomaly detection** - засичане на необичайни поведения, отклоняващи се от нормалните baselines
- **Alert generation** - уведомяване на security екипи за потенциални заплахи
- **Forensic logging** - поддържане на детайлни записи за разследване на инциденти

**IDS vs Firewall Differences:**
- **Firewall:** *Превентивен* - блокира трафика на базата на предварително дефинирани правила
- **IDS:** *Детективен* - мониторира и предупреждава, но не блокира трафика автоматично
- **Complement each other** - многопластов security подход
- **Different placement** - firewalls на периметъра, IDS в цялата мрежа

**IDS Deployment Architectures (7 мин)**

**Network-Based IDS (NIDS):**
- **Network segment monitoring** - анализ на трафика в конкретни мрежови сегменти
- **Strategic placement** - border routers, критични подмрежи, server farms
- **Promiscuous mode** - прихващане на целия мрежов трафик в сегмента
- **Scalability challenges** - високоскоростните мрежи изискват мощен хардуер
- **Coverage advantages** - мониторира всички устройства в мрежовия сегмент

*Placement Strategy:*
```
Internet → Firewall → [IDS Sensor] → Internal Network
                ↓
            [IDS Sensor] → DMZ Servers
                ↓
         [IDS Sensor] → Critical Server Farm
```

**Host-Based IDS (HIDS):**
- **Individual system monitoring** - анализ на дейностите на конкретни хостове
- **Local log analysis** - системни логове, логове на приложения, промени в файлове
- **Resource consumption** - използва CPU и памет на хоста за мониторинг
- **Detailed visibility** - вижда всички дейности на конкретната система
- **Agent-based deployment** - software, инсталиран на всеки наблюдаван хост

**Hybrid IDS Deployment:**
- **Combined approach** - мрежови сензори + хост агенти
- **Comprehensive coverage** - мониторинг на мрежово и хост ниво
- **Correlation capabilities** - свързване на мрежови събития с хост активности
- **Management complexity** - изисква координация между различни системи
- **Cost considerations** - множество технологии и лицензи

**Cloud и Modern IDS Architectures (7 мин)**

**Cloud-Based IDS:**
- **Virtual sensor deployment** - IDS в cloud среди
- **Auto-scaling capabilities** - динамично разпределяне на ресурси на базата на обема на трафика
- **Multi-tenant architecture** - споделена инфраструктура с изолиран мониторинг
- **API integration** - програмно управление и автоматизация
- **Global threat intelligence** - актуализации на signatures от cloud

**Software-Defined IDS:**
- **Programmable detection** - персонализирана логика за специфични среди
- **Container monitoring** - Docker и Kubernetes security
- **Microservices architecture** - разпределени IDS компоненти
- **DevSecOps integration** - автоматизирано security тестване в CI/CD pipelines

### **ЧАС 2: INTRUSION PREVENTION SYSTEMS (IPS) И ACTIVE RESPONSE (40 мин)**

#### **Теоретична част (25 мин):**

**IPS vs IDS Key Differences (8 мин)**

**IDS Characteristics (Passive Monitoring):**
- **Detection only** - идентифицира заплахи, но не ги блокира
- **Out-of-band deployment** - получава копия на трафика чрез SPAN портове
- **No traffic impact** - мрежовата производителност не се засяга от обработката на IDS
- **Forensic analysis** - подробно логване за разследване на инциденти
- **Alert-based response** - човешки аналитици разследват и реагират на alerts

**IPS Characteristics (Active Prevention):**
- **Real-time blocking** - автоматично спира засечените атаки
- **In-line deployment** - целият трафик преминава през IPS сензорите
- **Performance impact** - обработката на трафика може да засегне мрежовата латентност
- **Immediate response** - не се изисква човешка намеса за блокиране на заплахи
- **Prevention-focused** - спира атаките вместо само да ги докладва

**IPS Detection Technologies (8 мин)**

**Signature-Based Detection:**
- **Known attack patterns** - база данни с злонамерени traffic signatures
- **Pattern matching** - сравняване на трафика с базата от signatures
- **Low false positives** - висока точност за познати заплахи
- **Signature updates** - редовно изтегляне на нови attack patterns
- **Zero-day limitations** - не може да засече напълно нови атаки

*Example Signature:*
```
alert tcp any any -> any 80 (msg:"Possible SQL Injection"; content:"UNION SELECT"; sid:100001;)
```

**Anomaly-Based Detection:**
- **Baseline establishment** - научаване на нормалните traffic patterns
- **Statistical analysis** - идентифициране на отклонения от нормалното поведение
- **Machine learning** - разширени алгоритми за разпознаване на patterns
- **Zero-day detection** - може да идентифицира напълно нови атаки
- **Higher false positives** - легитимната необичайна активност предизвиква alerts

*Example Anomaly:*
- Normal: Потребителят обикновено изтегля 50MB/ден
- Anomaly: Същият потребител изтегля 5GB за 30 минути = потенциална ексфилтрация на данни

**Behavioral Analysis:**
- **User behavior profiling** - разбиране на индивидуалните потребителски patterns
- **Entity behavior analytics** - анализ на поведението на устройства и услуги
- **Contextual analysis** - отчитане на времето, местоположението, ролята при засичането
- **Adaptive learning** - непрекъснато обновяване на моделите за поведение
- **Advanced persistent threat detection** - идентифициране на дългосрочен компромис

**IPS Implementation Modes (9 мин)**

**In-Line Mode (Bridge Mode):**
- **Traffic flow:** Всички пакети преминават през IPS сензора
- **Performance impact:** Обработката на латентността засяга мрежовата скорост
- **Fail-open vs Fail-closed:**
  - **Fail-open:** Трафикът продължава ако IPS откаже (приоритет наличност)
  - **Fail-closed:** Трафикът спира ако IPS откаже (приоритет security)
- **High availability:** Резервно IPS разгръщане за критични мрежи

**Hybrid Mode:**
- **Critical traffic** - in-line защита за високостойностни активи
- **General traffic** - пасивен мониторинг за по-широката мрежа
- **Resource optimization** - баланс между security и производителност
- **Flexible deployment** - адаптиране към конкретни мрежови изисквания

**IPS Response Actions:**
- **Drop packets** - тихо изхвърляне на злонамерен трафик
- **Reset connections** - активно прекратяване на подозрителни сесии
- **Block source IP** - временно blacklisting на атакуващи хостове
- **Rate limiting** - намаляване на обема на трафика от подозрителни източници
- **Quarantine systems** - изолиране на потенциално компрометирани хостове

**Advanced IPS Features:**
- **SSL/TLS inspection** - декриптиране и анализ на криптиран трафик
- **Application control** - блокиране на конкретни приложения независимо от порта
- **Reputation filtering** - блокиране на трафика от известни злонамерени IP/домейни
- **Geo-IP filtering** - ограничаване на достъпа на базата на географско местоположение
- **Protocol validation** - осигуряване, че трафикът отговаря на стандартите за протоколи

### **ЧАС 3: SIEM FUNDAMENTALS И CENTRALIZED MONITORING (40 мин)**

#### **Теоретична част (24 мин):**

**SIEM Concept и Architecture (8 мин)**

**Security Information и Event Management (SIEM):**
SIEM платформата събира, анализира и корелира security данни от множество източници в организацията, за да осигури всеобхватна гледна точка за security позата и да позволи бърза реакция при инциденти.

**Core SIEM Functions:**
- **Log aggregation** - събиране на логове от firewalls, сървъри, приложения, IDS/IPS
- **Data normalization** - конвертиране на различни log формати в обща структура
- **Correlation analysis** - идентифициране на patterns в множество log източници
- **Real-time alerting** - незабавно уведомяване за security инциденти
- **Compliance reporting** - автоматизирани доклади за регулаторни изисквания

**SIEM Architecture Components:**

**Data Collection Layer:**
- **Log collectors** - агенти, syslog сървъри, API връзки
- **Supported sources** - firewalls, антивируси, domain controllers, бази данни
- **Data parsing** - извличане на релевантни полета от raw log записи
- **Secure transmission** - криптирано препращане на логове към SIEM платформата

**Storage и Processing:**
- **Big data platforms** - Elasticsearch, Hadoop, cloud storage
- **Index optimization** - бързо търсене в големи набори от данни
- **Data retention** - политики за съхранение, ръководени от съответствие
- **Backup и recovery** - защита на исторически security данни

**Analysis Engine:**
- **Correlation rules** - логика за свързване на свързани събития
- **Statistical analysis** - сравнение с baseline и засичане на отклонения
- **Machine learning** - разширено разпознаване на patterns
- **Threat intelligence** - интеграция с external threat feeds

**Log Sources и Integration (8 мин)**

**Network Security Sources:**
- **Firewall logs** - позволени/отказани връзки, конфигурационни промени
- **IDS/IPS alerts** - засечени прониквания, блокирани атаки
- **Router/Switch logs** - дейности на мрежовите устройства, удостоверяване
- **VPN logs** - сесии за отдалечен достъп, опити за свързване
- **DNS logs** - заявки за разрешаване на домейни, потенциална C&C комуникация

**Endpoint Security Sources:**
- **Antivirus logs** - засичане на malware, действия при карантина
- **Host-based IDS** - засичане на прониквания на системно ниво
- **Application logs** - достъп до бази данни, дейности на web сървъри
- **Operating system logs** - опити за вход, ескалация на привилегии
- **File integrity monitoring** - неоторизирани модификации на файлове

**Business Application Sources:**
- **Email security** - spam филтриране, злонамерени прикачени файлове
- **Web filtering** - блокирани уебсайтове, нарушения на политики
- **Data loss prevention** - опити за пренос на чувствителни данни
- **Identity management** - събития за удостоверяване и оторизация
- **Cloud services** - дейности на SaaS приложения

**Common Log Formats:**
- **Syslog** - стандартен протокол за логване на мрежови устройства
- **Windows Event Logs** - системни и приложни Windows събития
- **JSON logs** - структурирано логване на приложения
- **CEF (Common Event Format)** - стандартизиран формат за security събития
- **LEEF (Log Event Extended Format)** - стандарт за IBM security события

**SIEM Analytics и Correlation (8 мин)**

**Event Correlation Techniques:**

**Time-Based Correlation:**
- **Sequence analysis** - събития в конкретен ред, показващи прогресия на атака
- **Time window matching** - събития в рамките на определен период
- **Frequency detection** - необичаен брой подобни събития

*Пример:* Неуспешен опит за вход → Успешен вход → Достъп до привилегиован акаунт = потенциален компромис на credentials

**Geographic Correlation:**
- **Impossible travel detection** - потребителски входове от географски отдалечени местоположения
- **Geographic risk scoring** - оценка на риска на базата на репутацията на местоположението
- **Timezone analysis** - необичайни patterns на достъп на базата на местоположението на потребителя

*Пример:* Потребителски вход от България в 9:00 сутринта, последван от вход от Китай в 9:15 сутринта = компромис на акаунт

**Asset-Based Correlation:**
- **Asset criticality** - претегляне на събитията на базата на важността на актива
- **Asset relationship** - разбиране на зависимостите между системите
- **Attack path analysis** - предсказване на вероятните следващи цели в атакуващата верига

*Пример:* Компромис на работна станция на разработчик → Достъп до хранилище с изходен код → Модификация на production deployment скриптове

**User Behavior Analytics (UBA):**
- **Baseline establishment** - нормални patterns на потребителска активност
- **Deviation detection** - идентифициране на необичайно потребителско поведение
- **Risk scoring** - присвояване на risk scores на потребителски дейности
- **Peer group analysis** - сравняване на потребителското поведение с подобни роли

**Machine Learning Applications:**
- **Unsupervised learning** - идентифициране на неизвестни досега attack patterns
- **Supervised learning** - подобряване на точността на засичане на известни заплахи
- **Deep learning** - анализ на сложни данни и взаимовръзки
- **Natural language processing** - анализ на съдържанието на логове и описания на заплахи

**User и Entity Behavior Analytics (UEBA):**

**Behavioral Modeling Components:**

**User Behavior Profiles:**
- **Login patterns** - типични времена, местоположения, честота на потребителски достъп

- **Data access patterns** - до кои файлове/бази данни обикновено достъпва потребителят
- **Network activity** - използване на честотна лента, connection patterns
- **Device patterns** - кои устройства обикновено използва потребителят

**Entity Behavior Profiles:**
- **Server communication patterns** - типични flows на мрежовия трафик
- **Service usage** - нормална активност на приложения и бази данни
- **Resource consumption** - CPU, памет, baseline използване на мрежата
- **Update patterns** - обновления на software, конфигурационни промени
- **Access patterns** - кои системи обикновено достъпват този обект

- **Risk score calculation** - математически модели за вероятността от заплаха

- **Context consideration** - бизнес събития, засягащи нормалното поведение
- **Continuous learning** - адаптиране на моделите на базата на нови поведенчески данни

**APT Characteristics:**
- **Long-term presence** - атакуващите поддържат достъп за месеци/години

- **Target-specific** - персонализирани атаки за конкретни организации
- **Multi-stage approach** - постепенна прогресия в жизнения цикъл на атаката
- **Human operators** - сложни опоненти с дълбоки познания

**APT Detection Challenges:**
- **Low и slow** - дейностите са разпределени в дълги периоди от време
- **Living off the land** - използване на легитимни инструменти за злонамерени цели
- **Encrypted communication** - C&C трафикът е скрит в нормален HTTPS
- **Credential reuse** - използване на откраднати легитимни credentials
- **Anti-forensics** - техники за скриване на доказателства за атаката

**UEBA APT Detection Capabilities:**

**Long-term Pattern Analysis:**

**Phase 1: Preparation**

- **Team contacts** - графици за дежурство, процедури за ескалация

- **Training programs** - редовни учения и развитие на умения

**Phase 2: Identification**
- **Alert analysis** - определяне дали инцидентът е реална заплаха
- **Scope assessment** - разбиране на обхвата на потенциалния компромис

- **Monitoring enhancement** - увеличено наблюдение на засегнатите системи
- **Functionality verification** - осигуряване, че системите работят правилно
- **User communication** - информиране на засегнатите потребители за разрешаването

- **Control enhancement** - укрепване на security мерките
