### **ЧАС 1: INTRUSION DETECTION SYSTEMS (IDS) FUNDAMENTALS (40 мин)**

#### **Теоретична част (22 мин):**

**🔍 IDS Concept и Architecture (8 мин)**

**Definition и Purpose:**
Intrusion Detection System (IDS) е security technology която monitors network traffic или system activities за malicious activities или policy violations и produces alerts когато такива activities са detected.

**Core Functions:**
- **Traffic monitoring** - continuous analysis на network communications
- **Pattern recognition** - identify known attack signatures
- **Anomaly detection** - spot unusual behaviors deviating от normal baselines
- **Alert generation** - notify security teams на potential threats
- **Forensic logging** - maintain detailed records за incident investigation

**IDS vs Firewall Differences:**
- **Firewall:** *Preventive* - blocks traffic based на predefined rules
- **IDS:** *Detective* - monitors и alerts, но не blocks traffic automatically
- **Complement each other** - layered security approach
- **Different placement** - firewalls at perimeter, IDS throughout network

**🏗️ IDS Deployment Architectures (7 мин)**

**Network-Based IDS (NIDS):**
- **Network segment monitoring** - analyze traffic on specific network segments
- **Strategic placement** - border routers, critical subnets, server farms
- **Promiscuous mode** - capture all network traffic на segment
- **Scalability challenges** - high-speed networks require powerful hardware
- **Coverage advantages** - monitors all devices на network segment

*Placement Strategy:*
```
Internet → Firewall → [IDS Sensor] → Internal Network
                ↓
            [IDS Sensor] → DMZ Servers
                ↓  
         [IDS Sensor] → Critical Server Farm
```

**Host-Based IDS (HIDS):**
- **Individual system monitoring** - analyze activities на specific hosts
- **Local log analysis** - system logs, application logs, file changes
- **Resource consumption** - uses host CPU и memory за monitoring
- **Detailed visibility** - sees all activities на specific system
- **Agent-based deployment** - software installed на each monitored host

**Hybrid IDS Deployment:**
- **Combined approach** - network sensors + host agents
- **Comprehensive coverage** - network-level и host-level monitoring
- **Correlation capabilities** - connect network events с host activities  
- **Management complexity** - requires coordination между different systems
- **Cost considerations** - multiple technologies и licenses

**☁️ Cloud и Modern IDS Architectures (7 мин)**

**Cloud-Based IDS:**
- **Virtual sensor deployment** - IDS в cloud environments
- **Auto-scaling capabilities** - dynamic resource allocation based на traffic volume
- **Multi-tenant architecture** - shared infrastructure със isolated monitoring
- **API integration** - programmatic management и automation
- **Global threat intelligence** - cloud-based signature updates

**Software-Defined IDS:**
- **Programmable detection** - custom logic за specific environments
- **Container monitoring** - Docker и Kubernetes security
- **Microservices architecture** - distributed IDS components
- **DevSecOps integration** - automated security testing в CI/CD pipelines

### **ЧАС 2: INTRUSION PREVENTION SYSTEMS (IPS) И ACTIVE RESPONSE (40 мин)**

#### **Теоретична част (25 мин):**

**⚡ IPS vs IDS Key Differences (8 мин)**

**IDS Characteristics (Passive Monitoring):**
- **Detection only** - identifies threats но не blocks them
- **Out-of-band deployment** - receives traffic copies via SPAN ports
- **No traffic impact** - network performance не affected by IDS processing
- **Forensic analysis** - detailed logging за incident investigation
- **Alert-based response** - human analysts investigate и respond to alerts

**IPS Characteristics (Active Prevention):**
- **Real-time blocking** - automatically stops detected attacks
- **In-line deployment** - all traffic passes through IPS sensors
- **Performance impact** - traffic processing can affect network latency
- **Immediate response** - no human intervention required за threat blocking
- **Prevention-focused** - stops attacks rather than just reporting them

**🛡️ IPS Detection Technologies (8 мин)**

**Signature-Based Detection:**
- **Known attack patterns** - database на malicious traffic signatures
- **Pattern matching** - compare traffic против signature database
- **Low false positives** - high accuracy за known threats
- **Signature updates** - regular downloads на new attack patterns
- **Zero-day limitations** - cannot detect completely new attacks

*Example Signature:*
```
alert tcp any any -> any 80 (msg:"Possible SQL Injection"; content:"UNION SELECT"; sid:100001;)
```

**Anomaly-Based Detection:**
- **Baseline establishment** - learn normal network traffic patterns
- **Statistical analysis** - identify deviations от normal behavior
- **Machine learning** - advanced algorithms за pattern recognition
- **Zero-day detection** - can identify completely new attacks
- **Higher false positives** - legitimate unusual activity triggers alerts

*Example Anomaly:*
- Normal: User typically downloads 50MB/day
- Anomaly: Same user downloading 5GB в 30 minutes = potential data exfiltration

**Behavioral Analysis:**
- **User behavior profiling** - understand individual user patterns
- **Entity behavior analytics** - analyze device и service behaviors
- **Contextual analysis** - consider time, location, role в detection
- **Adaptive learning** - continuously update behavior models
- **Advanced persistent threat detection** - identify long-term compromise

**🔧 IPS Implementation Modes (9 мин)**

**In-Line Mode (Bridge Mode):**
- **Traffic flow:** All packets pass through IPS sensor
- **Performance impact:** Processing latency affects network speed
- **Fail-open vs Fail-closed:**
  - **Fail-open:** Traffic continues если IPS fails (availability priority)
  - **Fail-closed:** Traffic stops если IPS fails (security priority)
- **High availability:** Redundant IPS deployment за critical networks

**Hybrid Mode:**
- **Critical traffic** - in-line protection за high-value assets
- **General traffic** - passive monitoring за broader network
- **Resource optimization** - balance security и performance
- **Flexible deployment** - adapt to specific network requirements

**IPS Response Actions:**
- **Drop packets** - silently discard malicious traffic
- **Reset connections** - actively terminate suspicious sessions
- **Block source IP** - temporarily blacklist attacking hosts
- **Rate limiting** - reduce traffic volume от suspicious sources
- **Quarantine systems** - isolate potentially compromised hosts

**Advanced IPS Features:**
- **SSL/TLS inspection** - decrypt и analyze encrypted traffic
- **Application control** - block specific applications regardless на port
- **Reputation filtering** - block traffic от known malicious IPs/domains
- **Geo-IP filtering** - restrict access based на geographic location
- **Protocol validation** - ensure traffic conforms to protocol standards

### **ЧАС 3: SIEM FUNDAMENTALS И CENTRALIZED MONITORING (40 мин)**

#### **Теоретична част (24 мин):**

**📊 SIEM Concept и Architecture (8 мин)**

**Security Information и Event Management (SIEM):**
SIEM platform събира, анализира, и correlates security data от multiple sources в organization за provide comprehensive view на security posture и enable rapid incident response.

**Core SIEM Functions:**
- **Log aggregation** - collect logs от firewalls, servers, applications, IDS/IPS
- **Data normalization** - convert different log formats to common structure  
- **Correlation analysis** - identify patterns across multiple log sources
- **Real-time alerting** - immediate notification на security incidents
- **Compliance reporting** - automated reports за regulatory requirements

**SIEM Architecture Components:**

**Data Collection Layer:**
- **Log collectors** - agents, syslog servers, API connections
- **Supported sources** - firewalls, antivirus, domain controllers, databases
- **Data parsing** - extract relevant fields от raw log entries
- **Secure transmission** - encrypted log forwarding to SIEM platform

**Storage и Processing:**
- **Big data platforms** - Elasticsearch, Hadoop, cloud storage
- **Index optimization** - fast searching across large datasets
- **Data retention** - compliance-driven storage policies
- **Backup и recovery** - protect historical security data

**Analysis Engine:**
- **Correlation rules** - logic за connecting related events
- **Statistical analysis** - baseline comparison и deviation detection
- **Machine learning** - advanced pattern recognition
- **Threat intelligence** - integration с external threat feeds

**🔗 Log Sources и Integration (8 мин)**

**Network Security Sources:**
- **Firewall logs** - allowed/denied connections, configuration changes
- **IDS/IPS alerts** - detected intrusions, blocked attacks
- **Router/Switch logs** - network device activities, authentication
- **VPN logs** - remote access sessions, connection attempts
- **DNS logs** - domain resolution requests, potential C&C communication

**Endpoint Security Sources:**
- **Antivirus logs** - malware detection, quarantine actions
- **Host-based IDS** - system-level intrusion detection
- **Application logs** - database access, web server activities
- **Operating system logs** - login attempts, privilege escalation
- **File integrity monitoring** - unauthorized file modifications

**Business Application Sources:**
- **Email security** - spam filtering, malicious attachments
- **Web filtering** - blocked websites, policy violations  
- **Data loss prevention** - sensitive data transfer attempts
- **Identity management** - authentication, authorization events
- **Cloud services** - SaaS application activities

**Common Log Formats:**
- **Syslog** - standard network device logging protocol
- **Windows Event Logs** - Windows system и application events
- **JSON logs** - structured application logging
- **CEF (Common Event Format)** - standardized security event format
- **LEEF (Log Event Extended Format)** - IBM security event standard

**📈 SIEM Analytics и Correlation (8 мин)**

**Event Correlation Techniques:**

**Time-Based Correlation:**
- **Sequence analysis** - events occurring в specific order indicate attack progression
- **Time window matching** - events happening within defined timeframe
- **Frequency detection** - unusual number на similar events

*Example:* Failed login attempt → Successful login → Privileged account access = potential credential compromise

**Geographic Correlation:**
- **Impossible travel detection** - user logins от geographically distant locations
- **Geographic risk scoring** - assess risk based на location reputation  
- **Timezone analysis** - unusual access patterns based на user location

*Example:* User login от България at 9:00 AM followed by login от Китай at 9:15 AM = account compromise

**Asset-Based Correlation:**
- **Asset criticality** - weight events based на asset importance
- **Asset relationship** - understand dependencies between systems
- **Attack path analysis** - predict likely next targets в attack chain

*Example:* Compromise на developer workstation → Access to source code repository → Modification на production deployment scripts

**User Behavior Analytics (UBA):**
- **Baseline establishment** - normal user activity patterns
- **Deviation detection** - identify unusual user behaviors
- **Risk scoring** - assign risk scores to user activities
- **Peer group analysis** - compare user behavior to similar roles

**Machine Learning Applications:**
- **Unsupervised learning** - identify previously unknown attack patterns
- **Supervised learning** - improve accuracy на known threat detection
- **Deep learning** - analyze complex data relationships
- **Natural language processing** - analyze log content и threat descriptions

### **ЧАС 4: NETWORK TRAFFIC ANALYSIS И MONITORING TOOLS (40 мин)**

#### **Теоретична част (20 мин):**

**📡 Network Monitoring Protocols (10 мин)**

**SNMP (Simple Network Management Protocol):**
- **Device management** - routers, switches, servers monitoring
- **Performance metrics** - bandwidth utilization, error rates, system health
- **Version differences:**
  - **SNMPv1/v2c:** Plain text communication, community strings
  - **SNMPv3:** Encrypted authentication, secure management
- **Security considerations** - change default community strings, use SNMPv3 когато possible

**NetFlow/sFlow:**
- **Flow-based monitoring** - analyze traffic flows rather than individual packets
- **Scalable solution** - suitable за high-speed networks
- **Traffic summarization** - source/destination, protocols, volume, timing
- **Bandwidth analysis** - identify top talkers, applications, usage patterns
- **Security applications** - detect DDoS, data exfiltration, command & control

*NetFlow Record Example:*
```
Source IP: 192.168.1.45 → Destination: 173.194.74.139
Protocol: TCP | Source Port: 58392 | Dest Port: 443  
Packets: 1,247 | Bytes: 1.2MB | Duration: 342 seconds
Start: 14:23:17 | End: 14:29:59 | TCP Flags: Normal HTTPS
```

**Packet Capture Analysis:**
- **Deep packet inspection** - analyze individual packet contents
- **Protocol reconstruction** - rebuild application-layer communications
- **Security forensics** - detailed evidence collection
- **Performance troubleshooting** - identify network bottlenecks и issues
- **Compliance monitoring** - verify policy adherence

**🔬 Traffic Analysis Techniques (10 мин)**

**Baseline Development:**
- **Normal traffic patterns** - establish typical network behavior
- **Business hour patterns** - understand legitimate activity cycles
- **Application usage** - identify approved business applications
- **User behavior** - normal employee network usage patterns
- **Seasonal variations** - account за business cycle changes

**Anomaly Detection Methods:**

**Volume-Based Analysis:**
- **Bandwidth spikes** - unusual traffic volume increases
- **Connection count** - abnormal number на simultaneous connections
- **Protocol distribution** - changes в traffic composition
- **Geographic distribution** - traffic от unexpected countries

**Pattern-Based Analysis:**
- **Communication patterns** - identify command & control traffic
- **Data exfiltration patterns** - large outbound transfers
- **Lateral movement** - internal network scanning behaviors
- **Time-based patterns** - activity podczas off-business hours

**Content-Based Analysis:**
- **DNS analysis** - identify malicious domains, DGA detection
- **HTTP/HTTPS analysis** - suspicious web requests, malware downloads
- **Email traffic** - phishing detection, malicious attachments
- **File transfer analysis** - unauthorized data movement

**Advanced Analytics:**
- **Machine learning models** - automated pattern recognition
- **Behavioral analytics** - user и entity behavior analysis  
- **Threat intelligence correlation** - match traffic против known IOCs
- **Predictive analysis** - anticipate potential attack progression

UEBA technology анализира patterns на user и entity behavior за identify potential security threats by detecting deviations от established baselines.

**Behavioral Modeling Components:**

**User Behavior Profiles:**
- **Login patterns** - typical times, locations, frequency на user access
- **Application usage** - which applications user typically accesses
- **Data access patterns** - what files/databases user normally accesses
- **Network activity** - bandwidth usage, connection patterns
- **Device patterns** - what devices user typically uses

**Entity Behavior Profiles:**
- **Server communication patterns** - typical network traffic flows
- **Service usage** - normal application и database activity
- **Resource consumption** - CPU, memory, network baseline usage
- **Update patterns** - software updates, configuration changes
- **Access patterns** - which systems typically access this entity

**Anomaly Scoring System:**
- **Risk score calculation** - mathematical models за threat likelihood
- **Threshold tuning** - adjust sensitivity за false positive reduction
- **Context consideration** - business events affecting normal behavior
- **Continuous learning** - adapt models based на new behavioral data

**🎯 Advanced Persistent Threat (APT) Detection (8 мин)**

**APT Characteristics:**
- **Long-term presence** - attackers maintain access за months/years
- **Stealth techniques** - avoid detection чрез mimicking normal activities
- **Target-specific** - customized attacks за specific organizations
- **Multi-stage approach** - gradual progression through attack lifecycle
- **Human operators** - sophisticated adversaries с deep knowledge

**APT Detection Challenges:**
- **Low и slow** - activities distributed over long timeframes
- **Living off the land** - using legitimate tools за malicious purposes
- **Encrypted communication** - C&C traffic hidden в normal HTTPS
- **Credential reuse** - using stolen legitimate credentials
- **Anti-forensics** - techniques за hiding attack evidence

**UEBA APT Detection Capabilities:**

**Long-term Pattern Analysis:**
- **Seasonal behavior** - identify gradual changes в user behavior
- **Access creep** - users gaining access to additional resources over time
- **Communication evolution** - changes в network communication patterns
- **Off-hours activity** - consistent unusual timing patterns

**Lateral Movement Detection:**
- **Privilege escalation** - users gaining administrative rights
- **Cross-system access** - unusual access between different systems
- **Geographic impossibilities** - simultaneous logins от different locations
- **Tool usage analysis** - administrative tools used by non-admin accounts

**🔮 Threat Intelligence Integration (5 мин)**

**Threat Intelligence Sources:**
- **Commercial feeds** - paid threat intelligence services
- **Government sources** - national cybersecurity agencies  
- **Industry sharing** - sector-specific threat information
- **Open source intelligence** - freely available threat data
- **Internal intelligence** - organization-specific threat indicators

**Indicators на Compromise (IOCs):**
- **File hashes** - known malicious file signatures
- **IP addresses** - command и control servers, scanning sources
- **Domain names** - malicious websites, phishing domains
- **URL patterns** - suspicious web resource patterns
- **Email indicators** - malicious sender addresses, subject lines

**Automated Threat Hunting:**
- **IOC matching** - automatically search logs за known threats
- **Threat feed integration** - real-time updates на threat indicators
- **Retroactive hunting** - search historical data за newly discovered threats
- **False positive management** - refine indicators за accuracy improvement

**Step 1: Hypothesis Validation (6 мин)**

- **Behavioral change** - dramatic increase в file access и data uploads

- **Strong indicators** - geographic и temporal anomalies (high confidence)

- **Time correlation** - match activity spikes със known attack campaigns

**Step 3: Investigation Conclusions (5 мин)**

**Determine likelihood на compromise:**

**Evidence Summary:**
- **HIGH PROBABILITY** indicators: Geographic/time anomalies, massive data access increase
- **MEDIUM PROBABILITY** indicators: New tool usage, network volume changes

**Recommended Actions:**
1. **Immediate** - Contact maria.petrova за verify recent activity legitimacy

3. **Technical** - Malware scan на workstation, VPN access review
4. **Monitoring** - Enhanced logging за this account и similar patterns
