### **ЧАС 1: Virtualization технологии и security архитектура (40 мин)**

#### **Теоретична част (25 мин):**

**🖥️ Virtualization Fundamentals (10 мин)**

**Virtualization Types:**
- **Hardware virtualization:** complete computer simulation в software
- **Operating system virtualization:** containers sharing kernel
- **Application virtualization:** isolated application execution environments
- **Network virtualization:** virtual networks independent от physical infrastructure

**Hypervisor Types:**
- **Type 1 (Bare Metal):** hypervisor runs directly на hardware
  - **Examples:** VMware vSphere, Microsoft Hyper-V, Citrix XenServer
  - **Advantages:** better performance, stronger isolation
  - **Use cases:** enterprise data centers, cloud infrastructure

- **Type 2 (Hosted):** hypervisor runs на host operating system
  - **Examples:** VMware Workstation, VirtualBox, Parallels Desktop
  - **Advantages:** easier installation, better hardware compatibility
  - **Use cases:** development, testing, desktop virtualization

**🔒 Hypervisor Security Architecture (8 мин)**

**Isolation Mechanisms:**
- **Memory isolation:** each VM has dedicated memory space
- **CPU isolation:** processor time sharing със security boundaries
- **Storage isolation:** virtual disks separated от host filesystem
- **Network isolation:** virtual network interfaces със traffic separation

**Security Boundaries:**
- **VM-to-VM isolation:** preventing lateral movement between virtual machines
- **VM-to-host isolation:** protecting host system от compromised VMs
- **Hypervisor attack surface:** minimizing hypervisor vulnerability exposure
- **Hardware security:** leveraging hardware virtualization features

**Hardware-Assisted Security:**
- **Intel VT-x/VT-d:** processor virtualization extensions
- **AMD-V/IOMMU:** AMD processor virtualization support
- **ARM TrustZone:** secure и non-secure worlds separation
- **TPM (Trusted Platform Module):** hardware-based security anchoring

**🌐 Virtual Network Security (7 мин)**

**Virtual Switch Security:**
- **VLAN isolation:** virtual network segmentation
- **Traffic inspection:** monitoring inter-VM communications
- **Firewall integration:** virtual и physical firewall coordination
- **Access control:** virtual machine network access policies

**Virtual Network Types:**
- **NAT networks:** shared internet access със address translation
- **Bridged networks:** direct physical network connection
- **Host-only networks:** isolated communication със host system
- **Internal networks:** VM-to-VM communication only

**Software-Defined Networking (SDN):**
- **Centralized control:** programmable network management
- **Micro-segmentation:** granular network access control
- **Policy enforcement:** automated security rule application
- **Network monitoring:** comprehensive traffic analysis capabilities

### **ЧАС 2: Container security fundamentals (40 мин)**

#### **Теоретична част (24 мин):**

**📦 Container Technology Overview (10 мин)**

**Containers vs Virtual Machines:**
- **Resource efficiency:** containers share kernel, VMs include full OS
- **Startup time:** containers start в seconds, VMs take minutes
- **Resource overhead:** containers use fewer resources за same workload
- **Isolation level:** VMs provide stronger isolation, containers offer process-level isolation

**Docker Architecture:**
- **Docker Engine:** container runtime environment
- **Docker Images:** read-only templates за creating containers
- **Docker Containers:** running instances от images
- **Docker Registry:** centralized storage за container images
- **Dockerfile:** instructions за building custom images

**Container Benefits:**
- **Application portability:** runs consistently across different environments
- **Scalability:** easy horizontal scaling на applications
- **Development efficiency:** consistent environments from dev to production
- **Resource utilization:** higher density на applications per server

**🔐 Container Security Challenges (8 мин)**

**Security Differences from VMs:**
- **Shared kernel:** all containers use host kernel (potential attack vector)
- **Namespace isolation:** Linux namespaces provide process separation
- **Control groups (cgroups):** resource limiting и isolation
- **Capabilities:** fine-grained privilege control
- **SELinux/AppArmor integration:** mandatory access control за containers

**Container Attack Vectors:**
- **Container escape:** breaking out от container to host system
- **Malicious images:** infected или backdoored container images
- **Privilege escalation:** gaining elevated privileges within или outside container
- **Resource exhaustion:** DoS attacks through resource consumption
- **Network attacks:** lateral movement between containers

**Image Security Concerns:**
- **Base image vulnerabilities:** inherited security flaws from base images
- **Supply chain attacks:** compromised images в registries
- **Secrets management:** passwords и keys stored в container images
- **Image signing:** verification на image authenticity и integrity

**🏗️ Container Security Best Practices (6 мин)**

**Secure Image Management:**
- **Minimal base images:** using smallest possible base images (Alpine Linux)
- **Regular updates:** keeping base images current със security patches
- **Image scanning:** automated vulnerability detection в images
- **Signed images:** cryptographic verification на image authenticity
- **Private registries:** controlled access to container images

**Runtime Security:**
- **Non-root execution:** running containers as non-privileged users
- **Read-only filesystems:** preventing runtime modifications
- **Resource limits:** CPU, memory, disk usage restrictions
- **Network policies:** controlling container-to-container communication
- **Security contexts:** SELinux/AppArmor policies за containers

**Container Orchestration Security:**
- **Kubernetes security:** secure cluster management
- **Role-based access control (RBAC):** controlling who can do what
- **Pod security policies:** enforcing security standards на container groups
- **Network policies:** fine-grained network access control
- **Secrets management:** secure handling на passwords и certificates

### **ЧАС 3: Virtual network security и cloud fundamentals (40 мин)**

#### **Теоретична část (22 мин):**

**🌐 Virtual Network Security Architecture (10 мин)**

**Virtual Network Components:**
- **Virtual switches:** software-based network switching
- **Virtual routers:** routing between virtual networks  
- **Virtual firewalls:** traffic filtering at virtual network level
- **Virtual load balancers:** distributing traffic across virtual machines
- **Virtual VPN gateways:** secure connections between virtual networks

**Network Virtualization Benefits:**
- **Isolation:** separate networks за different security zones
- **Scalability:** easy addition/removal на network segments
- **Flexibility:** dynamic network reconfiguration
- **Cost efficiency:** multiple networks на single physical infrastructure

**Security Zones в Virtual Environments:**
- **DMZ (Demilitarized Zone):** public-facing services isolation
- **Internal networks:** corporate application communication
- **Management networks:** administrative access isolation
- **Backup networks:** data protection traffic separation

**☁️ Cloud Security Fundamentals (12 мин)**

**Cloud Service Models:**
- **Infrastructure as a Service (IaaS):** virtual machines, storage, networks
- **Platform as a Service (PaaS):** development platforms и frameworks
- **Software as a Service (SaaS):** complete applications delivered online
- **Security responsibilities:** shared между cloud provider и customer

**Shared Responsibility Model:**
- **Cloud provider responsibilities:** physical security, hypervisor security, network infrastructure
- **Customer responsibilities:** OS patching, application security, data protection
- **Shared responsibilities:** network access controls, platform security

**Cloud Security Benefits:**
- **Professional security teams:** cloud providers have dedicated security experts
- **Scale economies:** security investments spread across many customers
- **Automatic updates:** infrastructure automatically maintained и patched
- **Compliance certifications:** cloud providers maintain regulatory compliance

**Cloud Security Challenges:**
- **Data location:** uncertainty about physical data storage location
- **Vendor dependency:** reliance на cloud provider security practices
- **Network security:** protecting data в transit to/from cloud
- **Multi-tenancy:** sharing infrastructure с other cloud customers
- **Regulatory compliance:** meeting data residency и sovereignty requirements
