### **ЧАС 1: Virtualization технологии и security архитектура (40 мин)**

#### **Теоретична част (25 мин):**

**Virtualization Fundamentals (10 мин)**

**Видове Virtualization:**
- **Hardware virtualization:** пълна компютърна симулация в software
- **Operating system virtualization:** контейнери, споделящи kernel
- **Application virtualization:** изолирани среди за изпълнение на приложения
- **Network virtualization:** виртуални мрежи, независими от физическата инфраструктура

**Видове Hypervisor:**
- **Type 1 (Bare Metal):** hypervisor-ът работи директно на хардуера
  - **Примери:** VMware vSphere, Microsoft Hyper-V, Citrix XenServer
  - **Предимства:** по-добра производителност, по-силна изолация
  - **Случаи на употреба:** enterprise data centers, cloud инфраструктура

- **Type 2 (Hosted):** hypervisor-ът работи на host операционната система
  - **Примери:** VMware Workstation, VirtualBox, Parallels Desktop
  - **Предимства:** по-лесна инсталация, по-добра хардуерна съвместимост
  - **Случаи на употреба:** разработка, тестване, desktop virtualization

**Hypervisor Security Architecture (8 мин)**

**Isolation Mechanisms:**
- **Memory isolation:** всяка VM има отделно пространство в паметта
- **CPU isolation:** споделяне на процесорно време с security граници
- **Storage isolation:** виртуални дискове, отделени от host файловата система
- **Network isolation:** виртуални мрежови интерфейси с разделяне на трафика

**Security граници:**
- **VM-to-VM изолация:** предотвратяване на странично движение между виртуални машини
- **VM-to-host изолация:** защита на host системата от компрометирани VM-ове
- **Hypervisor attack surface:** минимизиране на излагането на hypervisor уязвимости
- **Hardware security:** използване на хардуерни virtualization функции

**Hardware-Assisted Security:**
- **Intel VT-x/VT-d:** процесорни virtualization разширения
- **AMD-V/IOMMU:** AMD процесорна virtualization поддръжка
- **ARM TrustZone:** разделяне на secure и non-secure светове
- **TPM (Trusted Platform Module):** хардуерно базирана security котва

**Virtual Network Security (7 мин)**

**Virtual Switch Security:**
- **VLAN изолация:** сегментация на виртуалната мрежа
- **Traffic inspection:** мониторинг на inter-VM комуникации
- **Firewall интеграция:** координация на виртуален и физически firewall
- **Access control:** политики за мрежов достъп на виртуалните машини

**Видове виртуални мрежи:**
- **NAT мрежи:** споделен интернет достъп с address translation
- **Bridged мрежи:** директна физическа мрежова връзка
- **Host-only мрежи:** изолирана комуникация с host системата
- **Internal мрежи:** само VM-to-VM комуникация

**Software-Defined Networking (SDN):**
- **Централизиран контрол:** програмируемо управление на мрежата
- **Micro-segmentation:** гранулиран контрол на мрежовия достъп
- **Policy enforcement:** автоматизирано прилагане на security правила
- **Network monitoring:** цялостни възможности за анализ на трафика

### **ЧАС 2: Container security fundamentals (40 мин)**

#### **Теоретична част (24 мин):**

**Преглед на Container технологията (10 мин)**

**Контейнери срещу Виртуални машини:**
- **Ресурсна ефективност:** контейнерите споделят kernel, VM-овете включват пълна OS
- **Време за стартиране:** контейнерите стартират за секунди, VM-овете отнемат минути
- **Ресурсни разходи:** контейнерите използват по-малко ресурси за същото натоварване
- **Ниво на изолация:** VM-овете осигуряват по-силна изолация, контейнерите предлагат изолация на ниво процес

**Docker Architecture:**
- **Docker Engine:** среда за изпълнение на контейнери
- **Docker Images:** read-only шаблони за създаване на контейнери
- **Docker Containers:** работещи инстанции от images
- **Docker Registry:** централизирано хранилище за container images
- **Dockerfile:** инструкции за изграждане на custom images

**Предимства на контейнерите:**
- **Преносимост на приложения:** работи последователно в различни среди
- **Scalability:** лесно хоризонтално мащабиране на приложения
- **Ефективност при разработка:** последователни среди от разработка до продукция
- **Използване на ресурси:** по-висока плътност на приложения на сървър

**Container Security предизвикателства (8 мин)**

**Security разлики от VM-ове:**
- **Споделен kernel:** всички контейнери използват host kernel (потенциален attack vector)
- **Namespace isolation:** Linux namespaces осигуряват разделяне на процесите
- **Control groups (cgroups):** ограничаване на ресурсите и изолация
- **Capabilities:** гранулиран контрол на привилегиите
- **SELinux/AppArmor интеграция:** задължителен контрол на достъпа за контейнери

**Container Attack Vectors:**
- **Container escape:** излизане от контейнера към host системата
- **Malicious images:** заразени или backdoored container images
- **Privilege escalation:** придобиване на повишени привилегии вътре или извън контейнера
- **Resource exhaustion:** DoS атаки чрез консумация на ресурси
- **Network атаки:** странично движение между контейнери

**Опасения за сигурността на images:**
- **Base image уязвимости:** наследени security пропуски от base images
- **Supply chain атаки:** компрометирани images в регистри
- **Secrets management:** пароли и ключове, съхранени в container images
- **Image signing:** верификация на автентичността и интегритета на images

**Container Security Best Practices (6 мин)**

**Сигурно управление на images:**
- **Минимални base images:** използване на най-малките възможни base images (Alpine Linux)
- **Редовни актуализации:** поддържане на base images актуални с security patches
- **Image сканиране:** автоматизирано засичане на уязвимости в images
- **Signed images:** криптографска верификация на автентичността на images
- **Частни регистри:** контролиран достъп до container images

**Runtime Security:**
- **Non-root изпълнение:** изпълнение на контейнери като непривилегировани потребители
- **Read-only файлови системи:** предотвратяване на модификации по време на изпълнение
- **Resource limits:** ограничения на CPU, памет, дисково използване
- **Network policies:** контролиране на container-to-container комуникация
- **Security contexts:** SELinux/AppArmor политики за контейнери

**Container Orchestration Security:**
- **Kubernetes security:** сигурно управление на клъстер
- **Role-based access control (RBAC):** контрол кой какво може да прави
- **Pod security policies:** налагане на security стандарти за групи контейнери
- **Network policies:** гранулиран контрол на мрежовия достъп
- **Secrets management:** сигурно обработване на пароли и сертификати

### **ЧАС 3: Virtual network security и cloud fundamentals (40 мин)**

#### **Теоретична част (22 мин):**

**Virtual Network Security Architecture (10 мин)**

**Компоненти на виртуалната мрежа:**
- **Virtual switches:** software-базирано мрежово switching
- **Virtual routers:** маршрутизиране между виртуални мрежи
- **Virtual firewalls:** филтриране на трафика на ниво виртуална мрежа
- **Virtual load balancers:** разпределяне на трафика между виртуални машини
- **Virtual VPN gateways:** сигурни връзки между виртуални мрежи

**Предимства на Network Virtualization:**
- **Изолация:** отделни мрежи за различни security зони
- **Scalability:** лесно добавяне/премахване на мрежови сегменти
- **Гъвкавост:** динамично преконфигуриране на мрежата
- **Ефективност на разходите:** множество мрежи на единична физическа инфраструктура

**Security зони в Virtual среди:**
- **DMZ (Demilitarized Zone):** изолация на публично достъпни услуги
- **Вътрешни мрежи:** комуникация на корпоративни приложения
- **Management мрежи:** изолация на административен достъп
- **Backup мрежи:** разделяне на трафика за защита на данни

**Cloud Security Fundamentals (12 мин)**

**Модели на Cloud услуги:**
- **Infrastructure as a Service (IaaS):** виртуални машини, съхранение, мрежи
- **Platform as a Service (PaaS):** платформи за разработка и frameworks
- **Software as a Service (SaaS):** пълни приложения, доставяни онлайн
- **Security отговорности:** споделени между cloud доставчик и клиент

**Модел на споделена отговорност:**
- **Отговорности на cloud доставчика:** физическа сигурност, hypervisor security, мрежова инфраструктура
- **Отговорности на клиента:** OS patching, application security, защита на данни
- **Споделени отговорности:** контроли на мрежовия достъп, платформена сигурност

**Предимства на Cloud Security:**
- **Професионални security екипи:** cloud доставчиците имат специализирани security експерти
- **Икономии от мащаба:** security инвестиции, разпределени между много клиенти
- **Автоматични актуализации:** инфраструктурата автоматично се поддържа и патчира
- **Compliance сертификации:** cloud доставчиците поддържат regulatory compliance

**Cloud Security предизвикателства:**
- **Местоположение на данните:** несигурност относно физическото местоположение за съхранение на данни
- **Зависимост от доставчика:** разчитане на security практиките на cloud доставчика
- **Мрежова сигурност:** защита на данни при транзит към/от cloud
- **Multi-tenancy:** споделяне на инфраструктура с други cloud клиенти
- **Regulatory compliance:** спазване на изисквания за местонахождение и суверенитет на данни

### **ЧАС 4: Cloud security и virtualization best practices (40 мин)**

#### **Теоретична част (20 мин):**

**Cloud Security в дълбочина (12 мин)**

**Security на основните Cloud платформи:**

**Amazon Web Services (AWS):**
