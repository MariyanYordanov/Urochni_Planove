### **ЧАС 1: iOS и Android security архитектури (40 мин)**

#### **Теоретична част (25 мин):**

**📱 Mobile OS Security Fundamentals (8 мин)**

**Unique Mobile Security Challenges:**
- **Always connected:** constant network exposure через cellular и Wi-Fi
- **Personal devices:** mixing personal и business data на same device
- **App ecosystem:** third-party applications със varying security standards
- **Physical security:** devices easily lost, stolen, или accessed without authorization
- **Location tracking:** GPS и location services creating privacy risks

**Security vs Usability Balance:**
- **Convenience expectations:** users expect instant access и seamless experience
- **Battery life:** security features consume additional power
- **Performance impact:** encryption и monitoring affecting responsiveness
- **User experience:** security measures shouldn't impede daily usage

**🍎 iOS Security Architecture (8 мин)**

**iOS Security Foundation:**
- **Secure Boot Chain:** hardware-verified boot process от bootloader to kernel
- **System Integrity Protection (SIP):** preventing unauthorized system modifications
- **Data Protection API:** automatic file-level encryption integration
- **App Sandbox:** strict isolation между applications
- **Code Signing:** all apps must be cryptographically signed

**iOS Permission Model:**
- **App Store review:** Apple manually reviews all applications
- **Entitlements:** specific capabilities apps can request
- **Runtime permissions:** user approval required за sensitive operations
- **Privacy prompts:** explicit consent за camera, microphone, location access

**iOS Security Features:**
- **Touch ID/Face ID:** biometric authentication integration
- **Keychain Services:** secure credential storage
- **App Transport Security:** encrypted network communications
- **FileVault equivalent:** full device encryption standard

**🤖 Android Security Architecture (9 мин)**

**Android Security Model:**
- **Linux kernel foundation:** leveraging proven Unix security model
- **Application sandbox:** each app runs в separate Linux user context
- **Permission-based access:** explicit user approval за sensitive operations
- **SELinux enforcement:** mandatory access control protection
- **Verified Boot:** cryptographic verification на system integrity

**Android Permission Categories:**
- **Install-time permissions:** granted during app installation (older Android)
- **Runtime permissions:** requested when feature is accessed (Android 6+)
- **Special permissions:** dangerous capabilities requiring user approval
- **System permissions:** capabilities only available to system apps

**Android Security Evolution:**
- **Android 4.0+:** app sandboxing и permission improvements
- **Android 6.0+:** runtime permissions model
- **Android 8.0+:** background app restrictions, notification channels
- **Android 10+:** scoped storage, privacy improvements
- **Android 11+:** one-time permissions, auto-reset unused app permissions

**Google Play Protect:**
- **Machine learning scanning:** automated malware detection
- **Real-time protection:** continuous app monitoring
- **Find My Device:** remote locate, lock, erase capabilities
- **Safe Browsing:** protection срещу malicious websites

### **ЧАС 2: Mobile application security и permission models (40 мин)**

#### **Теоретична част (22 мин):**

**📋 Mobile App Permission Systems (12 мин)**

**Android Permission Model Evolution:**

**Legacy Permission Model (Android <6):**
- **Install-time grants:** all permissions approved during installation
- **All-or-nothing:** users must accept all permissions или not install app
- **Static permissions:** cannot be modified after installation
- **User confusion:** long permission lists без context

**Modern Permission Model (Android 6+):**
- **Runtime permissions:** requested when feature is actually used
- **Granular control:** users can approve/deny individual permissions
- **Permission groups:** related permissions bundled together
- **Dynamic management:** permissions can be granted/revoked anytime

**Android Permission Categories:**
```
Normal Permissions (automatic approval):
- Internet access, Wi-Fi state, alarm setting
- Network state checking, vibration control

Dangerous Permissions (user approval required):
- Camera, microphone, location services
- Contacts, calendar, call logs access  
- Storage access, SMS reading/sending

Special Permissions (additional user action):
- Device administrator access
- Accessibility services usage
- System overlay windows
- Usage access statistics
```

**iOS Permission Philosophy:**
- **Privacy by design:** users control access to personal information
- **Just-in-time prompts:** permissions requested when needed
- **Clear purpose explanation:** apps must explain why permissions are needed
- **Easy revocation:** users can change permissions anytime в Settings

**🔒 App Security Mechanisms (10 мин)**

**Code Signing и App Store Security:**

**Apple App Store:**
- **Manual review process:** human reviewers examine each app
- **Static analysis:** automated code scanning за malware
- **Developer verification:** paid developer accounts със identity verification
- **Notarization:** additional security verification process

**Google Play Store:**
- **Play Protect scanning:** machine learning malware detection
- **Developer verification:** Google Play Console account verification
- **App bundles:** signed packages със integrity verification
- **Dynamic analysis:** runtime behavior monitoring

**App Isolation Techniques:**
- **Sandbox environments:** each app runs в isolated container
- **Inter-app communication:** controlled communication between apps
- **Shared storage restrictions:** limited access to shared device areas
- **Background restrictions:** limiting what apps can do when not active

**App Security Best Practices:**
- **Principle of least privilege:** request minimum necessary permissions
- **Data minimization:** collect only required user data
- **Secure coding:** input validation, proper authentication
- **Regular updates:** fixing security vulnerabilities promptly

### **ЧАС 3: Mobile network security и threat landscape (40 мин)**

#### **Теоретична част (24 мин):**

**📡 Mobile Network Security Challenges (10 мин)**

**Cellular Network Security:**
- **GSM vulnerabilities:** weak encryption (A5/1, A5/2 algorithms)
- **3G security improvements:** mutual authentication между device и network
- **4G LTE:** stronger encryption, но still vulnerable to stingray attacks
- **5G security:** enhanced authentication, но new attack vectors
- **Carrier security:** dependency на mobile network operator security practices

**Wi-Fi Security Risks:**
- **Public Wi-Fi dangers:** unencrypted networks, man-in-the-middle attacks
- **Evil twin attacks:** fake Wi-Fi hotspots mimicking legitimate networks
- **Wi-Fi password cracking:** WEP/WPA vulnerabilities
- **Packet sniffing:** eavesdropping на unencrypted communications
- **Captive portal risks:** fake login pages capturing credentials

**Bluetooth Security Issues:**
- **Bluejacking:** sending unsolicited messages to nearby devices
- **Bluesnarfing:** unauthorized access to device data
- **Bluetooth eavesdropping:** intercepting audio communications
- **Device tracking:** Bluetooth beacons enabling location monitoring

**📱 Mobile Threat Landscape (8 мин)**

**Common Mobile Threats:**
- **Malware:** mobile viruses, trojans, spyware, ransomware
- **Phishing:** SMS и email attacks targeting mobile users  
- **Social engineering:** manipulating users through mobile communications
- **Physical theft:** device loss и unauthorized physical access
- **Data leakage:** apps transmitting sensitive information unnecessarily

**Mobile-Specific Attack Vectors:**
- **SMS hijacking:** SIM swapping за account takeover
- **App store malware:** malicious apps в official stores
- **Zero-day exploits:** unpatched vulnerabilities в mobile OS
- **Jailbreaking/rooting:** bypassing built-in security restrictions
- **Mobile banking trojans:** targeting financial applications

**Emerging Mobile Threats:**
- **AI-powered attacks:** personalized social engineering using collected data
- **IoT integration:** mobile devices като entry point to smart home networks
- **Deepfake technology:** fake voice/video calls за social engineering
- **Supply chain attacks:** compromised hardware или firmware
- **5G security challenges:** new protocols creating new attack opportunities

**🛡️ Mobile Security Solutions (6 мин)**

**Built-in Security Features:**
- **Device encryption:** automatic full-device encryption (iOS default, Android configurable)
- **Remote wipe:** ability to erase device remotely if lost/stolen
- **Find My Device/Find My iPhone:** location tracking за lost devices
- **App sandboxing:** strict isolation between applications
- **Automatic security updates:** OS-level patches delivered automatically

**Mobile Device Management (MDM):**
- **Enterprise device management:** centralized control на corporate mobile devices
- **Policy enforcement:** automated configuration на security settings
- **App whitelisting/blacklisting:** controlling which apps can be installed
- **Remote management:** wiping devices, installing apps, updating configurations
- **Compliance monitoring:** ensuring devices meet corporate security standards

**Virtual Private Networks (VPN):**
- **Encrypted tunneling:** protecting data transmission over untrusted networks
- **Corporate VPN access:** secure connection to company networks
- **Consumer VPN services:** privacy protection за general internet usage
- **Kill switch functionality:** blocking internet if VPN connection drops

### **ЧАС 4: BYOD policies и enterprise mobile security (40 мин)**

#### **Теоретична част (20 мин):**

**💼 BYOD (Bring Your Own Device) Challenges (10 мин)**

**BYOD Benefits:**
- **Cost savings:** employees provide their own devices
- **User satisfaction:** working со familiar, preferred devices
- **Productivity:** immediate access to corporate resources
- **Flexibility:** working from любо location със personal device

**BYOD Security Risks:**
- **Data mixing:** personal и corporate data на same device
- **Update control:** users may delay security updates
- **App installation:** unrestricted app installation creating vulnerabilities
- **Physical security:** personal devices lack corporate physical protections
- **Data sovereignty:** corporate data stored на personal devices

**Compliance Challenges:**
- **Regulatory requirements:** GDPR, HIPAA data protection на personal devices
- **Audit difficulties:** tracking corporate data на personal devices
- **Legal discovery:** accessing corporate data за litigation purposes
- **Data residency:** ensuring corporate data stays в approved geographic regions

**🏢 Enterprise Mobile Security Solutions (10 мин)**

**Mobile Device Management (MDM):**
- **Device enrollment:** registering devices за corporate management
- **Policy enforcement:** mandatory security settings (screen lock, encryption)
- **App management:** required apps, blacklisted apps, managed app store
- **Remote capabilities:** locate, lock, wipe devices
- **Compliance reporting:** monitoring device adherence to security policies

**Mobile Application Management (MAM):**
- **App wrapping:** adding security controls to existing applications
- **Containerization:** isolating corporate apps от personal apps
- **App-specific policies:** VPN requirements, copy/paste restrictions
- **Data loss prevention:** preventing corporate data sharing със personal apps

**Unified Endpoint Management (UEM):**
- **Cross-platform management:** managing iOS, Android, Windows, macOS devices
- **Lifecycle management:** device provisioning, management, retirement
- **Security integration:** connecting mobile device security със enterprise security infrastructure
- **Analytics и reporting:** device usage, security compliance, threat detection

**BYOD Policy Framework:**
- **Device registration:** mandatory enrollment process за accessing corporate resources
- **Acceptable use policies:** defining appropriate personal device usage
- **Data classification:** separating personal і corporate data clearly
- **Incident response:** procedures за compromised personal devices
- **Privacy protection:** protecting employee personal data during corporate management
