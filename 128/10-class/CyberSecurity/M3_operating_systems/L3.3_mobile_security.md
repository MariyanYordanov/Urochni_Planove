### **ЧАС 1: iOS и Android security архитектури (40 мин)**

#### **Теоретична част (25 мин):**

**Основи на Mobile OS Security (8 мин)**

**Уникални предизвикателства на мобилната сигурност:**
- **Постоянна свързаност:** непрекъсната мрежова изложеност чрез cellular и Wi-Fi
- **Лични устройства:** смесване на лични и бизнес данни на едно устройство
- **App ecosystem:** приложения на трети страни с различни стандарти за сигурност
- **Физическа сигурност:** устройствата лесно се губят, крадат или достъпват без разрешение
- **Проследяване на местоположение:** GPS и location services създават privacy рискове

**Баланс между сигурност и удобство:**
- **Очаквания за удобство:** потребителите очакват незабавен достъп и безпроблемно изживяване
- **Живот на батерията:** security функциите консумират допълнителна мощност
- **Влияние върху производителността:** криптирането и мониторингът влияят на отзивчивостта
- **Потребителско изживяване:** мерките за сигурност не трябва да пречат на ежедневната употреба

**iOS Security Architecture (8 мин)**

**Основи на iOS Security:**
- **Secure Boot Chain:** хардуерно верифициран boot процес от bootloader до kernel
- **System Integrity Protection (SIP):** предотвратява неоторизирани системни промени
- **Data Protection API:** автоматична интеграция на криптиране на ниво файл
- **App Sandbox:** строга изолация между приложенията
- **Code Signing:** всички приложения трябва да бъдат криптографски подписани

**iOS Permission Model:**
- **App Store review:** Apple ръчно преглежда всички приложения
- **Entitlements:** специфични възможности, които приложенията могат да заявят
- **Runtime permissions:** необходимо одобрение от потребителя за чувствителни операции
- **Privacy prompts:** изрично съгласие за достъп до камера, микрофон, местоположение

**iOS Security Features:**
- **Touch ID/Face ID:** интеграция на биометрично удостоверяване
- **Keychain Services:** сигурно съхранение на идентификационни данни
- **App Transport Security:** криптирани мрежови комуникации
- **Пълно криптиране на устройство:** стандарт по подразбиране

**Android Security Architecture (9 мин)**

**Android Security Model:**
- **Linux kernel foundation:** използване на доказания Unix security model
- **Application sandbox:** всяко приложение се изпълнява в отделен Linux потребителски контекст
- **Permission-based access:** изрично одобрение от потребителя за чувствителни операции
- **SELinux enforcement:** задължителна защита с контрол на достъпа
- **Verified Boot:** криптографска верификация на системния интегритет

**Android Permission Categories:**
- **Install-time permissions:** предоставени по време на инсталация (по-стари Android версии)
- **Runtime permissions:** заявени при достъп до функция (Android 6+)
- **Special permissions:** опасни възможности, изискващи одобрение от потребителя
- **System permissions:** възможности, достъпни само за системни приложения

**Еволюция на Android Security:**
- **Android 4.0+:** подобрения в app sandboxing и permissions
- **Android 6.0+:** runtime permissions model
- **Android 8.0+:** ограничения за фонови приложения, notification channels
- **Android 10+:** scoped storage, подобрения в privacy
- **Android 11+:** еднократни permissions, автоматично нулиране на неизползвани app permissions

**Google Play Protect:**
- **Machine learning сканиране:** автоматично засичане на зловреден код
- **Защита в реално време:** непрекъснат мониторинг на приложения
- **Find My Device:** дистанционно намиране, заключване, изтриване
- **Safe Browsing:** защита срещу злонамерени уебсайтове

### **ЧАС 2: Mobile application security и permission models (40 мин)**

#### **Теоретична част (22 мин):**

**Mobile App Permission Systems (12 мин)**

**Еволюция на Android Permission Model:**

**Legacy Permission Model (Android <6):**
- **Предоставяне при инсталация:** всички permissions одобрени по време на инсталация
- **Всичко или нищо:** потребителите трябва да приемат всички permissions или да не инсталират приложението
- **Статични permissions:** не могат да се променят след инсталация
- **Объркване на потребителя:** дълги списъци с permissions без контекст

**Съвременен Permission Model (Android 6+):**
- **Runtime permissions:** заявени при реалната употреба на функцията
- **Гранулиран контрол:** потребителите могат да одобряват/отказват отделни permissions
- **Permission groups:** свързани permissions групирани заедно
- **Динамично управление:** permissions могат да се предоставят/отнемат по всяко време

**Android Permission Categories:**
```
Normal Permissions (automatic approval):
- Internet access, Wi-Fi state, alarm setting
- Network state checking, vibration control

Dangerous Permissions (user approval required):
- Camera, microphone, location services
- Contacts, calendar, call logs access
- Storage access, SMS reading/sending

Special Permissions (additional user action):
- Device administrator access
- Accessibility services usage
- System overlay windows
- Usage access statistics
```

**iOS Permission Philosophy:**
- **Privacy by design:** потребителите контролират достъпа до лична информация
- **Just-in-time prompts:** permissions заявени при необходимост
- **Ясно обяснение на целта:** приложенията трябва да обяснят защо са нужни permissions
- **Лесно отнемане:** потребителите могат да променят permissions по всяко време в Settings

**App Security Mechanisms (10 мин)**

**Code Signing и App Store Security:**

**Apple App Store:**
- **Ръчен процес на преглед:** хора-рецензенти разглеждат всяко приложение
- **Статичен анализ:** автоматично сканиране на код за зловреден код
- **Верификация на разработчика:** платени developer акаунти с верификация на идентичността
- **Notarization:** допълнителен процес за security верификация

**Google Play Store:**
- **Play Protect сканиране:** засичане на зловреден код с machine learning
- **Верификация на разработчика:** верификация на Google Play Console акаунт
- **App bundles:** подписани пакети с верификация на интегритет
- **Динамичен анализ:** мониторинг на поведението по време на изпълнение

**App Isolation Techniques:**
- **Sandbox environments:** всяко приложение се изпълнява в изолиран контейнер
- **Комуникация между приложения:** контролирана комуникация между apps
- **Ограничения за споделено съхранение:** ограничен достъп до споделени области на устройството
- **Background restrictions:** ограничаване на това, което приложенията могат да правят, когато не са активни

**Добри практики за сигурност на приложения:**
- **Принцип на най-малко привилегии:** заявяване на минималните необходими permissions
- **Минимизиране на данните:** събиране само на необходимите потребителски данни
- **Secure coding:** валидация на входни данни, правилна автентикация
- **Редовни актуализации:** бързо отстраняване на security уязвимости

### **ЧАС 3: Mobile network security и threat landscape (40 мин)**

#### **Теоретична част (24 мин):**

**Предизвикателства на Mobile Network Security (10 мин)**

**Сигурност на клетъчните мрежи:**
- **GSM уязвимости:** слабо криптиране (алгоритми A5/1, A5/2)
- **Подобрения в сигурността на 3G:** взаимна автентикация между устройство и мрежа
- **4G LTE:** по-силно криптиране, но все още уязвимо на stingray атаки
- **5G security:** подобрена автентикация, но нови attack vectors
- **Сигурност на оператора:** зависимост от security практиките на мобилния мрежов оператор

**Wi-Fi Security рискове:**
- **Опасности от публичен Wi-Fi:** некриптирани мрежи, man-in-the-middle атаки
- **Evil Twin атаки:** фалшиви Wi-Fi hotspots, имитиращи легитимни мрежи
- **Разбиване на Wi-Fi пароли:** WEP/WPA уязвимости
- **Packet sniffing:** подслушване на некриптирани комуникации
- **Captive portal рискове:** фалшиви login pages, улавящи идентификационни данни

**Bluetooth Security проблеми:**
- **Bluejacking:** изпращане на нежелани съобщения до близки устройства
- **Bluesnarfing:** неоторизиран достъп до данните на устройството
- **Bluetooth подслушване:** прихващане на аудио комуникации
- **Проследяване на устройства:** Bluetooth beacon-и, позволяващи мониторинг на местоположение

**Mobile Threat Landscape (8 мин)**

**Чести мобилни заплахи:**
- **Malware:** мобилни вируси, trojans, spyware, ransomware
- **Phishing:** SMS и email атаки, насочени към мобилни потребители
- **Social engineering:** манипулиране на потребители чрез мобилни комуникации
- **Физическа кражба:** загуба на устройство и неоторизиран физически достъп
- **Data leakage:** приложения, предаващи ненужно чувствителна информация

**Специфични за мобилни устройства attack vectors:**
- **SMS hijacking:** SIM swapping за превземане на акаунт
- **App store malware:** злонамерени приложения в официални магазини
- **Zero-day exploits:** неотстранени уязвимости в mobile OS
- **Jailbreaking/rooting:** заобикаляне на вградените security ограничения
- **Mobile banking trojans:** насочени срещу финансови приложения

**Нови мобилни заплахи:**
- **AI-powered атаки:** персонализиран social engineering с помощта на събрани данни
- **IoT интеграция:** мобилни устройства като входна точка към smart home мрежи
- **Deepfake технология:** фалшиви гласови/видео разговори за social engineering
- **Supply chain атаки:** компрометиран хардуер или firmware
- **5G security предизвикателства:** нови протоколи, създаващи нови атакуващи възможности

**Mobile Security Solutions (6 мин)**

**Вградени security функции:**
- **Криптиране на устройство:** автоматично пълно криптиране на устройство (iOS по подразбиране, Android конфигурируемо)
- **Дистанционно изтриване:** способност за дистанционно изтриване на устройството при загуба/кражба
- **Find My Device/Find My iPhone:** проследяване на местоположение за изгубени устройства
- **App sandboxing:** строга изолация между приложенията
- **Автоматични security актуализации:** OS-ниво patches доставени автоматично

**Mobile Device Management (MDM):**
- **Enterprise device management:** централизирано управление на корпоративни мобилни устройства
- **Policy enforcement:** автоматизирано конфигуриране на security настройки
- **App whitelisting/blacklisting:** контрол върху кои приложения могат да бъдат инсталирани
- **Дистанционно управление:** изтриване на устройства, инсталиране на приложения, актуализиране на конфигурации
- **Compliance monitoring:** осигуряване, че устройствата отговарят на корпоративните security стандарти

**Virtual Private Networks (VPN):**
- **Криптиран тунел:** защита на предаването на данни по ненадеждни мрежи
- **Корпоративен VPN достъп:** сигурна връзка към фирмените мрежи
- **Потребителски VPN услуги:** защита на privacy за обща употреба на интернет
- **Kill switch функционалност:** блокиране на интернет при прекъсване на VPN връзката

**Оценяване на Wi-Fi мрежа:**

**Сигурност на клетъчната връзка:**
- Провери оператора и силата на сигнала
- Идентифицирай модели на използване на данни (приложения, използващи cellular данни)
- Прегледай настройките за роуминг (съображения за международно пътуване)
- Документирай използването на VPN (кога/къде използваш VPN защита)

**Одит на Bluetooth сигурност:**
- Изброй свързани Bluetooth устройства
- Провери за неизвестни или ненадеждни свързани устройства
- Прегледай настройките за Bluetooth видимост
- Оцени необходимостта за всяко свързано устройство

**Оценяване на риска от публичен Wi-Fi:**
*Помисли за последния път, когато си използвал публичен Wi-Fi:*
- **Местоположение:** Летище/Кафене/Хотел/Друго
- **Дейности:** Имейл/Социални медии/Банкиране/Работа
- **Използвана защита:** VPN/само HTTPS/без специални предпазни мерки
- **Ниво на риск:** Висок/Среден/Нисък въз основа на дейностите

**Планиране на сценарий на атака:**

```
- Unencrypted Wi-Fi network

- Automatic email sync over unprotected connection

```

**Фаза 3: Планиране за подобрение на сигурността (4 мин)**
*Разработи стратегия за подобрение на мрежовата сигурност:*

**Незабавни действия за мрежова сигурност:**
- Премахни неизвестните запазени Wi-Fi мрежи
- Изключи автоматичното свързване към отворени мрежи
- Включи VPN за използване на публични мрежи
- Изключи Wi-Fi и Bluetooth когато не са нужни

**Мрежова сигурност на приложенията:**
- Прегледай кои приложения могат да използват cellular данни
- Изключи background app refresh на cellular за несъществени приложения
- Включи специфични security функции за приложения (криптиран backup, изисквания за сигурна връзка)
- Конфигурирай специфично за приложение VPN използване там, където е налично

### **ЧАС 4: BYOD policies и enterprise mobile security (40 мин)**

#### **Теоретична част (20 мин):**

**BYOD (Bring Your Own Device) предизвикателства (10 мин)**

**BYOD предимства:**
- **Икономии на разходи:** служителите осигуряват собствените си устройства
- **Удовлетворение на потребителите:** работа с познати, предпочитани устройства
- **Производителност:** незабавен достъп до корпоративни ресурси
- **Гъвкавост:** работа от всяко място с лично устройство

**BYOD Security рискове:**
- **Смесване на данни:** лични и корпоративни данни на едно устройство
- **Контрол на актуализациите:** потребителите могат да забавят security актуализациите
- **Инсталиране на приложения:** неограниченото инсталиране на приложения създава уязвимости
- **Физическа сигурност:** личните устройства нямат корпоративна физическа защита
- **Суверенитет на данните:** корпоративни данни, съхранени на лични устройства

**Compliance предизвикателства:**
- **Регулаторни изисквания:** GDPR, HIPAA защита на данни на лични устройства
- **Трудности при одит:** проследяване на корпоративни данни на лични устройства
- **Правно разкриване:** достъп до корпоративни данни за съдебни цели
- **Местонахождение на данни:** осигуряване, че корпоративните данни остават в одобрени географски региони

**Enterprise Mobile Security решения (10 мин)**

**Mobile Device Management (MDM):**
- **Регистрация на устройства:** регистриране на устройства за корпоративно управление
- **Policy enforcement:** задължителни security настройки (заключване на екрана, криптиране)
- **Управление на приложения:** задължителни приложения, забранени приложения, управляван app store
- **Дистанционни възможности:** намиране, заключване, изтриване на устройства
- **Compliance reporting:** мониторинг на спазването на security политиките от устройствата

**Mobile Application Management (MAM):**
- **App wrapping:** добавяне на security контроли към съществуващи приложения
- **Containerization:** изолиране на корпоративни приложения от лични
- **Специфични за приложение политики:** VPN изисквания, ограничения за copy/paste
- **Предотвратяване на загуба на данни:** предотвратяване на споделяне на корпоративни данни с лични приложения

**Unified Endpoint Management (UEM):**
- **Управление на различни платформи:** управление на iOS, Android, Windows, macOS устройства
- **Lifecycle management:** провизиониране, управление и извеждане от употреба на устройства
- **Security интеграция:** свързване на mobile device security с enterprise security инфраструктурата
- **Анализи и отчети:** използване на устройства, security compliance, засичане на заплахи

**BYOD Policy Framework:**
- **Регистрация на устройства:** задължителен процес на регистрация за достъп до корпоративни ресурси
- **Политики за приемливо използване:** дефиниране на подходящото използване на лични устройства
- **Класификация на данни:** ясно разделяне на лични и корпоративни данни
- **Incident response:** процедури за компрометирани лични устройства
- **Защита на privacy:** защита на личните данни на служителите при корпоративно управление

#### **Практическа част (20 мин):**

**Упражнение 3.3.4: "BYOD Policy Development Workshop" (20 мин)**

**Здравеопазване (Група A):**
- **Основни рискове:** излагане на данни на пациенти, HIPAA нарушения

- **Пилотна програма:** тестване с малка група потребители
- **Изисквания за обучение:** обучение на потребителите относно политиките и процедурите
- **Разгръщане на технологии:** MDM системи, security приложения, VPN настройка
