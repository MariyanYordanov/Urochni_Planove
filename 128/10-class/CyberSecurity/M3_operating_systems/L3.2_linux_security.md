### **ЧАС 1: Linux security архитектура и основни концепции (40 мин)**

#### **Теоретична част (25 мин):**

**🐧 Linux Security Model Overview (10 мин)**

**Unix Security Heritage:**
- **Principle of least privilege:** users получават minimum необходими права  
- **Everything is a file:** uniform access control model за всички resources
- **Multi-user design:** built-in separation между user contexts
- **Open source advantage:** community-driven security improvements и transparency

**Linux Security Architecture:**
- **Kernel space vs User space:** clear separation със controlled interfaces
- **System calls:** secure communication между user programs и kernel
- **Process isolation:** each process runs в own protected memory space
- **Capabilities system:** fine-grained privileges instead на all-or-nothing root

**Core Security Components:**
- **Authentication:** user identity verification (passwords, keys, certificates)
- **Authorization:** access control to resources based на identity
- **Accounting:** logging и auditing на user actions
- **Cryptography:** data protection through encryption и signing

**🔐 Permission Model Deep Dive (8 мин)**

**Traditional Unix Permissions:**
- **Owner (u):** user who owns the file/directory
- **Group (g):** group associated със file/directory  
- **Others (o):** everyone else на system
- **Permission types:** read (r), write (w), execute (x)

**Numeric Permission Representation:**
```bash
# Permission calculation
r = 4, w = 2, x = 1
755 = rwxr-xr-x (owner: rwx, group: r-x, others: r-x)
644 = rw-r--r-- (owner: rw-, group: r--, others: r--)
600 = rw------- (owner: rw-, group: ---, others: ---)
```

**Special Permissions:**
- **Setuid (4000):** execute файл със owner privileges
- **Setgid (2000):** execute със group privileges или inherit group ownership
- **Sticky bit (1000):** only owner can delete files в directory (e.g., /tmp)

**🏠 File System Security Features (7 мин)**

**File System Types и Security:**
- **ext4:** Extended attributes, ACLs, encryption support
- **XFS:** Advanced ACLs, quotas, high performance
- **Btrfs:** Snapshots, checksums, compression
- **ZFS:** Built-in encryption, integrity checking, snapshots

**Extended Attributes:**
- **Security labels:** SELinux, SMACK security contexts
- **User attributes:** custom metadata storage
- **System attributes:** filesystem-specific information
- **Trusted attributes:** kernel-controlled extended attributes

**Mount Options Security:**
- **noexec:** prevent execution на files от mounted filesystem
- **nosuid:** ignore setuid/setgid bits
- **nodev:** prevent device file access
- **ro (read-only):** mount filesystem като read-only

**Access Control Lists (ACLs):**
- **Extended permissions:** beyond traditional owner/group/other
- **Named users/groups:** specific permissions за individual users
- **Default ACLs:** inherited permissions за new files
- **Mask entries:** maximum effective permissions

### **ЧАС 2: User management и sudo configuration (40 мин)**

#### **Теоретична част (20 мин):**

**👤 Linux User Management (10 мин)**

**User Types:**
- **Root user (UID 0):** superuser със unlimited privileges
- **System users (UID 1-999):** service accounts за daemons и applications  
- **Regular users (UID 1000+):** human users със limited privileges
- **Service accounts:** specialized users за specific applications

**User Database Files:**
- **/etc/passwd:** user account information (username, UID, GID, home directory)
- **/etc/shadow:** encrypted passwords и aging information
- **/etc/group:** group definitions и memberships
- **/etc/gshadow:** group passwords (rarely used)

**Password Security:**
- **Hashing algorithms:** SHA-256, SHA-512, bcrypt, scrypt
- **Salt values:** random data added to prevent rainbow table attacks  
- **Password aging:** expiration, minimum age, warning periods
- **Account lockout:** protection срещу brute force attacks

**🔧 Sudo System Deep Dive (10 мин)**

**Sudo Architecture:**
- **sudoers file:** /etc/sudoers configuration file
- **visudo command:** safe editing на sudoers file
- **sudo log:** recording на privileged command execution
- **timestamp caching:** temporary privilege elevation

**Sudoers File Syntax:**
```bash
# User privilege specification
username    hostname=(runas_user:runas_group) COMMANDS

# Examples
alice       ALL=(ALL:ALL) ALL                    # Full sudo access
bob         webserver=(www-data) /usr/sbin/nginx # Specific command as specific user
%admin      ALL=(ALL) NOPASSWD: /bin/systemctl   # Group access без password
```

**Sudo Best Practices:**
- **Principle of least privilege:** grant minimum necessary permissions
- **Command aliases:** group related commands за easier management
- **Host aliases:** apply rules to specific systems
- **User aliases:** group users със similar needs
- **Logging и auditing:** track privileged command usage

**Advanced Sudo Features:**
- **Secure path:** controlled PATH environment variable
- **Command restrictions:** limiting arguments to commands
- **Environment handling:** controlling environment variable inheritance
- **Timeout settings:** how long sudo privileges remain cached

### **ЧАС 3: File permissions и access control (chmod, chown, ACL) (40 мин)**

#### **Теоретична част (18 мин):**

**📁 Advanced File Permissions (10 мин)**

**Standard Permission Management:**
```bash
# chmod syntax variations
chmod u+x filename          # Add execute за owner
chmod g-w filename          # Remove write за group  
chmod o=r filename          # Set others to read-only
chmod a+r filename          # Add read за all (owner, group, others)

# Numeric permissions
chmod 755 filename          # rwxr-xr-x
chmod 644 filename          # rw-r--r--
chmod 600 filename          # rw-------
```

**Directory Permissions:**
- **Read (r):** list directory contents
- **Write (w):** create/delete files в directory
- **Execute (x):** access directory (cd into it)
- **Directory traversal:** need execute permission на all parent directories

**Special File Attributes:**
```bash
# Extended file attributes (chattr/lsattr)
chattr +i filename          # Immutable (cannot be modified/deleted)
chattr +a filename          # Append-only (can only be written in append mode)
chattr +s filename          # Secure deletion (overwrite с zeros when deleted)
lsattr filename             # List current attributes
```

**🔒 Access Control Lists (ACLs) (8 мин)**

**ACL Concepts:**
- **Extended permissions:** more granular than traditional Unix permissions
- **Named users/groups:** specific permissions за individual users
- **Mask entry:** maximum effective permissions
- **Default ACLs:** inherited permissions за новосъздадени files

**ACL Commands:**
```bash
# Set ACL permissions
setfacl -m u:username:rwx filename       # Give user rwx access
setfacl -m g:groupname:r-x dirname       # Give group rx access to directory
setfacl -m m:r-x filename               # Set mask (maximum permissions)

# Default ACLs за directories
setfacl -d -m u:username:rwx dirname     # Default ACL за new files

# View ACLs
getfacl filename                        # Show all ACL entries
ls -la filename                         # Shows '+' if ACLs present

# Remove ACLs  
setfacl -x u:username filename          # Remove specific user ACL
setfacl -b filename                     # Remove all ACLs
```

**ACL Inheritance:**
- **Default ACLs:** applied to новосъздадени files и directories
- **Mask calculation:** automatic adjustment на effective permissions
- **ACL precedence:** order на evaluation (owner → named users → group → others)

### **ЧАС 4: iptables firewall и network security (40 мин)**

#### **Теоретична част (22 мин):**

**🔥 iptables Architecture (12 мин)**

**Netfilter Framework:**
- **Kernel-level packet filtering:** operates at kernel level за performance
- **Tables:** filter, nat, mangle, raw (different packet processing stages)
- **Chains:** INPUT, OUTPUT, FORWARD (packet flow directions)  
- **Rules:** specific instructions за packet handling
- **Targets:** actions taken when rule matches (ACCEPT, DROP, REJECT, LOG)

**iptables Tables:**
- **filter table (default):** packet filtering decisions
  - INPUT chain: packets destined за local system
  - OUTPUT chain: packets originated от local system  
  - FORWARD chain: packets routed through system
- **nat table:** Network Address Translation
  - PREROUTING, POSTROUTING, OUTPUT chains
- **mangle table:** packet modification
- **raw table:** connection tracking exemptions

**Rule Processing:**
- **Sequential evaluation:** rules processed в order until match found
- **Default policy:** action taken if no rules match
- **Jump targets:** redirect to user-defined chains
- **Stateful filtering:** connection tracking със NEW, ESTABLISHED, RELATED states

**🛡️ Common Security Configurations (10 мин)**

**Basic Firewall Strategies:**
- **Default deny:** block all traffic, explicitly allow necessary services
- **Stateful filtering:** allow established connections, block new unauthorized attempts
- **Service-specific rules:** granular control by port и protocol
- **Source-based filtering:** allow/deny by IP address or network

**Essential iptables Commands:**
```bash
# View current rules
iptables -L -n -v                    # List all rules със packet counters
iptables -L INPUT -n --line-numbers  # Show INPUT rules със line numbers

# Basic rule examples
iptables -A INPUT -p tcp --dport 22 -j ACCEPT     # Allow SSH
iptables -A INPUT -p tcp --dport 80 -j ACCEPT     # Allow HTTP
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT  # Stateful

# Default policies
iptables -P INPUT DROP               # Default deny incoming
iptables -P FORWARD DROP             # Default deny forwarding
iptables -P OUTPUT ACCEPT            # Default allow outgoing
```

**Advanced Security Features:**
- **Rate limiting:** prevent DoS attacks с connection limits
- **Port knocking:** hidden service access through sequence на connection attempts
- **Geographic blocking:** filter by country-based IP ranges
- **Application layer filtering:** deep packet inspection capabilities

### **ЧАС 7: SELinux/AppArmor mandatory access control (40 мин)**

#### **Теоретична част (22 мин):**

**🔒 Mandatory Access Control Overview (10 мин)**

**MAC vs DAC:**
- **Discretionary Access Control (DAC):** traditional Unix permissions, owner controls access
- **Mandatory Access Control (MAC):** system-wide policies override user preferences
- **MAC benefits:** prevent privilege escalation, contain compromised applications
- **Policy enforcement:** centralized security policy application

**SELinux (Security-Enhanced Linux):**
- **Developed by NSA:** originally created за government security needs
- **Label-based access control:** every file, process, и resource has security context
- **Policy types:** targeted (default), strict, MLS (Multi-Level Security)
- **Modes:** Enforcing (active), Permissive (log-only), Disabled

**AppArmor (Application Armor):**
- **Path-based access control:** controls access based на file paths
- **Profile-driven:** each application has specific profile defining allowed actions
- **Simpler than SELinux:** easier to understand и configure
- **Ubuntu/SUSE default:** standard в Ubuntu и SUSE distributions

**🛡️ SELinux Deep Dive (12 мин)**

**SELinux Components:**
- **Security contexts:** user:role:type:level labels на objects
- **Type enforcement:** rules defining interaction between types  
- **Role-based access:** users assigned specific roles
- **Multi-level security:** classifications и clearances (optional)

**SELinux Modes и Configuration:**
```bash
# Check SELinux status
getenforce                 # Current mode (Enforcing/Permissive/Disabled)
sestatus                   # Detailed SELinux information
selinux-config-tool        # GUI configuration (if available)

# SELinux contexts
ls -laZ                    # View file security contexts
ps -eZ                     # View process security contexts
id -Z                      # View user security context
```

**Common SELinux Commands:**
```bash
# Context management
chcon -t httpd_exec_t /path/to/file    # Change file context
restorecon /path/to/file               # Restore default context
setsebool httpd_can_network_connect on # Change boolean policy

# Policy management  
getsebool -a              # List all boolean policies
semanage fcontext -l      # List file context mappings
audit2allow -w -a         # Analyze AVC denial messages
```

**AppArmor Fundamentals:**
```bash
# AppArmor status
sudo apparmor_status      # Current profiles и mode
aa-enabled               # Check if AppArmor is enabled

# Profile management
sudo aa-enforce /path/to/profile    # Set profile to enforce mode
sudo aa-complain /path/to/profile   # Set profile to complain mode  
sudo aa-disable /path/to/profile    # Disable profile
```
