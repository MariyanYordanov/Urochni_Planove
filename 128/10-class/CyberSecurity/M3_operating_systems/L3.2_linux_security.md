### **ЧАС 1: Linux security архитектура и основни концепции (40 мин)**

#### **Теоретична част (25 мин):**

**Linux Security Model Overview (10 мин)**

**Unix Security Heritage:**
- **Принцип на минималните привилегии:** потребителите получават минимално необходимите права
- **Всичко е файл:** унифициран модел за контрол на достъп за всички ресурси
- **Multi-user дизайн:** вградено разделяне между потребителски контексти
- **Предимство на open source:** подобрения на security и прозрачност, задвижвани от общността

**Linux Security Architecture:**
- **Kernel space vs User space:** ясно разделяне с контролирани интерфейси
- **System calls:** сигурна комуникация между потребителски програми и kernel
- **Process isolation:** всеки процес работи в собствено защитено пространство в паметта
- **Capabilities system:** прецизни привилегии вместо all-or-nothing root

**Core Security Components:**
- **Authentication:** верификация на потребителска идентичност (пароли, ключове, сертификати)
- **Authorization:** контрол на достъп до ресурси базиран на идентичност
- **Accounting:** логване и одитиране на потребителски действия
- **Cryptography:** защита на данните чрез криптиране и подписване

**Permission Model Deep Dive (8 мин)**

**Traditional Unix Permissions:**
- **Owner (u):** потребител, притежаващ файла/директорията
- **Group (g):** група, свързана с файла/директорията
- **Others (o):** всички останали на системата
- **Permission types:** read (r), write (w), execute (x)

**Numeric Permission Representation:**
```bash
# Permission calculation
r = 4, w = 2, x = 1
755 = rwxr-xr-x (owner: rwx, group: r-x, others: r-x)
644 = rw-r--r-- (owner: rw-, group: r--, others: r--)
600 = rw------- (owner: rw-, group: ---, others: ---)
```

**Special Permissions:**
- **Setuid (4000):** изпълнение на файл с привилегиите на собственика
- **Setgid (2000):** изпълнение с привилегиите на групата или наследяване на собствеността на групата
- **Sticky bit (1000):** само собственикът може да изтрива файлове в директорията (напр. /tmp)

**File System Security Features (7 мин)**

**File System Types и Security:**
- **ext4:** разширени атрибути, ACLs, поддръжка на криптиране
- **XFS:** разширени ACLs, квоти, висока производителност
- **Btrfs:** snapshots, checksums, компресия
- **ZFS:** вградено криптиране, проверка на целостта, snapshots

**Extended Attributes:**
- **Security labels:** контексти за security на SELinux, SMACK
- **User attributes:** съхранение на потребителски метаданни
- **System attributes:** специфична за файловата система информация
- **Trusted attributes:** разширени атрибути, контролирани от kernel

**Mount Options Security:**
- **noexec:** предотвратяване на изпълнение на файлове от монтирана файлова система
- **nosuid:** игнориране на setuid/setgid битове
- **nodev:** предотвратяване на достъп до device файлове
- **ro (read-only):** монтиране на файловата система като read-only

**Access Control Lists (ACLs):**
- **Разширени разрешения:** отвъд традиционните Unix разрешения owner/group/other
- **Наименувани потребители/групи:** специфични разрешения за отделни потребители
- **Default ACLs:** наследени разрешения за нови файлове
- **Mask entries:** максимални ефективни разрешения

### **ЧАС 2: User management и sudo конфигурация (40 мин)**

#### **Теоретична част (20 мин):**

**Linux User Management (10 мин)**

**User Types:**
- **Root user (UID 0):** superuser с неограничени привилегии
- **System users (UID 1-999):** служебни акаунти за daemons и приложения
- **Regular users (UID 1000+):** обикновени потребители с ограничени привилегии
- **Service accounts:** специализирани потребители за конкретни приложения

**User Database Files:**
- **/etc/passwd:** информация за потребителски акаунти (потребителско име, UID, GID, home директория)
- **/etc/shadow:** криптирани пароли и информация за остаряване
- **/etc/group:** дефиниции на групи и членства
- **/etc/gshadow:** групови пароли (рядко използвани)

**Password Security:**
- **Hashing алгоритми:** SHA-256, SHA-512, bcrypt, scrypt
- **Salt стойности:** случайни данни, добавени за предотвратяване на rainbow table атаки
- **Password aging:** изтичане, минимална възраст, периоди на предупреждение
- **Account lockout:** защита срещу brute force атаки

**Sudo System Deep Dive (10 мин)**

**Sudo Architecture:**
- **sudoers file:** конфигурационен файл /etc/sudoers
- **visudo command:** безопасно редактиране на sudoers файл
- **sudo log:** записване на изпълнението на привилегировани команди
- **timestamp caching:** временно повишаване на привилегии

**Sudoers File Syntax:**
```bash
# User privilege specification
username    hostname=(runas_user:runas_group) COMMANDS

# Examples
alice       ALL=(ALL:ALL) ALL                    # Full sudo access
bob         webserver=(www-data) /usr/sbin/nginx # Specific command as specific user
%admin      ALL=(ALL) NOPASSWD: /bin/systemctl   # Group access без password
```

**Sudo Best Practices:**
- **Принцип на минималните привилегии:** предоставяне на минимално необходимите разрешения
- **Command aliases:** групиране на свързани команди за по-лесно управление
- **Host aliases:** прилагане на правила към конкретни системи
- **User aliases:** групиране на потребители с подобни нужди
- **Логване и одитиране:** проследяване на използването на привилегировани команди

**Advanced Sudo Features:**
- **Secure path:** контролирана environment variable PATH
- **Command restrictions:** ограничаване на аргументите към команди
- **Environment handling:** контрол на наследяването на environment variables
- **Timeout settings:** колко дълго остават кешираните sudo привилегии

### **ЧАС 3: Файлови разрешения и контрол на достъп (chmod, chown, ACL) (40 мин)**

#### **Теоретична част (18 мин):**

**Advanced File Permissions (10 мин)**

**Standard Permission Management:**
```bash
# chmod syntax variations
chmod u+x filename          # Add execute за owner
chmod g-w filename          # Remove write за group
chmod o=r filename          # Set others to read-only
chmod a+r filename          # Add read за all (owner, group, others)

# Numeric permissions
chmod 755 filename          # rwxr-xr-x
chmod 644 filename          # rw-r--r--
chmod 600 filename          # rw-------
```

**Directory Permissions:**
- **Read (r):** изброяване на съдържанието на директорията
- **Write (w):** създаване/изтриване на файлове в директорията
- **Execute (x):** достъп до директорията (влизане в нея)
- **Directory traversal:** нужно е разрешение за изпълнение на всички родителски директории

**Special File Attributes:**
```bash
# Extended file attributes (chattr/lsattr)
chattr +i filename          # Immutable (cannot be modified/deleted)
chattr +a filename          # Append-only (can only be written in append mode)
chattr +s filename          # Secure deletion (overwrite с zeros when deleted)
lsattr filename             # List current attributes
```

**Access Control Lists (ACLs) (8 мин)**

**ACL Concepts:**
- **Разширени разрешения:** по-прецизни от традиционните Unix разрешения
- **Наименувани потребители/групи:** специфични разрешения за отделни потребители
- **Mask entry:** максимални ефективни разрешения
- **Default ACLs:** наследени разрешения за новосъздадени файлове

**ACL Commands:**
```bash
# Set ACL permissions
setfacl -m u:username:rwx filename       # Give user rwx access
setfacl -m g:groupname:r-x dirname       # Give group rx access to directory
setfacl -m m:r-x filename               # Set mask (maximum permissions)

# Default ACLs за directories
setfacl -d -m u:username:rwx dirname     # Default ACL за new files

# View ACLs
getfacl filename                        # Show all ACL entries
ls -la filename                         # Shows '+' if ACLs present

# Remove ACLs
setfacl -x u:username filename          # Remove specific user ACL
setfacl -b filename                     # Remove all ACLs
```

**ACL Inheritance:**
- **Default ACLs:** прилагани към новосъздадени файлове и директории
- **Mask calculation:** автоматично коригиране на ефективните разрешения
- **ACL precedence:** ред на оценяване (owner → наименувани потребители → група → others)

### **ЧАС 4: iptables firewall и мрежова security (40 мин)**

#### **Теоретична част (22 мин):**

**iptables Architecture (12 мин)**

**Netfilter Framework:**
- **Филтриране на пакети на kernel ниво:** работи на ниво kernel за производителност
- **Tables:** filter, nat, mangle, raw (различни етапи на обработка на пакети)
- **Chains:** INPUT, OUTPUT, FORWARD (посоки на потока на пакетите)
- **Rules:** специфични инструкции за обработка на пакети
- **Targets:** действия, предприети когато правилото съвпадне (ACCEPT, DROP, REJECT, LOG)

**iptables Tables:**
- **filter table (default):** решения за филтриране на пакети
  - INPUT chain: пакети, предназначени за локалната система
  - OUTPUT chain: пакети, изпратени от локалната система
  - FORWARD chain: пакети, маршрутизирани през системата
- **nat table:** Network Address Translation
  - PREROUTING, POSTROUTING, OUTPUT chains
- **mangle table:** модификация на пакети
- **raw table:** изключения от проследяването на връзки

**Rule Processing:**
- **Последователно оценяване:** правилата се обработват по ред до намиране на съвпадение
- **Default policy:** действие, предприето ако никое правило не съвпадне
- **Jump targets:** пренасочване към дефинирани от потребителя chains
- **Stateful filtering:** проследяване на връзки с NEW, ESTABLISHED, RELATED състояния

**Common Security Configurations (10 мин)**

**Basic Firewall Strategies:**
- **Default deny:** блокиране на целия трафик, изрично разрешаване на необходимите услуги
- **Stateful filtering:** разрешаване на установени връзки, блокиране на нови неоторизирани опити
- **Service-specific rules:** прецизен контрол по порт и протокол
- **Source-based filtering:** разрешаване/отказване по IP адрес или мрежа

**Essential iptables Commands:**
```bash
# View current rules
iptables -L -n -v                    # List all rules със packet counters
iptables -L INPUT -n --line-numbers  # Show INPUT rules със line numbers

# Basic rule examples
iptables -A INPUT -p tcp --dport 22 -j ACCEPT     # Allow SSH
iptables -A INPUT -p tcp --dport 80 -j ACCEPT     # Allow HTTP
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT  # Stateful

# Default policies
iptables -P INPUT DROP               # Default deny incoming
iptables -P FORWARD DROP             # Default deny forwarding
iptables -P OUTPUT ACCEPT            # Default allow outgoing
```

**Advanced Security Features:**
- **Rate limiting:** предотвратяване на DoS атаки с ограничения на връзките

### **ЧАС 7: SELinux/AppArmor задължителен контрол на достъп (40 мин)**

#### **Теоретична част (22 мин):**

**Mandatory Access Control Overview (10 мин)**

**MAC vs DAC:**
- **Discretionary Access Control (DAC):** традиционни Unix разрешения, собственикът контролира достъпа
- **Mandatory Access Control (MAC):** системните политики заменят потребителските предпочитания
- **MAC ползи:** предотвратяване на повишаване на привилегии, задържане на компрометирани приложения
- **Policy enforcement:** централизирано прилагане на security политика

**SELinux (Security-Enhanced Linux):**
- **Разработен от NSA:** първоначално създаден за правителствени нужди за security
- **Label-based access control:** всеки файл, процес и ресурс има security контекст
- **Policy types:** targeted (по подразбиране), strict, MLS (Multi-Level Security)
- **Modes:** Enforcing (активен), Permissive (само логване), Disabled

**AppArmor (Application Armor):**
- **Path-based access control:** контролира достъпа базиран на файловите пътища
- **Profile-driven:** всяко приложение има специфичен профил, дефиниращ разрешените действия
- **По-прост от SELinux:** по-лесен за разбиране и конфигуриране
- **Ubuntu/SUSE default:** стандарт в Ubuntu и SUSE дистрибуции

**SELinux Deep Dive (12 мин)**

**SELinux Components:**
- **Security contexts:** user:role:type:level етикети на обектите
- **Type enforcement:** правила, дефиниращи взаимодействието между типове
- **Role-based access:** потребители, присвоени към конкретни роли
- **Multi-level security:** класификации и допускания (по избор)

**SELinux Modes и Configuration:**

### **ЧАС 8: Linux incident response и forensics (40 мин)**

#### **Теоретична част (20 мин):**

**Linux Incident Response Process (12 мин)**

**Incident Detection Indicators:**
- **Performance anomalies:** необичайно използване на CPU, памет или I/O
- **Network activity:** неочаквани връзки или трансфери на данни
- **Authentication issues:** неуспешни опити за влизане, подозрителна активност на акаунти
- **File system changes:** неоторизирани модификации на файлове, изтривания
- **Process anomalies:** непознати процеси, опити за повишаване на привилегии

**Evidence Collection Priorities:**
1. **Volatile data:** дъмпове на паметта, изпълнявани процеси, мрежови връзки
2. **System state:** влезли потребители, отворени файлове, системна конфигурация
3. **Log files:** системни логове, логове на приложения, security events
4. **File system:** модификации на файлове, изтрити файлове, скрити файлове
5. **Network evidence:** заснемане на пакети, логове на връзки

**Incident Response Workflow:**
1. **Detection:** идентифициране на потенциален security incident
2. **Analysis:** определяне на обхвата и характера на incident
3. **Containment:** изолиране на засегнатите системи
4. **Eradication:** премахване на заплахи от средата
5. **Recovery:** възстановяване на системите към нормална работа
6. **Lessons learned:** подобряване на security въз основа на констатациите

1. **Multiple choice (4т):** В Linux permission model, chmod 755 означава:
   - A) rwxr-xr-x (owner: rwx, group: r-x, others: r-x)
   - B) rwxrwxr-x (owner: rwx, group: rwx, others: r-x)
   - C) r-xr-xr-x (всички: r-x само)
   - D) rwx------ (owner: rwx само)

2. **Command identification (4т):** Свържете командите с тяхната основна security функция:
   - `sudo visudo` → Безопасно редактиране на sudoers файл
   - `iptables -L` → Изброяване на firewall правила
   - `getfacl filename` → Показване на Access Control Lists
   - `setenforce 1` → Активиране на SELinux enforcing mode

3. **Short answer (4т):** Обяснете разликата между DAC (Discretionary Access Control) и MAC (Mandatory Access Control) в Linux security context.

**Section B: Practical Application (15т)**

4. **Command sequence (8т):** Потребител се нуждае от временен sudo достъп само за рестартиране на nginx услугата. Напишете пълния sudoers запис и обяснете всеки компонент:
   ```
   username hostname=(root) NOPASSWD: /usr/sbin/systemctl restart nginx
   ```
   *Обяснете всяка част на този entry и защо NOPASSWD може или не може да е подходящо.*
