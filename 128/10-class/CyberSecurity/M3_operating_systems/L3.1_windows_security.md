### **ЧАС 1: Windows Security архитектура и основни компоненти (40 мин)**

#### **Теоретична част (25 мин):**

**Windows Security Architecture Overview (10 мин)**

**Security Reference Monitor (SRM):**
- **Core component:** централният security enforcement механизъм
- **Object access control:** проверява разрешенията за всички системни ресурси
- **Audit logging:** записва security events за мониторинг
- **Trusted Computing Base (TCB):** най-критичните security компоненти

**Windows Security Subsystems:**
- **Local Security Authority (LSA):** услуги за автентикация и авторизация
- **Security Accounts Manager (SAM):** база данни за локални потребителски акаунти
- **Active Directory (AD):** управление на идентичности на enterprise ниво
- **Windows Security Center:** унифициран мониторинг на security статус

**Security Identifiers (SID):**
- **Уникална идентификация:** всеки потребител, група, компютър има уникален SID
- **Формат на структурата:** S-1-5-domain-RID (пример: S-1-5-21-123456789-987654321-555666777-1001)
- **Well-known SIDs:** вградени акаунти като Administrator, Everyone, Authenticated Users
- **SID наследяване:** как разрешенията се предават в обектната йерархия

**Access Control Model (8 мин)**

**Discretionary Access Control Lists (DACL):**
- **Access Control Entries (ACE):** индивидуални записи за разрешения
- **Типове разрешения:** Full Control, Modify, Read & Execute, Read, Write
- **Разширени разрешения:** детайлни опции за прецизен контрол
- **Правила за наследяване:** как разрешенията преминават надолу по структурата на папки/файлове

**Security Descriptors:**
- **Информация за собственик:** кой притежава обекта
- **Основна група:** група по подразбиране за нови обекти
- **DACL:** кой има какви разрешения
- **System Access Control List (SACL):** конфигурация за одитиране

**Token-based Security:**
- **Access tokens:** security контекст за всеки изпълняван процес
- **Съдържание на token:** потребителски SID, членства в групи, привилегии
- **Impersonation:** процес, действащ от името на различен потребител
- **Elevated tokens:** контекст на администратор спрямо стандартен потребител

**Built-in Security Features (7 мин)**

**User Account Control (UAC):**
- **Принцип на минималните привилегии:** потребителите работят със стандартни привилегии по подразбиране
- **Elevation prompts:** искане за администраторски достъп при необходимост
- **Admin Approval Mode:** администраторите използват стандартни токени до повишаване на привилегии
- **UAC нива:** Always notify, Notify without dimming, Never notify

**Windows Defender:**
- **Real-time protection:** непрекъснато сканиране за злонамерен софтуер
- **Cloud-delivered protection:** използване на разузнавателни данни за заплахи от Microsoft
- **Automatic sample submission:** подпомагане на подобряването на глобалната защита
- **Интеграция:** вграден в Windows Security Center

**BitLocker Drive Encryption:**
- **Пълно криптиране на диска:** защита на данните в покой
- **TPM интеграция:** хардуерна защита на ключовете
- **Recovery keys:** резервни методи за достъп
- **Network unlock:** автоматично декриптиране в корпоративни среди

**Windows Firewall:**
- **Stateful packet inspection:** наблюдение на стиловете на свързване
- **Profile-based rules:** различни настройки за public/private/domain мрежи
- **Inbound/outbound filtering:** контрол на трафика в двете посоки
- **Application-based rules:** разрешения по програма

### **ЧАС 2: User Account Control (UAC) и управление на привилегии (40 мин)**

#### **Теоретична част (20 мин):**

**UAC Deep Dive (10 мин)**

**UAC Architecture:**
- **Application Information Service (AIS):** обработва заявки за повишаване на привилегии
- **Consent UI:** диалогови прозорци за повишаване на привилегии
- **Secure Desktop:** защитена среда за UAC подкани
- **Virtualization:** пренасочване на достъпа до файлове/регистър за legacy приложения

**UAC Levels Explained:**
1. **Always notify:** максимална security, подканва при всички опити за повишаване на привилегии
2. **Notify me only when apps try to make changes (default):** баланс между security и usability
3. **Notify me only when apps try to make changes (no desktop dimming):** малко по-малко сигурно
4. **Never notify:** минимална security, без подкани за повишаване на привилегии

**UAC Bypass Techniques (Осведоменост):**
- **Eventvwr.exe exploit:** използване на манипулация на регистъра на Event Viewer
- **Fodhelper.exe:** експлоатация на помощника Features on Demand
- **ComputerDefaults.exe:** манипулация на програмите по подразбиране
- *Важно:* Тези техники съществуват за осведоменост; съществуват и легитимни административни инструменти

**Windows Privileges System (10 мин)**

**Built-in Privileges:**
- **SeDebugPrivilege:** отстраняване на грешки в други процеси
- **SeBackupPrivilege:** архивиране на файлове независимо от разрешенията
- **SeRestorePrivilege:** възстановяване на файлове независимо от разрешенията
- **SeTakeOwnershipPrivilege:** поемане на собственост върху всеки обект
- **SeShutdownPrivilege:** изключване на локалната система

**Privilege Assignment:**
- **Group Policy:** присвояване на привилегии за целия домейн
- **Local Security Policy:** присвояване за локалния компютър
- **User Rights Assignment:** свързване на привилегии с потребители/групи
- **Temporary elevation:** придобиване на привилегии по време на изпълнение

**Service Accounts:**
- **LocalSystem:** най-високи привилегии, без мрежови идентификационни данни
- **NetworkService:** способности за мрежова автентикация
- **LocalService:** минимални привилегии, без мрежов достъп
- **Managed Service Accounts:** автоматично управлявани пароли

### **ЧАС 3: Windows Authentication методи и протоколи (40 мин)**

#### **Теоретична част (22 мин):**

**Authentication Mechanisms (12 мин)**

**Local Authentication:**
- **SAM Database:** Security Accounts Manager за локални акаунти
- **Password hashing:** съхранение на NTLM хеш (базиран на MD4)
- **Политики за пароли:** изисквания за сложност, дължина, история
- **Account lockout:** защита срещу brute force атаки

**Domain Authentication:**
- **Kerberos:** предпочитан authentication протокол за Active Directory
- **NTLM (NT LAN Manager):** legacy протокол, все още използван за обратна съвместимост
- **Ticket Granting Service (TGS):** управление на Kerberos tickets
- **Authentication flow:** клиент → domain controller → целева услуга

**Modern Authentication:**
- **Windows Hello:** интеграция на биометрична автентикация
- **PIN authentication:** специфични за устройство PIN кодове
- **Smart card authentication:** влизане базирано на сертификати
- **Multi-factor authentication:** комбиниране на множество фактори за автентикация

**Kerberos Protocol Deep Dive (10 мин)**

**Kerberos Components:**
- **Key Distribution Center (KDC):** централен сървър за автентикация
- **Authentication Server (AS):** първоначална автентикация на потребителя
- **Ticket Granting Server (TGS):** издаване на service tickets
- **Principal:** потребител или услуга, нуждаеща се от автентикация

**Kerberos Authentication Flow:**
1. **AS Request:** Потребителят иска authentication ticket от KDC
2. **AS Reply:** KDC връща Ticket Granting Ticket (TGT)
3. **TGS Request:** Потребителят иска service ticket използвайки TGT
4. **TGS Reply:** KDC връща service ticket
5. **Service Request:** Потребителят получава достъп до услугата с service ticket
6. **Service Response:** Услугата валидира ticket и предоставя достъп

**Kerberos Security Benefits:**
- **Mutual authentication:** и клиентът, и сървърът верифицират идентичността
- **Single sign-on:** едно влизане предоставя достъп до множество услуги
- **Ticket expiration:** ограниченото времево валидиране намалява риска
- **Без предаване на паролата:** паролите никога не се изпращат по мрежата

**Security Considerations:**
- **Ticket lifetime:** балансиране на security с usability
- **Clock synchronization:** Kerberos изисква синхронизация на времето
- **Delegation:** разрешаване на услуги да действат от името на потребители
- **Golden ticket attacks:** компрометиран KDC може да създаде постоянен достъп

### **ЧАС 4: Windows Firewall конфигурация и network security (40 мин)**

#### **Теоретична част (18 мин):**

**Windows Firewall Architecture (8 мин)**

**Firewall Profiles:**
- **Domain Profile:** когато е свързан с домейн мрежа
- **Private Profile:** доверени мрежи (домашна/работна)
- **Public Profile:** ненадеждни мрежи (летища, кафенета)
- **Profile switching:** автоматично разпознаване базирано на характеристиките на мрежата

**Rule Types:**
- **Inbound rules:** контрол на входящите връзки
- **Outbound rules:** контрол на изходящите връзки
- **Connection security rules:** изисквания за IPSec автентикация
- **Monitoring rules:** проследяване на мрежовата активност

**Advanced Features:**
- **Windows Firewall с Advanced Security:** подобрен интерфейс за управление
- **Group Policy интеграция:** централизирано управление на firewall
- **IPSec интеграция:** криптирани мрежови комуникации
- **Network location awareness:** автоматично превключване на профили

**Network Security Integration (10 мин)**

**Windows Network Stack Security:**
- **SMB (Server Message Block):** security на протокола за споделяне на файлове
- **RDP (Remote Desktop Protocol):** защитен отдалечен достъп
- **WinRM (Windows Remote Management):** security на PowerShell remoting
- **Network Discovery:** контрол на обявяването на услуги

**Common Attack Vectors:**
- **SMB relay attacks:** прихващане и повторно предаване на SMB автентикация
- **RDP brute force:** отгатване на парола срещу remote desktop
- **WinRM exploitation:** уязвимости в PowerShell remoting
- **Network enumeration:** откриване на услуги и споделени ресурси

**Defense Strategies:**
- **Принцип на минималните привилегии:** активирани само необходимите портове/услуги
- **Network segmentation:** изолиране на критичните системи
- **Intrusion detection:** мониторинг за подозрителна мрежова активност
- **Редовно patch:** адресиране на известни уязвимости
