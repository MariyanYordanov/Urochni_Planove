#### **Теоретична част (25 мин):**

**🏗️ Windows Security Architecture Overview (10 мин)**

**Security Reference Monitor (SRM):**
- **Core component:** централният security enforcement механизъм
- **Object access control:** проверява permissions за всички system resources
- **Audit logging:** записва security events за monitoring
- **Trusted Computing Base (TCB):** най-критичните security компоненти

**Windows Security Subsystems:**
- **Local Security Authority (LSA):** authentication и authorization services
- **Security Accounts Manager (SAM):** local user account database
- **Active Directory (AD):** enterprise-level identity management
- **Windows Security Center:** unified security status monitoring

**Security Identifiers (SID):**
- **Unique identification:** всеки user, group, computer има уникален SID
- **Format structure:** S-1-5-domain-RID (exemple: S-1-5-21-123456789-987654321-555666777-1001)
- **Well-known SIDs:** Built-in accounts като Administrator, Everyone, Authenticated Users
- **SID inheritance:** как permissions се предават в object hierarchy

**🔐 Access Control Model (8 мин)**

**Discretionary Access Control Lists (DACL):**
- **Access Control Entries (ACE):** individual permission entries
- **Permission types:** Full Control, Modify, Read & Execute, Read, Write
- **Advanced permissions:** detailed granular control options
- **Inheritance rules:** how permissions flow down folder/file structures

**Security Descriptors:**
- **Owner information:** кой owns the object
- **Primary group:** default group за new objects
- **DACL:** кой има какви permissions
- **System Access Control List (SACL):** auditing configuration

**Token-based Security:**
- **Access tokens:** security context за every running process
- **Token contents:** user SID, group memberships, privileges
- **Impersonation:** process acting on behalf на different user
- **Elevated tokens:** administrator vs standard user contexts

**🛡️ Built-in Security Features (7 мин)**

**User Account Control (UAC):**
- **Principle of least privilege:** users run с standard privileges by default
- **Elevation prompts:** requesting administrator access when needed
- **Admin Approval Mode:** administrators use standard tokens until elevation
- **UAC levels:** Always notify, Notify without dimming, Never notify

**Windows Defender:**
- **Real-time protection:** continuous malware scanning
- **Cloud-delivered protection:** leveraging Microsoft's threat intelligence
- **Automatic sample submission:** helping improve global protection
- **Integration:** built into Windows Security Center

**BitLocker Drive Encryption:**
- **Full disk encryption:** protecting data at rest
- **TPM integration:** hardware-based key protection
- **Recovery keys:** backup access methods
- **Network unlock:** automated decryption в corporate environments

**Windows Firewall:**
- **Stateful packet inspection:** monitoring connection states
- **Profile-based rules:** different settings за public/private/domain networks
- **Inbound/outbound filtering:** controlling traffic в both directions
- **Application-based rules:** permissions per program

### **ЧАС 2: User Account Control (UAC) и privilege управление (40 мин)**

#### **Теоретична част (20 мин):**

**🎭 UAC Deep Dive (10 мин)**

**UAC Architecture:**
- **Application Information Service (AIS):** handles elevation requests  
- **Consent UI:** elevation dialog prompts
- **Secure Desktop:** protected environment за UAC prompts
- **Virtualization:** redirecting legacy app file/registry access

**UAC Levels Explained:**
1. **Always notify:** maximum security, prompts за all elevation attempts
2. **Notify me only when apps try to make changes (default):** balance between security и usability
3. **Notify me only when apps try to make changes (no desktop dimming):** slightly less secure
4. **Never notify:** minimum security, no elevation prompts

**UAC Bypass Techniques (Awareness):**
- **Eventvwr.exe exploit:** using Event Viewer registry manipulation
- **Fodhelper.exe:** exploiting Features on Demand helper
- **ComputerDefaults.exe:** default programs manipulation
- *Important:* These exist за awareness; legitimate admin tools also exist

**🔑 Windows Privileges System (10 мин)**

**Built-in Privileges:**
- **SeDebugPrivilege:** debugging other processes
- **SeBackupPrivilege:** backing up files regardless на permissions
- **SeRestorePrivilege:** restoring files regardless на permissions  
- **SeTakeOwnershipPrivilege:** taking ownership на any object
- **SeShutdownPrivilege:** shutting down local system

**Privilege Assignment:**
- **Group Policy:** domain-wide privilege assignments
- **Local Security Policy:** local computer assignments
- **User Rights Assignment:** mapping privileges към users/groups
- **Temporary elevation:** runtime privilege acquisition

**Service Accounts:**
- **LocalSystem:** highest privileges, no network credentials
- **NetworkService:** network authentication capabilities  
- **LocalService:** minimal privileges, no network access
- **Managed Service Accounts:** automatically managed passwords

### **ЧАС 3: Windows Authentication methods и protocols (40 мин)**

#### **Теоретична част (22 мин):**

**🔐 Authentication Mechanisms (12 мин)**

**Local Authentication:**
- **SAM Database:** Security Accounts Manager за local accounts
- **Password hashing:** NTLM hash storage (MD4-based)
- **Password policies:** complexity, length, history requirements
- **Account lockout:** protection срещу brute force attacks

**Domain Authentication:**
- **Kerberos:** preferred authentication protocol за Active Directory
- **NTLM (NT LAN Manager):** legacy protocol, still used за backwards compatibility
- **Ticket Granting Service (TGS):** Kerberos ticket management
- **Authentication flow:** client → domain controller → target service

**Modern Authentication:**
- **Windows Hello:** biometric authentication integration
- **PIN authentication:** device-specific PIN codes
- **Smart card authentication:** certificate-based login
- **Multi-factor authentication:** combining multiple authentication factors

**🎫 Kerberos Protocol Deep Dive (10 мин)**

**Kerberos Components:**
- **Key Distribution Center (KDC):** централен authentication server
- **Authentication Server (AS):** initial user authentication
- **Ticket Granting Server (TGS):** service ticket issuance
- **Principal:** user или service needing authentication

**Kerberos Authentication Flow:**
1. **AS Request:** User requests authentication ticket from KDC
2. **AS Reply:** KDC returns Ticket Granting Ticket (TGT)
3. **TGS Request:** User requests service ticket using TGT
4. **TGS Reply:** KDC returns service ticket
5. **Service Request:** User accesses service с service ticket
6. **Service Response:** Service validates ticket и grants access

**Kerberos Security Benefits:**
- **Mutual authentication:** both client и server verify identity
- **Single sign-on:** one login provides access to multiple services  
- **Ticket expiration:** limited time validity reduces risk
- **No password transmission:** passwords never sent over network

**Security Considerations:**
- **Ticket lifetime:** balancing security с usability
- **Clock synchronization:** Kerberos requires time synchronization
- **Delegation:** allowing services to act на behalf на users
- **Golden ticket attacks:** compromised KDC can create persistent access

**⚙️ Упражнение 3.1.7: "Security Hardening Implementation" (18 мин)**
**Работа в двойки**

*Scenario:* Implement security hardening за Windows workstation в corporate environment

**Phase 1: Service и Feature Hardening (6 мин)**
*Reduce attack surface through service management:*

**Service Analysis:**
```powershell
# List all running services
Get-Service | Where-Object {$_.Status -eq "Running"} | Sort-Object Name

# Identify unnecessary services
$UnnecessaryServices = @("Fax", "WebClient", "RemoteRegistry", "Telnet")
foreach ($Service in $UnnecessaryServices) {
    $ServiceObj = Get-Service -Name $Service -ErrorAction SilentlyContinue
    if ($ServiceObj) {
        Write-Host "$Service is installed - consider disabling" -ForegroundColor Yellow
    }
}
```

**Feature Management:**
```cmd
# List installed Windows features
dism /online /get-features | findstr Enabled

# Disable unnecessary features (example)
dism /online /disable-feature /featurename:TelnetClient /norestart
dism /online /disable-feature /featurename:TFTP /norestart
```

**Network Service Hardening:**
- Disable SMBv1 protocol
- Configure RDP security settings  
- Review и secure network shares
- Test firewall rule effectiveness

**Phase 2: Account и Authentication Hardening (8 мин)**
*Strengthen user account security:*

**Local Security Policy Configuration:**
Open secpol.msc → Account Policies

*Password Policy Settings:*
- [ ] Minimum password length: 12 characters
- [ ] Password complexity requirements: Enabled
- [ ] Maximum password age: 90 days
- [ ] Enforce password history: 12 passwords remembered

*Account Lockout Policy:*
- [ ] Account lockout threshold: 5 invalid attempts
- [ ] Account lockout duration: 30 minutes
- [ ] Reset account lockout counter after: 30 minutes

**Advanced Authentication Settings:**
```cmd
# Disable Guest account

```

**User Rights Assignment:**

- Configure "Deny log on locally" за service accounts  
- Restrict "Debug programs" privilege

**Phase 3: Registry и System Hardening (4 мин)**
*Implement advanced security configurations:*

**Registry Security Settings:**
```cmd
# Disable autorun functionality

# Enable UAC registry protection

# Disable anonymous SID enumeration
reg add "HKLM\SYSTEM\CurrentControlSet\Control\Lsa" /v RestrictAnonymousSAM /t REG_DWORD /d 1 /f
```

**System Security Configuration:**
