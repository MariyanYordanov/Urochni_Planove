### **ЧАС 1: Основни концепции - Заплахи, уязвимости и риск**

#### **1.1. Фундаменталните понятия**

В киберсигурността три термина формират основата на всичко останало:

**ЗАПЛАХА (THREAT)**
*Определение:* Всяко потенциално събитие или действие, което може да причини вреда на информационен актив.

*Характеристики:*
- Има **мотивация** (финансова, политическа, лична)
- Притежава **възможности** (technical skills, ресурси)
- Търси **възможности** (уязвимости за експлоатация)

*Примери:*
- **Hacker** - технически експерт с злонамерени цели
- **Malware** - злонамерен софтуер (вирус, троян, ransomware)  
- **Природно бедствие** - наводнение, земетресение, пожар
- **Human error** - грешки на служителите
- **Hardware failure** - повреди на сървъри, дискове

**УЯЗВИМОСТ (VULNERABILITY)**
*Определение:* Слабост в система, процес или контрол, която може да бъде експлоатирана от заплаха.

*Типове:*
- **Техническа** - software bugs, misconfiguration
- **Физическа** - незаключени врати, липса на камери
- **Административна** - слаби политики, липса на training
- **Човешка** - social engineering susceptibility

*Примери:*
- Unpatched software с известни уязвимости
- Слаби пароли или default credentials  
- Неконфигуриран firewall
- Необучени служители, податливи на phishing
- Physical access без контрол

**РИСК (RISK)**
*Определение:* Вероятността заплаха да експлоатира уязвимост и потенциалното въздействие.

*Формула:* **РИСК = ЗАПЛАХА × УЯЗВИМОСТ × ВЪЗДЕЙСТВИЕ**

*Матрица на риска:*
```
ВЪЗДЕЙСТВИЕ    |  НИСКА    |  СРЕДНА   |  ВИСОКА   |
               |ВЕРОЯТНОСТ |ВЕРОЯТНОСТ |ВЕРОЯТНОСТ |
─────────────────────────────────────────────────────
ВИСОКО         |   СРЕДНО  |   ВИСОКО  |  КРИТИЧНО |
СРЕДНО         |   НИСКО   |   СРЕДНО  |   ВИСОКО  |
НИСКО          |   НИСКО   |   НИСКО   |   СРЕДНО  |
```

### **ЧАС 2: Типология на заплахите**

#### **2.1. Класификация по source/origin**

**ВЪТРЕШНИ ЗАПЛАХИ (Insider Threats)**
*80% от incidents включват insider component*

**Типове insiders:**
- **Malicious insider** - служител със злонамерени цели
- **Compromised insider** - хакнат служител (credentials са компрометирани)
- **Careless insider** - служител, който прави грешки

*Case Study:* **Edward Snowden (2013)**
- NSA contractor със legitimate access
- Използва trusted position за exfiltration на класифицирана информация
- Последствия: $35 милиона директни загуби + неизчислими репутационни щети

**ВЪНШНИ ЗАПЛАХИ (External Threats)**

**Видове external actors:**

**1. Script Kiddies**
- *Характеристики:* Ниски technical skills, използват готови tools
- *Мотивация:* Fun, престиж в hacker communities
- *Methods:* Automated attacks, известни exploits
- *Impact:* Generally low, но могат да cause значително disruption

**2. Hacktivists**  
- *Характеристики:* Politically motivated, moderate skills
- *Мотивация:* Social causes, протест срещу политики
- *Methods:* DDoS, website defacements, data leaks
- *Example:* Anonymous срещу scientology, government sites

**3. Cybercriminals**
- *Характеристики:* High skills, организирани групи
- *Мотивация:* Печалба (ransomware, stolen data, fraud)
- *Methods:* Sophisticated attacks, custom malware
- *Example:* REvil ransomware group - $100+ милиона profits

**4. Nation-State Actors (APTs)**
- *Характеристики:* Unlimited resources, advanced techniques
- *Мотивация:* Political espionage, economic advantage
- *Methods:* Zero-day exploits, multi-year campaigns
- *Example:* APT1 (China), Fancy Bear (Russia), Lazarus (North Korea)

#### **2.2. Класификация по attack vector**

**СОЦИАЛНО ИНЖЕНЕРСТВО (Social Engineering)**
*Определение:* Психологическа манипулация на хора за извършване на действия или разкриване на информация.

**Основни техники:**

**Phishing** - измамни emails
*Статистика:* 96% от phishing се доставя чрез email
*Red flags:*
- Urgent language ("Веднага обновете паролата!")
- Suspicious sender addresses  
- Generic greetings ("Уважаеми клиент")
- Spelling и grammar грешки
- Suspicious links или attachments

**Spear Phishing** - targeted phishing
*Характеристики:*
- Персонализирани съобщения  
- Research за specific victims
- Higher success rate (70% vs 3% за mass phishing)

**Vishing** - voice phishing (телефонни обаждания)
**Smishing** - SMS phishing

**Претexting** - създаване на false scenarios
*Пример:* "Обаждам се от IT отдела, трябва ми паролата ви за maintenance"

**Baiting** - предлагане на нещо привлекательно
*Пример:* USB drive с етикет "Заплати 2024" оставен в паркинга

#### **2.3. Технически заплахи**

**MALWARE СЕМЕЙСТВА**

**Virus**
- *Определение:* Self-replicating код, който се прикача към други програми
- *Infection method:* Изпълнение на заразена програма
- *Damage:* File corruption, system slowdown
- *Example:* ILOVEYOU virus (2000) - $15 милиарда щети

**Trojan**
- *Определение:* Malware disguised като legitimate софтуер
- *Infection method:* Заблуден потребител го инсталира
- *Capabilities:* Backdoors, information theft, remote control
- *Example:* Banking trojans като Zeus

**Worm**  
- *Определение:* Self-propagating malware през network connections
- *Infection method:* Автоматично разпространение
- *Damage:* Network congestion, mass infection
- *Example:* WannaCry ransomware worm (2017)

**Ransomware**
- *Определение:* Malware, който криптира данни и изисква откуп
- *Attack process:*
  1. Initial infection (phishing, exploit kits)
  2. Lateral movement в мрежата  
  3. Data encryption
  4. Ransom note display
  5. Payment negotiation
- *Statistics:* $20 милиарда projected losses за 2024

**Advanced Persistent Threat (APT)**
- *Определение:* Дългосрочна, targeted campaign
- *Characteristics:*
  - Stealth operations (остават скрити месеци/години)
  - Multiple attack vectors
  - Custom tools и zero-days
  - Continuous adaptation
- *Example:* APT1 - Chinese military unit targeting intellectual property

#### **2.4. Attack vectors и methods**

**ТЕХНИЧЕСКИ ЕКСПЛОИТИ**

**Software vulnerabilities:**
- **Buffer overflow** - твърде много данни в memory buffer
- **SQL injection** - злонамерен SQL код в web forms  
- **Cross-site scripting (XSS)** - malicious scripts в web browsers
- **Zero-day exploits** - attacks срещу неизвестни vulnerabilities

**Network-based attacks:**
- **Man-in-the-middle** - intercept communication
- **DDoS** - overwhelming на services с traffic
- **Packet sniffing** - capture на network traffic
- **Rogue access points** - fake WiFi networks

### **ЧАС 3: Реални case studies и анализ**

#### **3.1. Case Study: Target Data Breach (2013)**

**Background:**
Target Corporation - един от най-големите американски retailers със:
- 1,797 stores в САЩ
- 40 милиона credit/debit card accounts компрометирани
- 70 милиона customer records откраднати

**Timeline на атаката:**

**Ноември 15, 2013:** 
- Атакуващите получават достъп чрез **third-party vendor** (Fazio Mechanical Services - HVAC компания)
- Използват **stolen credentials** от малката компания

**Ноември 27, 2013:**
- Започва **installation на malware** в Target's point-of-sale (POS) системи
- Malware: **BlackPOS** - специално designed за кражба на card data

**Ноември 27 - Декември 15, 2013:**
- **Data exfiltration period**
- Всяка транзакция с карта се записва и изпраща към external servers
- 40 милиона транзакции компрометирани

**Декември 13, 2013:**
- **Detection:** Brian Krebs (journalist) уведомява Target за подозрителна активност
- Target започва investigation

**Декември 19, 2013:**
- **Public disclosure** - Target обявява breach-а
- Immediate impact: 46% drop в profits, CEO resignation

**ТЕХНИЧЕСКА АНАЛИЗА:**

**Entry Point:**
- **Vendor network access** - Fazio had remote access за HVAC monitoring
- **Network segmentation failure** - vendor network не е било изолирано от POS systems
- **Credential reuse** - същите credentials за multiple systems

**Lateral Movement:**
- След влизане, attackers use **privilege escalation**  
- **Network reconnaissance** за mapping на infraструктурата
- **Domain admin compromised** - highest privileges obtained

**Data Theft Mechanism:**
- **RAM scraping** - malware чете card data от POS memory
- **Encrypted exfiltration** към external command & control servers
- **Staging servers** в Target network за временно съхранение

**Detection Failures:**
- Target имаше **FireEye security system** - но alerts бяха игнорирани!
- **$1.6 милиона investment** в advanced detection, но poor response процедури
- **Alerts fatigue** - твърде много false positives

#### **3.2. Case Study: WannaCry Ransomware (2017)**

**The Global Crisis:**
На 12 май 2017, света се събуди с **най-голямата ransomware атака** в историята:

**Жертви по география:**
- **300,000+ computers** в 150+ държави
- **UK NHS** - 16,000 appointments отменени, операции postponed
- **Spanish Telefonica** - corporate network парализиран
- **Deutsche Bahn** - train station displays афектирани
- **Renault** - спря production в няколко завода

**Техническите детайли:**

**Exploit Used:** **EternalBlue**
- Разработен от **NSA** за cyber warfare
- Leaked от **Shadow Brokers group** (април 2017)
- Targets Windows SMB protocol vulnerability
- Microsoft пусна patch (MS17-010) през март 2017, но много системи не бяха updated

**Propagation Method:**
```
1. Initial infection (email attachment или website)
2. EternalBlue scan за vulnerable Windows systems
3. Automatic spreading без user interaction  
4. File encryption със strong cryptography
5. Ransom note display: $300-600 в Bitcoin
```

**Kill Switch Discovery:**
- **Marcus Hutchins** (MalwareTech) - 22-годишен UK researcher
- Случайно открива **hardcoded domain** в malware code  
- Registration на domain-а спира разпространението
- Inadvertent героизъм спасява millions of computers

**Lessons Learned:**
- **Patch management** е критично important
- **Legacy systems** са major vulnerability
- **Network segmentation** може да ограничи damage  
- **Backup strategies** са essential за recovery
- **International cooperation** е необходима за cyber threats

**Conti Group (2020-2022):**
- *Business model:* Franchise approach - аренда на ransomware tools

- *Innovation:* Automated affiliate recruitment
- *Scale:* 1,700+ victims globally
- *Tactics:* Double extortion (encryption + data leak threats)
- *Recent:* Major law enforcement takedown (февруари 2024)

**NATION-STATE GROUPS**

**APT29 "Cozy Bear" (Russia, SVR):**
- *Targets:* Government agencies, think tanks, healthcare  
- *Signature:* Living-off-the-land techniques, minimal malware footprint
- *Major operation:* SolarWinds supply chain attack (2020)

**APT1 "Comment Crew" (China, PLA Unit 61398):**
- *Focus:* Intellectual property theft
- *Victims:* 141 organizations, predominantly US companies
- *Timeline:* 6+ years of continuous operations
- *Exposure:* Mandiant report (2013) с detailed attribution

**Lazarus Group (North Korea):**
- *Motivation:* Financial gain за regime funding  
- *Major attacks:* Sony Pictures (2014), WannaCry (2017), $81M Bangladesh Bank robbery
- *Evolution:* От political sabotage към sophisticated financial crimes

#### **4.2. Attack lifecycles и techniques**

**THE CYBER KILL CHAIN (Lockheed Martin Model):**

```
1. RECONNAISSANCE
   ↓ Target selection и information gathering
2. WEAPONIZATION  
   ↓ Create malicious payload
3. DELIVERY
   ↓ Transmit weapon to target (email, web, USB)
4. EXPLOITATION
   ↓ Execute code on victim system  
5. INSTALLATION
   ↓ Install persistent backdoor
6. COMMAND & CONTROL
   ↓ Establish communication channel
7. ACTIONS ON OBJECTIVES
   ↓ Achieve final goal (steal data, destroy systems)
```

**За всеки етап има specific defensive measures!**

### **ЧАС 5: Vulnerability assessment и threat modeling**

#### **5.1. Как откриваме уязвимости?**

**VULNERABILITY ASSESSMENT METHODS:**

**1. Automated Scanning**
*Tools:*
- **Nessus** - network vulnerability scanner
- **OpenVAS** - open source alternative  
- **Qualys** - cloud-based scanning
- **Nmap** - network discovery и port scanning

*Процес:*
- Network discovery → Port scanning → Service detection → Vulnerability identification → Report generation

**2. Manual Testing**  
*Methods:*
- **Code review** - analysis на source code
- **Configuration review** - проверка на system settings
- **Penetration testing** - authorized simulated attacks
- **Social engineering tests** - phishing simulations

**3. Threat Intelligence**
*Sources:*
- **Vendor advisories** - Microsoft, Cisco, Adobe security bulletins

#### **5.3. Introduction към threat hunting**

**PROACTIVE vs REACTIVE SECURITY**

**Traditional approach (Reactive):**
```
ATTACK HAPPENS → DETECTION → INVESTIGATION → RESPONSE → RECOVERY
```

**Modern approach (Proactive):**
```
THREAT INTELLIGENCE → HUNTING → EARLY DETECTION → PREVENTION → CONTINUOUS MONITORING
```

**Basic threat hunting techniques:**

**1. Indicator-based hunting**
- Search за known bad IP addresses, domains, file hashes
- Example: IOCs от recent malware campaigns

**2. Hypothesis-driven hunting**  
- "What if" scenarios based на threat intelligence
- Example: "What if attackers targeting our industry use PowerShell for persistence?"

**3. Behavioral analysis**
- Looking за unusual patterns в network/system behavior
- Example: Large data transfers в unusual hours
